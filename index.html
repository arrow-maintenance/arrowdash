  • Mailing List Summary

The content below is an LLM summary of this month’s activity on the mailing list.

  • Specification clarification for nested validity: Discussion on how to handle validity bitmaps for non-nullable child fields within nullable StructArrays, leaning toward a “liberal reader, conservative writer” approach to maintain compatibility. Thread

  • Proposed “ApplicationData” IPC message type: Proposal to add a new IPC message type for arbitrary opaque bytes to enable embedding logs, signals, or custom metadata within streams without using dummy RecordBatches. Thread

  • Release of Apache Arrow 23.0.1: A maintenance release resolving 27 issues was approved and published, following a discussion on ensuring release integrity through explicit SHA checksums in vote emails. Thread

  • Arrow Rust 58.0.0 release: The Rust implementation reached version 58.0.0, including several breaking API changes and performance improvements. Thread

  • 10-Year Anniversary of Apache Arrow: The community celebrated a decade since the project’s first commit on February 5, 2016, highlighting its growth into a universal standard for columnar data. Thread

  • Release provenance and security: Intense discussion regarding how the project records and verifies the provenance of release candidates, leading to improvements in how checksums are included in public vote threads. Thread

  • IPC format equivalence: Discussion on whether to formalize the requirement that all IPC file format writers produce a sequential stream of messages that can be read by any standard IPC stream reader. Thread

  • Revisiting 24:00:00 support in TIME type: Proposal to support “24:00:00” in Arrow’s Time types to ensure full data fidelity when round-tripping data from databases like PostgreSQL and DuckDB. Thread

  • Deprecation of pyarrow.gandiva and Feather V1: Proposals to deprecate the Gandiva Python bindings and the legacy Feather V1 format due to low maintenance and the availability of modern alternatives. Thread

  • nanoarrow 0.8.0 release: Successful release of the nanoarrow implementation, including 28 resolved issues and improved support for C and R integrations. Thread
