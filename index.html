  • Mailing List Summary

The content below is an LLM summary of this months activity on the mailing list.

Ongoing Discussions
  • Java Extension Type API Refinement: Simplifying extension type writer creation in Java from factory to type-based. This aims to reduce boilerplate and API complexity, with breaking changes planned for v18.4.0.
    • [DISCUSS][JAVA] Extension type type-based implementation Thread
  • Timestamp Precision and Timezone Handling: Discussions on high-precision (picosecond) timestamp/duration types progress towards a two-integer representation. Concurrently, a canonical TimestampWithOffset extension type (UTC + offset) was approved, balancing ANSI SQL compatibility with simpler implementation.
    • Re: [DISCUSS][Format] Introduction of picosecond-precision 128-bit timestamp and duration types Thread
    • [VOTE][FORMAT] TimestampWithOffset canonical extension type Thread
  • Late Materialization for Parquet Data: Efforts focus on enabling Arrow to handle new Parquet encodings (ALP, FSST) in their encoded form longer. This boosts performance and memory efficiency by allowing engines to operate on compressed data before full decoding, potentially via new Arrow vector types or negotiation.
    • [DISCUSS] Support for late materialization in Parquet -> Arrow Thread
    • Re: [DISCUSS] Support for late materialization in Parquet -> Arrow Thread
Emerging Themes
  • Ecosystem Expansion with DACP: A proposal for the Data Access and Collaboration Protocol (DACP), built on Arrow Flight, seeks integration. DACP targets scientific and distributed computing for cross-node data access, offering catalog management, provenance, and collaboration.
    • [DACP] Proposal: Integrate DACP (Data Access and Collaboration Protocol) into Apache Arrow Ecosystem Thread
  • Logical vs. Physical Type Separation: The exploration of late materialization for Parquet data highlights growing interest in decoupling logical data types from physical storage. This allows optimized, encoded physical representations while ensuring interoperability via standard Arrow formats.
    • [DISCUSS] Support for late materialization in Parquet -> Arrow Thread
    • Re: [DISCUSS] Support for late materialization in Parquet -> Arrow Thread
Potential Roadblocks
  • CI Stability and Reliability: Frequent, often unrelated, failures in GitHub Actions, attributed to third-party dependencies or CI platform issues, challenge consistent development and contribution validation.
    • Failing Github Actions Thread
  • Language Version Compatibility: The Go 18.5.0 release faced delays due to compatibility with Go 1.25s new leak checker, highlighting hurdles in maintaining compatibility with evolving toolchains and strict memory safety.
    • [VOTE][Go] Release Apache Arrow Go 18.5.0 RC0 Thread
    • [VOTE][Go] Release Apache Arrow Go 18.5.0 RC1 Thread
Strategic Plans
  • Accelerating ADBC Development: A new GitHub organization, ADBC Driver Foundry, formed to centralize and expedite Arrow Database Connectivity driver development, addressing previous bottlenecks and fostering efficient contributions.
    • ADBC drivers contrib org Thread
  • C++ Standard Upgrade to C++20: A significant PR is in review to update Arrows C++ build requirements to C++20. This move incorporates modern language features for enhanced performance and maintainability, targeting Arrow 23.0.0.
    • [C++] PR for C++20 Thread
