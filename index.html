  • Mailing List Summary

The content below is an LLM summary of this month’s activity on the mailing list.

Ongoing Discussions
  • Data Type Extensions: Standardizing new data types remains a focus, including the recently approved TimestampWithOffset for ANSI SQL compatibility and ongoing exploration of picosecond-precision 128-bit timestamps and durations as canonical extension types.
    • [VOTE][FORMAT] TimestampWithOffset canonical extension type](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BVOTE%5D%5BFORMAT%5D%20TimestampWithOffset%20canonical%20extension%20type)
    • [DISCUSS][Format] Introduction of picosecond-precision 128-bit timestamp and duration types](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BDISCUSS%5D%5BFormat%5D%20Introduction%20of%20picosecond-precision%20128-bit%20timestamp%20and%20duration%20types)
  • Late Materialization for Parquet: Discussions are active on integrating new Parquet encodings (ALP, FSST) into Arrow, aiming to support “late materialization” to defer data decompression, which could lead to significant performance improvements.
    • [DISCUSS] Support for late materialization in Parquet -> Arrow](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BDISCUSS%5D%20Support%20for%20late%20materialization%20in%20Parquet%20%2D%3E%20Arrow)
Emerging Themes
  • Ecosystem Integration & Growth: New contributors, including from Apache Spark, highlight a growing interest in deeper integration and broadening Arrow’s interoperability within the data ecosystem.
    • Request for review on several small PRs
  • Advanced Data Access Protocols: The proposal to integrate DACP (Data Access and Collaboration Protocol) as an Arrow Flight Extension signals a direction towards robust, collaborative data access for scientific and distributed computing, including alignment with existing standards like Substrait.
    • [DACP] Proposal: Integrate DACP (Data Access and Collaboration Protocol) into Apache Arrow Ecosystem](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BDACP%5D%20Proposal%20Integrate%20DACP%20(Data%20Access%20and%20Collaboration%20Protocol)%20into%20Apache%20Arrow%20Ecosystem)
Potential Roadblocks
  • CI Instability & Maintenance: Persistent GitHub Actions failures, often due to external dependencies, continue to pose a challenge for consistent and reliable development.
    • Failing Github Actions
  • Release Verification Issues: Recent Go releases faced hurdles with new leak checkers, uncovering and resolving actual code leaks that required careful attention during the release candidate process.
    • [VOTE][Go] Release Apache Arrow Go 18.5.0 RC0](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BVOTE%5D%5BGo%5D%20Release%20Apache%20Arrow%20Go%2018.5.0%20RC0)
  • Architectural Challenges for Encoded Data: Implementing late materialization for Parquet encodings demands significant architectural shifts within Arrow libraries, with developer bandwidth being a key concern for such extensive changes.
    • [DISCUSS] Support for late materialization in Parquet -> Arrow](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BDISCUSS%5D%20Support%20for%20late%20materialization%20in%20Parquet%20%2D%3E%20Arrow)
Strategic Plans
  • Accelerating ADBC Driver Development: A new adbc-drivers contrib organization, the ADBC Driver Foundry, has been formed to streamline and accelerate the development and maintenance of ADBC drivers.
    • ADBC drivers contrib org
  • C++ Standard Modernization: The C++ implementation has successfully upgraded its build requirements to C++20, laying the groundwork for further modernization and leveraging advanced language features in future development.
    • [C++] PR for C++20](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BC%2B%2B%5D%20PR%20for%20C%2B%2B20)
  • Formalizing Ecosystem Extensions: Discussions around DACP’s potential as an “Arrow Flight Extension” reflect a strategic intent to formally integrate and support external projects that enhance the Arrow ecosystem’s capabilities.
    • [DACP] Proposal: Integrate DACP (Data Access and Collaboration Protocol) into Apache Arrow Ecosystem](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:%5BDACP%5D%20Proposal%20Integrate%20DACP%20(Data%20Access%20and%20Collaboration%20Protocol)%20into%20Apache%20Arrow%20Ecosystem)
