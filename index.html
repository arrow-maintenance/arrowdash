Mailing list items from the last month
  • Mailing List Summary
Ongoing Discussions
  • ADBC Configuration: Defining configuration files and locations for ADBC drivers to improve developer experience (May 27, June 2). The discussion revolves around using configuration manifests vs. environment variables for driver discovery.
  • C++ CMake Build System: Refactoring the C++ CMake build system to simplify static/shared library linkage (June 3). The goal is to avoid repeating conditional logic for linking targets, with potential consideration for supporting simultaneous building of static and shared libraries.
  • Arrow Summit Volunteers: Selection committee formed to aggregate volunteers (May 18, May 19).
Emerging Themes
  • Repository Splitting: Continuing trend of splitting language implementations into separate repositories (May 16). A vote to split Swift was successful (May 20, May 23), mirroring earlier efforts for Go, Java, R, and C#.
  • Monorepo Issue Management: Addressing stale issues in the main repository, particularly those related to specific language implementations like Rust, by migrating or closing them (June 12).
  • Resource Allocation: Utilizing donated AWS credits for CI improvements, GPU testing, and benchmarking infrastructure (June 12). This includes plans for spot EC2 instances and large memory tests.
  • Adoption and Usage Insights: A Kapa.ai bot on dev docs has received a high amount of usage, averaging around 191 questions per week and is providing value for users (June 12).
Potential Roadblocks
  • Swift Implementation Support: Concerns about the level of activity and support for the Swift implementation, even after repository separation (May 16, May 20).
  • C++ Standard Migration: Ensuring a smooth transition to C++20, particularly regarding potential impacts on users relying on older compiler versions or direct C++ usage, however it appears most users consume C++ via other languages (May 19).
  • Feather V1 Deprecation: Balancing the desire to remove legacy code (Feather V1 reader/writer) with potential disruption to users still relying on the format, with a compromise to keep the reader (June 3).
Strategic Plans
  • C++20 Upgrade: The project is moving towards requiring C++20 support for the C++ codebase to leverage modern language features and standard library improvements (May 19).
  • Feature Freeze: Setting a feature freeze date for the 21.0.0 release, indicating a focus on stability and bug fixes for the upcoming version (June 4).
  • GPU Acceleration: Exploring GPU implementations of Arrow using wgpu, potentially through donations of existing projects like arrow-gpu (June 6).
