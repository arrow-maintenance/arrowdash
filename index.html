  • Mailing List Summary

The content below is an LLM summary of this month’s activity on the mailing list.

  • Arrow 23.0.1 release and provenance improvements: The patch release 23.0.1 was approved following a discussion on release verifiability, leading to a new requirement to include SHA256 hashes directly in future release vote emails. Thread

  • 10-year anniversary of Apache Arrow: The community celebrated the project’s tenth anniversary since its founding in February 2016, reflecting on its evolution into a universal standard for columnar data exchange. Thread

  • Release artifact integrity and GitHub stability: Developers discussed SHA512 mismatches for ADBC 21, reinforcing that GitHub-generated source archives are not stable and canonical Apache release artifacts must be used for reliable verification. Thread

  • IPC format extensions for application data: Proposal to extend the IPC format with ApplicationData fields on messages and a new Empty message type to support custom metadata transmission without full record batches. Thread

  • Support for 24:00:00 in TIME types: Discussion on expanding the TIME type range to include 24:00:00 to improve data fidelity and compatibility with databases like PostgreSQL and DuckDB. Thread

  • vgi-rpc framework introduction: Announcement of an open-source, typed RPC framework built on Arrow IPC that supports shared memory transports and provides a Python-first alternative to Flight/gRPC. Thread

  • Arrow Rust 58.0.0 release: The Rust implementation released version 58.0.0, which includes breaking API changes for performance optimizations and improved ergonomics in the core crates. Thread

  • Gandiva Python bindings deprecation: Proposal to deprecate pyarrow.gandiva due to low maintenance activity and lack of inclusion in PyPI wheels, focusing development on more widely used components. Thread

  • Java Flight UDS regression: Report of a critical bug in Java Flight versions 16-18 where using Unix Domain Sockets with Epoll triggers a ClassCastException, breaking compatibility with certain Linux deployments. Thread
