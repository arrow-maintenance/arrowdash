The content below is an LLM summary of this month’s activity on the mailing list.

Ongoing Discussions
  • Parquet-Arrow Late Materialization: Discussions continue on efficient Parquet to Arrow conversion, focusing on “late materialization” to avoid unnecessary data expansion and improve filter pushdown performance. This involves exploring ways to decouple physical and logical data types in Arrow’s in-memory format and within compute kernels. Thread
  • PyArrow Code Review: General requests for community review on PyArrow changes indicate ongoing development and maintenance efforts across language bindings. Thread
Emerging Themes
  • DACP Ecosystem Integration: The Data Access and Collaboration Protocol (DACP) is seeking deeper integration with Arrow Flight, evaluating alignment with Substrait for expression handling and formalization as an “Arrow Flight Extension.” This highlights Arrow’s role as a foundation for broader data protocols. Thread
  • Independent Parquet Implementations: The announcement of “Carquet,” a standalone C library for Parquet, signals diverse community interest and specialized needs for the Parquet format, potentially independent of full Arrow dependencies, yet complementing the ecosystem. Thread
Potential Roadblocks
  • Java Dataset API Maturity: The Java Dataset API remains experimental, prompting questions from users regarding its stability, production readiness, and roadmap for parity with C++ functionalities like scanner pushdowns and partitioning. Thread
  • Data Model Rigidity: The current Arrow data model’s coupling of physical and logical types is identified as a challenge to efficiently support advanced Parquet encodings and late materialization, potentially requiring significant architectural rethinking and developer bandwidth. Thread
  • Infrastructure Issues: An issue was raised regarding missing Date/Message-ID headers in emails to the builds mailing list, highlighting a potential silent communication problem that could impact project health. Thread
Strategic Plans
  • C++20 Modernization: Arrow C++ has officially upgraded its build requirements to C++20, with follow-up issues opened to leverage new C++20 features for further modernization. This is a foundational step for future C++ development. Thread
  • Upcoming Release Cycle: The feature freeze for Arrow 23.0.0 was announced for January 12th, indicating active progress towards the next major release. ADBC 22, Arrow Rust Object Store 0.13.0/0.13.1/0.12.5, and Arrow Rust 57.2.0 were released. Thread Thread Thread Thread
