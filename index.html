  • Mailing List Summary

The content below is an LLM summary of this month’s activity on the mailing list.

  • Arrow 23.0.0 release: The project completed the release of version 23.0.0, which resolved over 330 issues across the various language implementations. Thread

  • AI-generated contributions policy: Discussion on establishing guidelines for PRs created with AI tools to ensure contributors can debug their changes and reduce maintainer burden. Thread

  • Security model for Arrow format: Proposal to create formal documentation regarding the security considerations of the Arrow in-memory format, IPC, and specific implementations. Thread

  • IPC OpaqueBytes proposal: Discussion on adding a new IPC message type, tentatively named OpaqueBytes or ApplicationData, to allow arbitrary metadata or signaling bytes within IPC streams. Thread

  • StructArray validity specification: Debate regarding whether child arrays of a nullable StructArray must strictly omit nulls when the field is marked non-nullable, or if redundant nulls should be allowed for compatibility. Thread

  • ADBC release provenance: Investigation into a checksum mismatch for ADBC artifacts led to a discussion on the importance of using official Apache infrastructure for releases to ensure integrity. Thread

  • Arrow Rust 58.0.0 planning: Community members have started planning for the next major Rust release, which is expected to include several breaking API changes. Thread

  • Nanoarrow 0.8.0 release: The lightweight C-centric implementation nanoarrow reached version 0.8.0, including improved verification scripts and updated Python/R bindings. Thread

  • C Stream generic array support: Technical discussion on evolving the C Stream interface to support yielding generic ArrowArray types rather than assuming a StructArray or RecordBatch. Thread
