The content below is an LLM summary of this month’s activity on the mailing list.

  • Arrow 23.0.0 release: The community announced the major release of version 23.0.0, which includes over 330 resolved issues across the project’s various components and language bindings. Thread

  • AI-generated contributions guidelines: Discussion on establishing documentation and policies for pull requests created using AI tools to ensure contributor ownership and reduce maintainer burden. Thread

  • Security model for the Arrow format: Proposal to draft formal security considerations for the Arrow in-memory and IPC formats, with implementation-specific details for C++ to follow. Thread

  • Late materialization in Parquet: Technical discussion on optimizing string comparison performance by exposing Parquet dictionaries directly to readers to avoid expensive string materialization. Thread

  • Carquet standalone C library: Announcement of a pure C99 library for reading and writing Parquet files, designed for embedded systems or environments where C++ or the full Arrow dependency is not feasible. Thread

  • Java Dataset API roadmap: Users sought clarification on the “experimental” status and production readiness of the Java Dataset API, leading to discussions on future compatibility goals. Thread

  • ADBC 22 release: The Arrow Database Connectivity (ADBC) libraries reached version 22, featuring updates to the Go, Python, Java, and C# implementations. Thread

  • JDBC driver thread-safety: Clarification on the thread-safety guarantees of the Arrow JDBC driver, confirming that Connection objects and their children are generally not intended for concurrent multi-threaded use. Thread
