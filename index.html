  • Mailing List Summary

The content below is an LLM summary of this months activity on the mailing list.

Ongoing Discussions
  • Timestamp Format Evolution: Active discussions center on expanding Arrows time types, including formalizing TimestampWithOffset for database compatibility and introducing picosecond precision timestamps. The latter design evolved to a two-integer canonical extension type for simpler operations.
    • [DISCUSS] TimestampWithOffset canonical extension type Thread
    • [DISCUSS][Format] Introduction of picosecond-precision 128-bit timestamp and duration types Thread
    • [VOTE][FORMAT] TimestampWithOffset canonical extension type Thread
  • Java Extension Type Refactoring: A proposed breaking change aims to simplify custom extension type writer creation in arrow-java, shifting from a factory-based to a more intuitive type-based pattern for improved developer experience.
    • [DISCUSS][JAVA] Extension type type-based implementation Thread
  • Rust Bitwise Performance: Plans are active to significantly enhance bitwise operation performance within arrow-rs, critical for optimizing core data structures like null bitmaps during data processing.
    • [RUST] Improving performance of bitwise operations Thread
Emerging Themes
  • ADBC Driver Ecosystem: The new ADBC Driver Foundry GitHub organization indicates a strategic shift to accelerate ADBC driver development within a dedicated, independent contrib space. Apache Arrow ADBC 21 was recently released.
    • ADBC drivers contrib org Thread
    • [ANNOUNCE] Apache Arrow ADBC 21 Released Thread
  • New Protocol Integration (DACP): A proposal to integrate DACP (Data Access and Collaboration Protocol), built on Arrow Flight, signals interest in expanding Arrows capabilities for cross-node, scientific computing data access and collaboration.
    • [DACP] Proposal: Integrate DACP (Data Access and Collaboration Protocol) into Apache Arrow Ecosystem Thread
  • Advanced Data Type Standardization: Recent approval of TimestampWithOffset and ongoing work on high-precision timestamps demonstrate commitment to standardizing representations for complex, database-compatible data types.
    • [VOTE][FORMAT] TimestampWithOffset canonical extension type Thread
    • [DISCUSS][Format] Introduction of picosecond-precision 128-bit timestamp and duration types Thread
Potential Roadblocks
  • CI Stability Issues: Persistent, unrelated failures in GitHub Actions, particularly Ruby-related tests, challenge developers and hinder consistent, efficient development workflows across the project.
    • Failing Github Actions Thread
  • Breaking Changes Impact: The proposed refactoring of Java extension types introduces backward incompatibilities, necessitating careful release planning and robust migration support for users.
    • [DISCUSS][JAVA] Extension type type-based implementation Thread
  • Balancing Format Versatility & Simplicity: Expanding Arrows format for advanced time types requires careful consideration to maintain simplicity, ensure efficient interoperability, and avoid overly complex representations.
    • [DISCUSS][Format] Introduction of picosecond-precision 128-bit timestamp and duration types Thread
    • [DISCUSS] TimestampWithOffset canonical extension type Thread
Strategic Plans
  • Expanding ADBC Ecosystem: The ADBC Driver Foundry signifies a strategic initiative to foster a robust, independently evolving ADBC driver ecosystem, accelerating development and broader adoption.
    • ADBC drivers contrib org Thread
  • Strengthening Language Bindings & Format: Continuous improvements in language performance (Rust bitwise operations) and developer experience (Java extension types), alongside the evolution of the Arrow format (e.g., TimestampWithOffset), demonstrate a commitment to core growth. Releases like Rust 57.1.0 and upcoming Go 18.5.0 and Java 19.0.0 reflect this.
    • [RUST] Improving performance of bitwise operations Thread
    • [DISCUSS][JAVA] Extension type type-based implementation Thread
    • [VOTE][FORMAT] TimestampWithOffset canonical extension type Thread
    • [DISCUSS][Format] Introduction of picosecond-precision 128-bit timestamp and duration types Thread
