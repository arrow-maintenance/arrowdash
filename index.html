Ongoing Discussions
  • Parquet Late Materialization: Discussions on how Arrow can efficiently support new Parquet encodings (ALP, FSST) by operating on encoded data for performance. Challenges include Arrow’s type system rigidity and the complexity of adding new vector types.
    • Support for late materialization in Parquet -> Arrow: Thread
    • Support for late materialization in Parquet -> Arrow: Thread
  • DACP & Substrait Alignment: Efforts to integrate DACP with Arrow Flight and align its expression language with Substrait, enhancing ecosystem interoperability and standardizing query representations.
    • DACP Integration - Substrait Alignment & Ecosystem Collaboration: Thread
  • Java Dataset API Status: Inquiry about the stability, roadmap, and production readiness of the experimental Java Dataset API, highlighting a need for clearer guidance.
    • Current status and roadmap for Java Dataset API: Thread
Emerging Themes
  • Encoded Data Performance: Strong push to leverage encoded formats directly for performance gains, minimizing data expansion.
  • Ecosystem Interoperability: Increased focus on formalizing integrations for projects building on Arrow or using complementary standards like Substrait.
  • Language Modernization: Arrow C++ migrated to C++20, setting a precedent for leveraging modern language features.
    • PR for C++20: Thread
Potential Roadblocks
  • Data Model Rigidity: Arrow’s current data model may struggle to adapt to advanced encodings without extensive changes, potentially limiting performance.
    • Support for late materialization in Parquet -> Arrow: Thread
  • Developer Bandwidth: Significant architectural changes to support new encoding paradigms demand substantial effort.
  • API Maturity Gaps: Inconsistent stability and lack of clear roadmaps for experimental APIs hinder adoption.
    • Current status and roadmap for Java Dataset API: Thread
  • External Parquet Implementations: Projects like Carquet (pure C, no Arrow dependency) highlight demand for lightweight solutions, challenging Arrow’s comprehensive approach.
    • Carquet - standalone C library for Parquet files: Thread
Strategic Plans
  • Evolving Data Representation: Future focus on enhancing Arrow’s internal data representation to directly support and leverage encoded data.
  • Deepening Ecosystem Integration: Continued efforts to integrate external projects and align with data standards, extending Arrow’s reach.
  • Continuous Modernization & Stabilization: Ongoing commitment to modernizing language implementations and stabilizing experimental APIs.
  • Regular Release Cadence: Multiple successful release votes confirm a robust schedule for updates and new features, with an upcoming feature freeze for Arrow 23.0.0.
    • Feature Freeze for Arrow 23.0.0 - 12th January: Thread
