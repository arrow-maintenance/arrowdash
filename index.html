  • Mailing List Summary

The content below is an LLM summary of this month’s activity on the mailing list.

  • Apache Arrow 10th anniversary: The project celebrated 10 years since its first commit in February 2016, reflecting on its growth as a universal standard for columnar data. Thread

  • Release candidate provenance and security: Discussion on tightening release procedures, including mandatory SHA hashes in vote emails and ensuring all artifacts are hosted on ASF-controlled infrastructure to prevent artifact substitution. Thread

  • vgi-rpc — typed Arrow RPC for Python: Introduction of a new RPC framework built directly on Arrow IPC that offers typed proxies and shared-memory transport without gRPC or Protobuf dependencies. Thread

  • IPC format extensions: Proposal to extend the IPC format with ApplicationData on messages and a new Empty message type to support custom metadata and improved stream signaling. Thread

  • TIME support for 24:00:00: Debate on whether Arrow’s Time types should support “24:00:00” to ensure full data fidelity when round-tripping data from databases like PostgreSQL and DuckDB. Thread

  • Equivalence of IPC file and stream formats: Technical discussion on whether IPC files should be strictly readable as streams by skipping the header and how to handle potential metadata discrepancies in the footer. Thread

  • Deprecation of legacy components: Proposals to deprecate the Feather V1 reader/writer and the pyarrow.gandiva module due to low usage and high maintenance burdens. Thread

  • AI-generated contribution policy: A new proposal suggests that contributors must disclose if mailing list posts or questions are generated using AI to maintain transparency within the community. Thread

  • Arrow 23.0.1 maintenance release: The community approved and released version 23.0.1 to address 28 resolved issues and improve release verification logging. Thread
