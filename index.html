  • Mailing List Summary

The content below is an LLM summary of this month’s activity on the mailing list.

  • Apache Arrow 23.0.1 release: The community approved a patch release resolving 28 issues, primarily focusing on stability and bug fixes across the C++, Python, and R implementations. Thread

  • Release candidate provenance and security: Discussion regarding the persistence of release candidate SHAs in vote threads and the requirement to maintain artifacts on ASF-controlled hardware to ensure supply chain integrity. Thread

  • Arrow’s 10th anniversary: The project celebrated its 10th anniversary since the first commit in February 2016, highlighting its growth into a universal standard for columnar data. Thread

  • IPC format extensions (ApplicationData): A proposal to extend the IPC format by adding an ApplicationData field to messages and introducing an “Empty” message type to facilitate better metadata exchange. Thread

  • Flight SQL execution path improvements: Proposal to add an explicit is_update boolean to ActionCreatePreparedStatementResult to help clients correctly choose between query and update execution paths. Thread

  • Equivalence of IPC file and stream formats: Technical discussion on whether the IPC file format should be strictly required to be a valid IPC stream when the header and footer are ignored, aiding compatibility with simple readers. Thread

  • Time support for 24:00:00: Debate on whether the Arrow TIME type should be updated to support a “24:00:00” value to ensure full data fidelity when round-tripping from databases like PostgreSQL and DuckDB. Thread

  • Deprecation of legacy components: Discussions are underway to deprecate the Feather V1 reader/writer and pyarrow.gandiva due to low maintenance and the availability of modern alternatives. Thread

  • Map type field names: Investigation into whether implementations should strictly enforce field names for Map types as “entries”, “key”, and “value” or support custom names found in some data sources. Thread
