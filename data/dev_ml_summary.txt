* **BigDecimal canonical extension type:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scales, improving compatibility with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Apache Arrow 24.0.0 release:** The community announced the 24.0.0 release, featuring 259 resolved issues across the ecosystem since the last major update. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Rust security policy:** Discussion on establishing a formal security policy for the Arrow and Parquet Rust implementations to standardize the handling of vulnerability reports. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **IBM Db2 integration:** IBM engineers proposed adding Arrow support for Db2, evaluating the development of an ADBC driver and the use of Arrow Flight for high-performance data transport. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which currently utilizes the Rust implementation via bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **UTF8 metadata restrictions:** Debate over whether field and schema metadata should be strictly validated as UTF8 to ensure compliance with the underlying Flatbuffers specification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **SIMD optimizations for Parquet:** Proposal to implement AVX2 and xsimd-based optimizations for Parquet Bloom filter probes, targeting significant performance gains on the read path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** A new proposal for an ADBC API that supports coordinated, parallel bulk ingestion for distributed processing engines like Spark and Flink. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Flight SQL dialect metadata:** Proposal to add new SqlInfo codes to Flight SQL to help clients discover backend-specific dialect features like LIMIT/OFFSET syntax and boolean literal support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)
