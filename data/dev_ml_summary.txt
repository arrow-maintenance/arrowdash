* **Numba support for PyArrow:** Proposal to integrate a Numba support layer directly into PyArrow, allowing JIT-compiled computations on Arrow data while managing the complexities of the non-stable Arrow C++ ABI. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Arrow Java 19.0.0 and JDK 17 requirement:** Following the release of Java 19.0.0, the community reached a consensus to bump the minimum requirement to JDK 17 starting with the version 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEJava%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1) | [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL Prepared Statement execution paths:** Discussion and implementation of protocol changes to help Flight SQL clients explicitly determine whether to use query or update network flows for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **ADBC driver migration to contrib organization:** Drivers for BigQuery, Databricks, and Snowflake are being moved from the main ADBC repository to a dedicated `adbc-drivers` organization to streamline maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Stability of C Device Data Interface:** Developers discussed the experimental status of the C Device Data Interface, concluding that it requires broader adoption and testing across non-CUDA devices before it can be marked stable. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Arrow Rust 58.1.0 release:** The Rust implementation reached version 58.1.0, including various improvements and a corresponding update to the Object Store implementation (v0.13.2). [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1)

* **Map type field naming consistency:** Discussion on standardizing field names within the Map type to "key", "value", and "entries" across C++, Rust, and nanoarrow to improve cross-implementation compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type%27s%20key%2Fitem%2Fvalue%20field%20names)

* **ADBC 23 release:** Voting opened for version 23 of the ADBC libraries, which includes over 40 resolved issues across C/C++, Java, Python, Go, and Rust. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **LLMs for issue maintenance:** Proposal to use LLMs to automate repo management tasks, such as automatically applying component labels to unlabelled issues based on their content. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)
