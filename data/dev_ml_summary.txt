* **Apache Arrow 24.0.0 release:** The community announced the major 24.0.0 release, featuring over 250 resolved issues across multiple language bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Apache Arrow ADBC 23 release:** A new release of the Arrow Database Connectivity (ADBC) subproject is now available, including 41 bug fixes and improvements across C++, Go, Java, and Python. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **Numba support for PyArrow:** Discussion continues on integrating Numba support directly into PyArrow to enable high-performance JIT compilation of Arrow data while navigating complex ABI and maintenance challenges. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Status of C Device Data Interface:** Developers discussed whether to mark the C Device Data Interface as non-experimental, concluding that more independent implementations beyond CUDA are needed to validate the design. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs codebase, which will bring high-quality type annotations directly into the Apache Arrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal Extension Type:** A new proposal introduces a canonical extension type for arbitrary-precision decimals to support variable scale per row, common in engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Flight SQL Prepared Statement updates:** Discussion and voting on adding an `is_update` field to `ActionCreatePreparedStatementResult` to help clients determine the correct execution path for Flight SQL statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Dictionary-encoded extension types in IPC:** Research into IPC serialization revealed limitations in representing dictionary-encoded extension types, leading to a proposal for adding KeyValue metadata to the dictionary encoding. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Rust Security Policy:** A new security policy is being drafted for the Rust implementation to formalize how vulnerabilities are reported and handled in `arrow-rs` and `parquet-rs`. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **ADBC Logo Proposal:** The community is reviewing a proposed visual identity and logo for the ADBC subproject to align it with the broader Arrow ecosystem branding. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
