* **Arrow 24.0.0 release:** Voting is underway for the 24.0.0 release candidate, with discussions focusing on resolving Gandiva test failures in specific timezones and build timeouts for Almalinux. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Proposal to integrate a Numba support layer directly into PyArrow to enable JIT-compiled calculations while avoiding stable ABI issues with Arrow C++. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Apache Arrow ADBC 23 release:** The community announced the release of ADBC version 23, including 41 resolved issues across the C++, Go, Java, and Python implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Donation of pyarrow-stubs:** The independent `pyarrow-stubs` codebase is being donated to the Apache Arrow project to ensure type annotations are officially maintained and stay in sync with the library. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **JDK 17 minimum for Arrow Java:** Lazy consensus was reached to upgrade the minimum requirement for the Arrow Java implementation to JDK 17 starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Experimentality of C Device Data Interface:** Discussion on why the C Device Data Interface remains experimental after three years and whether to prioritize an asynchronous version of the standard C Data Interface. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **IPC representation of dictionary-encoded extension types:** Investigation into an IPC serialization limitation that prevents distinguishing between dictionaries with extension values and extensions with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL protocol updates:** Approval of a new `is_update` field for prepared statements to help clients determine the correct execution path (Query vs. Update) in Flight SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)
