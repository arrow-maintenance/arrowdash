* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type to represent arbitrary-precision decimals with variable scale per row, supporting high-fidelity data transport from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes over 250 resolved issues across the various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Erlang implementation donation:** A formal vote passed to accept the donation of a new Erlang implementation for Apache Arrow, initially based on Rust bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Rust security policy and maintenance:** Discussion regarding a new security policy for the Rust implementation to better handle vulnerability reporting and a series of patch releases (58.3.0, 57.3.1, 56.2.1) to address potential offset overflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **IBM Db2 support via ADBC:** Engineers from IBM proposed adding support for Db2 through the ADBC (Arrow Database Connectivity) framework and are exploring further integration via Arrow Flight SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBCDB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Metadata UTF-8 enforcement:** Discussion on whether to strictly enforce UTF-8 encoding for field and schema metadata to ensure compliance with Flatbuffers requirements and prevent interoperability issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **SIMD optimizations for Parquet Bloom Filters:** Proposal to port AVX2 and xsimd-based Bloom Filter probe optimizations from the Go implementation to the C++ library to improve reader performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** A new proposal to add a partitioned bulk ingest API to ADBC, allowing distributed writers like Spark or Flink to perform atomic, parallel data loads. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Flight SQL dialect metadata:** Proposal to add new SqlInfo codes to Flight SQL to allow servers to communicate specific SQL dialect features, such as limit/offset grammar and null ordering, to clients. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)
