* **Arrow 24.0.0 release:** The community released version 24.0.0, resolving 259 issues across the project including significant work on the C++, Python, and R implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** The independently maintained library of PEP 484 type stub annotations was formally donated to the Apache Arrow project to ensure they stay in sync with PyArrow and follow project governance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** A proposal was introduced for a new extension type supporting arbitrary-precision decimals with variable scale per row to improve compatibility with database engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **ADBC 23 release:** The Apache Arrow ADBC subproject released version 23, featuring 41 bug fixes and improvements across the database connectivity abstraction and its various drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **LargeMap format addition:** Discussion regarding adding a `LargeMap` type to the specification that would use 64-bit offsets, preventing casting issues for libraries like Polars that rely on large types internally. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Flight SQL prepared statement updates:** Proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` to allow servers to explicitly signal whether a statement follows a query or update network flow. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IBM Db2 integration:** Engineers from IBM proposed adding deep Arrow support for Db2, exploring options for native Arrow query results, ADBC drivers, and high-performance transport via Arrow Flight. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **IPC representation of extension types:** Developers are discussing updates to the IPC schema serialization to better distinguish between dictionary-encoded extension types and extension types that use a dictionary as their storage type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
