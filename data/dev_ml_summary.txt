* **Apache Arrow 24.0.0 Release:** The community released version 24.0.0, resolving over 250 issues across multiple language bindings and including significant CI and verification improvements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donation of pyarrow-stubs:** Developers voted to accept the donation of the pyarrow-stubs project to provide official PEP 484 type annotations for PyArrow, ensuring they stay in sync with the core library. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal Extension Type:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scales, facilitating high-fidelity data transfer from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Numba support for PyArrow:** Discussion on providing native Numba support for Arrow arrays to enable efficient JIT-compiled operations, focusing on the use of the C Data Interface to maintain ABI stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Apache Arrow ADBC 23 Release:** The ADBC subproject released version 23, which includes 41 bug fixes and updates to C++, Go, Python, Java, and Rust subcomponents. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, allowing maps to exceed the 2GB limit imposed by 32-bit offsets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Flight SQL Prepared Statement execution paths:** Proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` to help clients correctly choose between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IPC representation of dictionary-encoded extensions:** Investigation into ambiguity within the IPC schema regarding dictionary-encoded extension types versus extension types with dictionary storage, with a proposed metadata-based solution. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **C Device Data Interface stability:** A discussion on whether the C Device Data Interface should remain marked as experimental, with contributors suggesting more non-CUDA implementations are needed before stabilization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **ADBC Logo Proposal:** The community is evaluating a new logo for the ADBC subproject to provide a distinct visual identity while maintaining brand consistency with the main Apache Arrow project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
