* **pyarrow-stubs donation:** The community voted to accept the donation of `pyarrow-stubs` into the Apache Arrow project to provide better type hinting support for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Apache Arrow 24.0.0 Release:** Version 24.0.0 was released, resolving 259 issues across multiple language implementations and including updated binary artifacts for major distributions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Canonical BigDecimal Extension Type:** Proposal for a new canonical extension type for arbitrary-precision decimals to support variable scale per row, improving interoperability with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IBM Db2 Integration:** IBM developers proposed adding Arrow support for Db2, focusing on a new ADBC driver and exploring Flight SQL for high-performance data transport. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSADBCDB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Metadata UTF-8 Restriction:** Discussion on whether to formally restrict field and schema key-value metadata to UTF-8 to ensure alignment with Flatbuffers requirements and cross-language safety. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Rust Security Policy:** The community is discussing a formal security policy for the Rust implementation to better handle vulnerability reporting and distinguish between standard bugs and security risks. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Flight SQL is_update field:** Proposal to add an `is_update` boolean to prepared statement results, allowing drivers to distinguish between DDL/DML and data-returning queries without using heuristics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Apache Arrow .NET 23.0.0 Release:** The .NET implementation released version 23.0.0, which includes significant updates to Flight SQL and enhanced verification support for Windows users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20.NET%2023.0.0%20released)

* **ADBC Logo Proposal:** A new logo design for the ADBC (Arrow Database Connectivity) subproject was introduced to give the project a distinct identity within the Arrow ecosystem. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **Glasgow 2026 Hackathon:** Planning has begun for an Arrow hackathon at the Community Over Code event in Glasgow to focus on bug fixes, documentation, and new contributor onboarding. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Hackathon%20at%20Community%20Over%20Code%20Glasgow%202026%20%E2%80%94%20interest%20from%20Arrow%3F)
