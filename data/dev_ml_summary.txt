* **Arrow 24.0.0 release:** The major release consists of 259 resolved GitHub issues across the project, including significant performance improvements and bug fixes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs repository to the Apache Arrow project to provide better type-hinting support for Python developers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Proposal for a Canonical BigDecimal extension type:** Discussion on adding a new extension type for arbitrary-precision decimals with variable scales to support engines like PostgreSQL, Trino, and Oracle. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Security policy for Rust implementation:** Proposal to establish a formal security policy for the arrow-rs and parquet-rs crates to handle vulnerability reporting and distinguish between bugs and security flaws. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **IBM Db2 integration:** IBM engineers proposed adding deep Arrow support for Db2, including the development of an ADBC driver and exploring Flight SQL for high-performance data transport. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Glasgow 2026 Hackathon planning:** Initial coordination for a dedicated Arrow hackathon at the Community Over Code event in October to focus on documentation and "needs champion" issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Hackathon%20at%20Community%20Over%20Code%20Glasgow%202026%20%E2%80%94%20interest%20from%20Arrow%3F)

* **Metadata UTF-8 restriction:** Discussion on whether to enforce UTF-8 encoding for field and schema metadata to ensure compatibility with Flatbuffers and improve cross-language interoperability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%20schema%20custom%20metadata%20restriction%20to%20UTF8)

* **Arrow .NET 23.0.0 release:** The .NET implementation successfully completed its first release cycle following the repository split, including updated packages for Flight and Flight SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20.NET%20Release%20Apache%20Arrow%20.NET%2023.0.0%20RC0)

* **ADBC logo proposal:** A new logo design was proposed for the ADBC subproject to establish a distinct brand identity while remaining recognizable within the Arrow family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **Erlang implementation IP clearance:** Progress on the donation of the Erlang Arrow implementation, with IP clearance forms being finalized to move the code into the official incubator. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Erlang%20Arrow%20Erlang's%20Grant%20Documents)
