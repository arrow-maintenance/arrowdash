* **Arrow 24.0.0 release:** The community announced the release of version 24.0.0, featuring over 250 resolved issues across the C++, Python, R, and Ruby implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Donation of pyarrow-stubs:** A proposal and successful vote to move the independently maintained `pyarrow-stubs` codebase into the official Arrow repository to provide better type safety for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL execution path updates:** Discussion and vote on adding an `is_update` field to `ActionCreatePreparedStatementResult` to help clients correctly route prepared statements as queries or updates. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **BigDecimal canonical extension type:** Proposal for a new extension type to support arbitrary-precision decimals with variable scales, facilitating high-fidelity data transfer from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC serialization for extension types:** Technical investigation into the IPC format's inability to distinguish between a dictionary of extension values and an extension type using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **ADBC 23 release:** Official release of the Arrow Database Connectivity (ADBC) subproject version 23, including updates to the Go, Java, Python, and C# drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **LargeMap format addition:** Proposal to add a `LargeMap` type to the specification to support 64-bit offsets for map collections, addressing limitations in current 32-bit implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Rust security policy:** Initial discussion on establishing a formal security reporting and handling policy for the `arrow-rs` and `parquet-rs` crates. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)
