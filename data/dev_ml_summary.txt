* **Donation of Erlang implementation:** The community voted to accept the donation of an Erlang implementation (built on Rust bindings), which is now being transferred to the official Apache Arrow Erlang repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** A major proposal to add a write-side mirror to the partitioned read API, enabling distributed writers like Spark or Flink to perform atomic, parallel bulk ingestion. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **IBM Db2 ADBC driver:** Engineers from IBM are contributing a new ADBC driver for Db2, initially developed under the community-maintained driver foundry to allow for independent maintenance and releases. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Parquet Bloom filter optimization:** Discussion on implementing AVX2 and autovectorized probes for Parquet's Split-Block Bloom Filters, yielding significant performance gains (up to 5x) for in-cache workloads. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet/bloom_filter.cc)

* **Proposed arrow.range extension type:** Proposal to add a canonical extension type for representing bounded mathematical intervals, with a focus on standardizing how "closedness" (e.g., inclusive vs. exclusive bounds) is handled. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Metadata UTF-8 restrictions:** Technical discussion regarding whether IPC metadata for fields and schemas should be strictly restricted to UTF-8 to align with Flatbuffers encoding standards and improve cross-language safety. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field/schema/custom%20metadata%20restriction%20to%20UTF8%3F)

* **Enhanced Flight SQL dialect metadata:** Proposal to add new `SqlInfo` codes to communicate backend-specific SQL grammar, such as supported `LIMIT/OFFSET` syntax and boolean literal formats, to improve query pushdown for clients. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **Automated GitHub Copilot reviews:** The project is trialing automated AI-generated reviews for pull requests to help reduce human reviewer burden and identify trivial issues before formal committer review. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)
