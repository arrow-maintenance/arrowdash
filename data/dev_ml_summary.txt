* **PyArrow stubs donation:** The community voted to accept the donation of the pyarrow-stubs codebase, which will be integrated into the main Arrow repository to provide comprehensive type hints for Python developers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Arrow 24.0.0 release:** A new major version of the project was released, resolving 259 issues across the multi-language toolbox and canonical in-memory representations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scales per row, facilitating high-fidelity transport between systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Rust implementation security policy:** Discussion regarding a formal security policy for the arrow-rs and parquet-rs crates to establish clear guidelines for reporting vulnerabilities and distinguishing them from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **IBM Db2 support via ADBC:** Engineers from IBM proposed adding Arrow support for Db2, leading to the development of a new driver within the community-maintained ADBC driver foundry. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)
