* **Conbench v2 proposal:** Wes McKinney proposed a rewrite of the Conbench benchmarking platform using a Go backend and TypeScript frontend to improve performance and maintainability. The discussion focused on data migration paths and utilizing DuckDB and Parquet for more scalable result storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **ADBC Extension for DuckDB:** Columnar.tech proposed donating a new hand-coded ADBC extension for DuckDB to the Arrow project to provide a neutral, ASF-governed home for the implementation. The thread included discussion on how this overlaps with existing community extensions and the impact of AI-assisted development on project governance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **JSON representation of Arrow schemas:** Discussion on establishing a canonical, human-readable JSON format for Arrow schemas to simplify application-level tasks like REST API contracts and ADBC metadata. Participants debated the balance between a verbose, lossless mapping of Flatbuffers and a more compact, typable representation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating IPC Tensor and SparseTensor:** A proposal was made to deprecate the experimental dedicated Tensor and SparseTensor messages in the IPC format in favor of modern canonical extension types. This move aims to reduce maintenance burden and security risks associated with validating these little-used message types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Arrow 25.0.0 release:** The community approved the release of Apache Arrow 25.0.0, which includes over 200 resolved issues across multiple language implementations. A late discussion regarding a performance fix for PyArrow's `to_pylist` was deferred to a potential 25.0.1 patch release to avoid delaying the main release candidate. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **CI resource consumption reduction:** Following reports from ASF Infrastructure, the project successfully reduced its GitHub Actions runner usage by approximately 27% through targeted workflow optimizations. Maintainers emphasized continued vigilance to ensure fair resource sharing with other Apache projects. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Parquet Variant field specification:** A proposal to align the Parquet-format description with Arrow by allowing the `value` field to be omitted in Variant types, particularly when data is shredded into typed columns. This change reflects existing testing patterns and improves compatibility between the two projects. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **C Data Interface field name clarification:** Developers discussed how the C Data Interface (FFI) should handle field names containing embedded nul bytes, which are valid in Arrow's UTF-8 strings but problematic for C strings. The consensus is that producers should detect these cases and error out rather than allowing potentially unsafe behavior during import. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **ADBC documentation overhaul:** The ADBC documentation site was significantly restructured to better explain the ecosystem of drivers, including those maintained by third-party vendors. The updates aim to make the documentation more navigable for new users and clarify the role of the ASF-hosted components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Documentation%20overhauled%20for%20ADBC)

* **Go and Rust implementation releases:** New versions of the Go (18.7.0) and Rust (59.1.0 and 58.4.0) implementations were released, alongside a maintenance update for the Rust Object Store library (0.14.1). [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Go%20Release%20Apache%20Arrow%20Go%2018.7.0%20RC0)
