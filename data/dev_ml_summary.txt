* **JDK 17 minimum for Arrow Java 20.0.0:** The community reached lazy consensus to upgrade the minimum requirement for Arrow Java to JDK 17 for the upcoming 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Proposal to integrate Numba support directly into PyArrow to enable JIT-compiled scientific calculations on Arrow data while avoiding C++ ABI stability issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **Donating pyarrow-stubs to Apache Arrow:** Discussion on moving the independently maintained `pyarrow-stubs` project into the official Arrow repository to provide better type safety and synchronization for Python developers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL protocol enhancement:** Proposal and vote to add an `is_update` field to `ActionCreatePreparedStatementResult`, allowing clients to distinguish between query and update execution paths for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Experimentality of C Device Data Interface:** Developers discussed why the C Device Data Interface remains marked as experimental after three years and considered the utility of adding an async version for non-device (CPU) workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Arrow ADBC 23 release:** Successful release of ADBC version 23, including 41 resolved issues and updates across multiple language bindings including C#, Java, R, and Rust. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **IPC representation of dictionary-encoded extension types:** Investigation into an IPC serialization flaw where dictionary-encoded extension types cannot be correctly round-tripped, leading to errors during deserialization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **ADBC driver migration:** Progress update on moving BigQuery, Databricks, and Snowflake drivers from the core ADBC repository to a dedicated community-driven GitHub organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)
