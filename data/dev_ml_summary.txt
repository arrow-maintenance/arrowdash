* **LargeMap format addition:** Proposal to add a `LargeMap` type to the Arrow specification using 64-bit offsets for parity with other `Large*` types and better compatibility with Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes 259 resolved issues across various implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Donation of pyarrow-stubs:** A vote passed to donate the `pyarrow-stubs` codebase to the project to provide official PEP 484 type annotations for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type to represent arbitrary-precision decimals with variable scale per row, supporting high-fidelity transport from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC representation of dictionary-encoded extension types:** Discussion on addressing a limitation in the IPC schema serialization that currently cannot distinguish between dictionary-encoded extension types and extensions with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL is_update field:** Addition of an `is_update` boolean to the `ActionCreatePreparedStatementResult` to help clients determine whether to use query or update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Arrow ADBC 23 release:** Announcement of the 23rd release of the ADBC subproject, featuring 41 improvements and bug fixes across various database drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **IBM Db2 integration:** IBM engineers proposed adding official Arrow support for Db2, focusing on ADBC driver development and high-performance transport via Arrow Flight. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Rust implementation security policy:** Discussion on establishing a formal security policy for the `arrow-rs` and `parquet` crates to provide clearer guidelines for reporting and handling vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)
