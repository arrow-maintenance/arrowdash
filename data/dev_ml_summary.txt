* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, providing parity with existing `LargeList` and `LargeString` types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes 259 resolved issues across various language implementations and the core format. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Donation of pyarrow-stubs:** The project voted to accept the donation of the `pyarrow-stubs` codebase, which will bring PEP 484 type annotations for PyArrow directly into the main repository for better maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** A proposal for a new canonical extension type for arbitrary-precision decimals to support variable scale data from systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IBM Db2 support for Arrow:** IBM engineers are proposing deeper integration between Db2 and Arrow, specifically through the development of a new ADBC driver and exploring Flight SQL support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **IPC representation for dictionary-encoded extension types:** Discussion on addressing a limitation in the IPC format that prevents distinguishing between a dictionary of extension types and an extension type with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL prepared statement improvements:** Proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` to help clients determine the correct execution path for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **ADBC 23 release:** The Arrow Database Connectivity (ADBC) subproject released version 23, containing over 40 bug fixes and performance improvements for analytical database access. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **ADBC logo proposal:** Discussion on a new visual identity for the ADBC subproject to differentiate it while remaining within the broader Apache Arrow brand family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **Go and .NET releases:** The Go implementation reached version 18.6.0 and the .NET implementation released version 23.0.0, both focusing on language-specific bug fixes and feature parity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Go%20Release%20Apache%20Arrow%20Go%2018.6.0%20RC0)
