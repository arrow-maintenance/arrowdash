* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes 259 resolved issues covering multiple language implementations and format improvements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Apache%20Arrow%2024.0.0%20released)

* **Donation of pyarrow-stubs:** The project voted to accept the donation of the pyarrow-stubs codebase, which will bring high-quality Python type annotations directly into the Arrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Proposal for Canonical BigDecimal extension type:** Discussion is underway for a new extension type to support arbitrary-precision decimals with variable scale per row, facilitating high-fidelity data transfer from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IBM Db2 integration via ADBC:** IBM engineers proposed adding official support for Db2, with initial focus on building an ADBC driver and exploring Flight SQL for high-performance data transport. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **IPC representation of dictionary-encoded extension types:** Developers identified an ambiguity in IPC serialization where dictionary-encoded extension types are indistinguishable from extensions with dictionary storage, proposing a metadata-based fix. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL is_update field proposal:** A proposal and vote were initiated to add an is_update boolean to prepared statement results, allowing clients to automatically determine the correct network flow for DDL/DML vs. queries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust implementation security policy:** A new security policy for the arrow-rs and Parquet Rust implementations is being discussed to better manage vulnerability reporting and distinguish security issues from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Apache Arrow ADBC 23 release:** The ADBC subproject announced its 23rd release, featuring 41 bug fixes and improvements across several language bindings and drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Apache%20Arrow%20ADBC%2023%20Released)

* **Arrow C Device Data Interface experimentality:** Discussion on separating the experimental status of the device-specific and async components of the C API to encourage faster validation of the async interface. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)
