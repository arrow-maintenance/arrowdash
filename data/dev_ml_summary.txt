* **Arrow 24.0.0 release:** The community finalized the release of Apache Arrow 24.0.0, which resolves over 250 issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** The project voted to accept the donation of the pyarrow-stubs repository, integrating PEP 484 type annotations directly into the PyArrow codebase for better developer tooling support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL execution path:** Proposal and vote to add an is_update field to prepared statements to help clients distinguish between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **BigDecimal Extension Type:** Discussion on a new canonical extension type for arbitrary-precision decimals to support variable scale per row, facilitating data transfer from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **C Device Data Interface stability:** Developers discussed the experimental status of the C Device Data Interface and explored the potential for a non-device-specific async API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **IPC for dictionary-encoded extension types:** Investigation into serialization limitations that prevent distinguishing between certain dictionary and extension type nestings in the IPC schema. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **LargeMap type addition:** Proposal to add a LargeMap type to the Arrow specification, supporting 64-bit offsets to handle map structures exceeding 2GB. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **ADBC 23 release:** The Apache Arrow ADBC subproject released version 23, including 41 bug fixes and improvements across its multi-language database drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **ADBC logo proposal:** Discussion regarding a new visual identity for the ADBC subproject to align it with the core Apache Arrow branding. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **Numba support for PyArrow:** Technical discussion on improving Numba integration, specifically addressing challenges related to C++ ABI instability and internal ABI exposures. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)
