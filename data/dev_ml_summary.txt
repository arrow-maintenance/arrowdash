* **Erlang implementation donation:** The community approved the donation of a new Erlang implementation (initially based on Rust bindings), which is currently being transferred to an official Apache repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **IBM Db2 support via ADBC:** Developers from IBM proposed adding Arrow support for Db2, starting with an ADBC driver that will be hosted under the community-maintained driver foundry. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **SIMD optimizations for Parquet Bloom Filters:** Proposal to optimize Parquet SBBF (Split-Block Bloom Filter) probes in C++ using AVX2 and NEON, with the final approach favoring branchless autovectorization for better cross-platform maintainability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** Discussion on a new API to support parallel, distributed bulk writes for engines like Spark and Flink, featuring atomic commit and abort semantics across multiple worker connections. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical Range extension type:** Proposal for a standard `arrow.range` extension type to represent bounded mathematical intervals, including debate on whether boundary closedness should be defined at the schema level or per-value. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **UTF-8 restriction for metadata:** Discussion on whether to strictly enforce UTF-8 encoding for field and schema metadata to align with Flatbuffers requirements and prevent interoperability issues between implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Flight SQL dialect metadata:** Proposal to add new `SqlInfo` codes for limit/offset syntax and literal formats to help clients generate backend-specific SQL for pushdown operations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **Automated GitHub Copilot reviews:** The project is trialing automated AI-generated reviews for pull requests to help catch trivial issues early and reduce the workload on human maintainers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)
