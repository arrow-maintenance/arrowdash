* **Donation of Arrow Erlang implementation:** The community voted to accept the donation of a new Erlang implementation, which is currently based on bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** A major proposal was introduced to add partitioned bulk ingestion to ADBC, enabling distributed writers like Spark or Flink to perform parallel, atomic writes to RDBMS and table formats. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **UTF-8 enforcement in metadata:** Discussion on whether the C++ IPC writer and other implementations should strictly enforce UTF-8 for key-value metadata to ensure compliance with Flatbuffers requirements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **IBM Db2 ADBC Driver:** IBM engineers proposed and initiated work on a new ADBC driver for IBM Db2, which will be maintained under the adbc-drivers "foundry" organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **SIMD optimizations for Parquet Bloom Filters:** Proposal to port AVX2-accelerated Split Block Bloom Filter (SBBF) probes from the Go implementation to C++ using the xsimd library for significant performance gains. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **New Flight SQL dialect metadata:** Proposal to add new SqlInfo codes to Flight SQL to describe backend-specific SQL dialects, such as supported LIMIT/OFFSET syntax and boolean literal forms. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Apache Arrow .NET 23.0.0 release:** The .NET implementation was updated to version 23.0.0, including new support for Flight SQL and improved operation handling. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20.NET%2023.0.0%20released)

* **Community Over Code Glasgow Hackathon:** Planning has begun for a dedicated Arrow hackathon and contributor room at the Community Over Code conference in October 2026. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Hackathon%20at%20Community%20Over%20Code%20Glasgow%202026%20%E2%80%94%20interest%20from%20Arrow%3F)

* **Flight SQL "is_update" field:** A vote passed to add an `is_update` field to prepared statement results, allowing clients to programmatically distinguish between DDL/DML and queries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)
