* **Arrow Erlang donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which currently provides bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **ADBC partitioned bulk ingest:** Proposal for a new API to support distributed write workloads by allowing coordinators to manage parallel partition writes via workers before an atomic commit. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **UTF8 metadata restriction:** Discussion on enforcing UTF8 validation for field and schema metadata in the C++ implementation to align with Flatbuffers specifications and improve interoperability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Parquet Bloom Filter optimization:** Investigation into using SIMD and compiler autovectorization to optimize Split-Block Bloom Filter (SBBF) probes, yielding significant performance gains on modern CPUs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **IBM Db2 ADBC support:** IBM engineers proposed adding Arrow support for Db2, starting with a community-maintained ADBC driver to enable efficient columnar data exchange. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Flight SQL protocol enhancements:** Vote passed to add an `is_update` field to `ActionCreatePreparedStatementResult`, allowing clients to distinguish between queries and DDL/DML updates during statement preparation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust patch releases:** The community coordinated and released several patch versions for the Rust implementation (58.3.0, 57.3.1, and 56.2.1) to address potential offset overflow vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.3.0%20RC1)

* **Arrow .NET 23.0.0 release:** The .NET subproject announced the release of version 23.0.0, which includes improved release verification scripts for both Linux and Windows environments. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20.NET%20Release%20Apache%20Arrow%20.NET%2023.0.0%20RC0)

* **Flight SQL dialect metadata:** Proposal to add new SqlInfo codes to the Flight SQL protocol to help clients detect backend-specific dialect details like limit/offset grammar and boolean literal formats. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)
