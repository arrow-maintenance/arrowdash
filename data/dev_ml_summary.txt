* **Conbench v2 development:** Wes McKinney proposed a major update to the Conbench benchmarking platform, including a rewritten backend in Go and a Svelte-based frontend to improve performance and UI ergonomics. An experimental parallel deployment is live to evaluate new features like DuckDB and Parquet-backed result storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Arrow 25.0.0 release:** The community approved the 25.0.0 release, featuring over 220 resolved issues. There is ongoing discussion regarding a follow-up 25.0.1 patch release to address a significant performance regression in PyArrow's `to_pylist` conversion impacting Spark 4.2 users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **ADBC extension for DuckDB:** Sam Arch proposed donating a hand-coded DuckDB extension for ADBC to the Arrow project to provide a neutrally governed, official client for connecting DuckDB to various databases like Snowflake and BigQuery. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **JSON representation of Arrow schemas:** Discussion is underway to define a canonical human-readable JSON format for schemas to improve ergonomics in REST APIs and ADBC metadata tasks where binary IPC blobs are inconvenient. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating Tensor IPC messages:** Antoine Pitrou proposed deprecating the experimental Tensor and SparseTensor IPC messages, suggesting developers use the more widely supported canonical extension types (FixedShapeTensor) instead. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **CI resource optimization:** The project successfully reduced its GitHub Actions CI usage by 73% following concerns from ASF Infrastructure regarding the consumption of shared runner resources. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Variant Type implementation:** Developers are working on bringing Variant type support to the C++ implementation, using the existing Rust implementation as a reference for handling shredded and unshredded data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **Spec clarification on field names:** Technical discussion clarified that while the Arrow format supports arbitrary UTF-8 field names, the C Data Interface requires producers to handle or error on nul bytes to prevent issues with C-style string reading. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **Bug squashing initiative:** Maintainers are calling for community help to clear out a backlog of older bug reports and issues, specifically looking for volunteers to verify if old bugs are reproducible in current Arrow versions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)

* **Rust Object Store 0.14.0 release:** The Rust implementation of the Object Store reached version 0.14.0, including new integration tests against Azure Storage and Microsoft OneLake. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%20Object%20Store%200.14.0%20RC1)
