* **Arrow 25.0.0 release:** The community approved the major July release, which includes over 220 resolved issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **JSON representation of Arrow schemas:** Discussion on establishing a canonical human-readable JSON format for schemas to improve ergonomics for REST APIs and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecation of IPC Tensor messages:** Proposal to deprecate experimental Tensor and SparseTensor messages in the IPC format in favor of more modern and integrated extension types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **DuckDB ADBC extension donation:** Columnar proposed donating a new DuckDB extension for ADBC to the ASF, leading to a technical comparison with the existing community-led `adbc_scanner` project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Conbench v2 modernization:** Wes McKinney demonstrated a prototype for a rewritten benchmarking dashboard using Go and Svelte to provide better performance and a modern UI for tracking regressions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Variant type support and Parquet alignment:** Discussions on implementing Variant type support in C++ and adjusting the Parquet format to allow omitting the value field to match Arrow's behavior. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **C Data Interface field name spec:** Technical debate over how the C FFI should handle field names containing nul bytes, which are valid in Flatbuffers/UTF-8 but cause issues for standard C string pointers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **CI resource usage reduction:** Following concerns from ASF Infrastructure, the project successfully reduced its GitHub Actions usage by approximately 30% through more efficient workflow management. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Community "Bug Squash" initiative:** Maintainers are calling for help to review and close outdated issues, providing a spreadsheet to track progress across different language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
