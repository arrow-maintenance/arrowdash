* **Arrow 24.0.0 release:** The community approved and released version 24.0.0, which resolves over 250 issues across multiple language implementations including C++, Python, and Go. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Discussion on integrating a Numba extension layer directly into PyArrow to enable JIT compilation for Arrow data while managing C++ ABI stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **Java minimum JDK upgrade:** Consensus was reached to set JDK 17 as the minimum required version for Arrow Java starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **PyArrow stubs donation:** The community voted to accept the donation of the pyarrow-stubs codebase into the main repository to improve the maintenance and availability of type hints for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **BigDecimal extension type:** Proposal for a new canonical extension type to represent arbitrary-precision decimals with variable scale per row, facilitating high-fidelity data transport from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **C Device Data Interface status:** Developers discussed whether to move the C Device Data Interface out of "experimental" status and weighed the pros and cons of adding a non-device async API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **LargeMap format addition:** Proposal to add a LargeMap type to the Arrow specification using 64-bit offsets to support data processing in libraries that exclusively use "large" variants of types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Rust security policy:** Discussion on establishing formal security policies and vulnerability reporting guidelines for the Arrow-rs and Parquet Rust implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Security%20Policy)

* **Flight SQL execution paths:** Proposal to add an is_update field to the Flight SQL protocol to help clients correctly distinguish between query and update paths when using prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)
