* **Apache Arrow 24.0.0 Release:** The community completed the release of version 24.0.0, which includes 259 resolved issues across various implementations and format updates. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** The community voted to accept the donation of the pyarrow-stubs codebase to officially integrate PEP 484 type annotations into the PyArrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **C Device Data Interface experimentality:** Discussion on moving the C Device Data Interface out of its experimental status and potential design changes to support async stream workflows for non-CUDA devices. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **BigDecimal Extension Type proposal:** A proposal for a new canonical extension type to support arbitrary-precision decimals with variable scale per row, facilitating high-fidelity data transfer from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the official specification to support 64-bit offsets, mirroring the existing `LargeList` and `LargeString` types for high-volume data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC dictionary-encoded extension types:** Identification of a limitation in the IPC format where dictionary-encoded extension types cannot be distinguished from extension types with dictionary storage, leading to a proposal for metadata-based fixes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **ADBC 23 Release:** The ADBC subproject released version 23, featuring 41 resolved issues and version updates for C, C++, Go, Java, Python, and Rust drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Flight SQL execution path updates:** A proposal to add an `is_update` field to prepared statements to help clients correctly choose between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **ADBC Logo Proposal:** The community is reviewing a new logo design for the ADBC subproject to provide a distinct identity within the Arrow family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **Rust Security Policy:** Discussion on establishing a formal security policy for the arrow-rs and parquet-rs crates to better manage vulnerability reporting and bug classification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)
