* **Numba support for PyArrow:** Discussion on integrating a Numba layer into PyArrow to allow scientific calculations on Arrow data using JIT compilation while managing ABI stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Arrow 24.0.0 and ADBC 23 releases:** The community finalized the releases of Apache Arrow 24.0.0 and ADBC 23, resolving hundreds of issues and updating drivers across multiple languages. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Donation of pyarrow-stubs:** The project voted to accept the donation of the pyarrow-stubs codebase to provide built-in PEP 484 type annotations for PyArrow users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **BigDecimal canonical extension type:** Proposal for a new canonical extension type to support arbitrary-precision decimals, enabling better compatibility with systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Flight SQL execution paths:** Discussion and vote on adding an `is_update` field to prepared statements to help Flight SQL clients distinguish between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **C Device Data Interface stability:** Developers discussed the experimental status of the C Device Data Interface, noting that more non-CUDA usage is needed before it can be considered stable. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **IPC representation of dictionary-encoded extension types:** A proposal to add KeyValue metadata to dictionary encoding to fix a gap where dictionary-encoded extension types could not be correctly roundtripped via IPC. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Rust and Parquet security policies:** Introduction of proposed security policies and bug reporting guidelines specifically for the Rust implementation of Arrow and Parquet. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **ADBC logo proposal:** A new logo design for the ADBC subproject was proposed to provide a distinct identity while remaining visually consistent with the broader Arrow ecosystem. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
