* **Arrow Java 19.0.0 release:** The community released version 19.0.0 of the Java implementation, the first release following its move to a dedicated repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Java%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1)

* **JDK 17 minimum for Java 20.0.0:** Consensus was reached to increase the minimum required Java version to JDK 17 for the next major release to align with modern ecosystem standards. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Developers are investigating integrating PyArrow with Numba to enable JIT-compiled scientific calculations on Arrow data, weighing the performance gains against maintenance of internal C++ ABIs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **pyarrow-stubs donation:** A proposal is underway to donate the independently maintained `pyarrow-stubs` library to the project to provide official PEP 484 type annotations for PyArrow. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL Prepared Statements improvement:** A new `is_update` field was proposed and voted on for `ActionCreatePreparedStatementResult` to allow servers to signal whether a statement should be executed via a query or update path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **C Device Data Interface status:** Discussion focused on why the C Device Data Interface remains "experimental" and the potential need for an asynchronous non-device API to better support ADBC workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **ADBC 23 release and driver migration:** Version 23 of the ADBC libraries was released while several drivers (BigQuery, Snowflake, Databricks) continue their migration to the `adbc-drivers` community organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **LargeMap format addition:** A request was made to add a `LargeMap` type to the Arrow specification to support 64-bit offsets, enabling the encoding of maps that exceed the 32-bit limit. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **LLMs for project maintenance:** Community members are exploring using LLMs to automate repository management tasks, such as automatically triaging issues and applying component labels. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LLMs%20to%20aid%20with%20maintenance)
