* **Arrow 10-year anniversary:** The project celebrated its tenth anniversary on February 5th, marking a decade of developing agnostic, efficient standards for columnar data exchange. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20is%2010%20years%20old%20%F0%9F%8E%89)

* **Deprecation of pyarrow.gandiva:** Proposal to deprecate the Gandiva bindings in PyArrow due to low maintenance activity and the fact that they are not currently distributed in PyPI wheels. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Deprecate%20pyarrow.gandiva)

* **IPC format extensions:** Discussion on extending the IPC format with `ApplicationData` on messages and a new `Empty` message type to improve flexibility for stream readers and metadata handling. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20ApplicationData%20and%20IPC%20Empty%20Message%20Type%20Discussion%20-%20Document)

* **Arrow 23.0.1 maintenance release:** The community approved and released version 23.0.1, resolving 28 issues identified since the previous major release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2023.0.1%20-%20RC0)

* **Introducing vgi-rpc:** Announcement of a new open-source RPC framework built on Arrow IPC as a typed, gRPC-free alternative to Flight, featuring shared-memory transports for co-located processes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Introducing%20vgi-rpc%20%E2%80%94%20typed%20Arrow%20RPC%20for%20Python%20%28and%20soon%20other%20languages%29)

* **Flight SQL Prepared Statements:** Proposal to add an `is_update` boolean to help clients explicitly differentiate between query and update execution paths in Flight SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **TIME support for 24:00:00:** Developers are reconsidering the decision to exclude "24:00:00" from the TIME type to ensure full data fidelity when round-tripping data from databases like PostgreSQL and DuckDB. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:TIME%20support%20for%20240000)

* **LLMs for project maintenance:** Discussion on leveraging Large Language Models to automate triage tasks, such as automatically applying component labels to unlabelled issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)
