* **Donation of Erlang implementation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow (currently based on Rust bindings), which has now been transferred to the `apache/arrow-erlang` repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Parquet Bloom Filter SIMD optimization:** Proposal to port AVX2-optimized Split Block Bloom Filter (SBBF) probes to the C++ implementation, resulting in a PR that uses xsimd and branchless autovectorization for 3–5x performance gains. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Flight SQL dialect metadata:** Discussion on adding new `SqlInfo` codes for dialect-specific features like LIMIT/OFFSET syntax and NULLS ordering to help clients like Spark generate correct SQL during pushdown. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new write-side API in ADBC to support distributed parallel ingestion, allowing workers to write partitions independently before an atomic commit by a coordinator. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **New arrow.range canonical extension type:** Proposal to add a canonical extension type for mathematical intervals (e.g., numeric or temporal ranges), including debate on whether to support per-value boundary flags for PostgreSQL compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Automated GitHub Copilot reviews:** The project is testing automated Copilot reviews on pull requests to help identify trivial issues, with ongoing discussion on how to minimize noise for contributors and handle draft PRs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **Conbench benchmarking infrastructure future:** Discussion on the future of the project's benchmarking tool, including a proposal to rewrite the backend in Go (Conbench v2) to improve maintainability and performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Variant type support in C++:** Coordination between multiple contributors to implement a `Variant` type in Arrow C++, which is a key requirement for supporting Iceberg v3 features. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)
