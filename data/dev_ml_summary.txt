* **Arrow Java 19.0.0 and JDK 17 minimum:** The community announced the release of Apache Arrow Java 19.0.0 and is now discussing moving to a minimum of JDK 17 for the upcoming 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Java%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1) [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Typed Arrow RPC for Python (vgi-rpc):** Introduction of vgi-rpc, an open-source RPC framework built on Arrow IPC that offers typed proxies and supports shared-memory transports for zero-copy performance without gRPC dependencies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Introducing%20vgi-rpc%20%E2%80%94%20typed%20Arrow%20RPC%20for%20Python%20%28and%20soon%20other%20languages%29)

* **Flight SQL Prepared Statements execution:** Proposal to add an `is_update` boolean to the prepared statement result to help clients explicitly determine whether to use the query or update execution path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **TIME support for 24:00:00:** Discussion on extending the `TIME` type or adding a new `TIME_WITH_LEAP_SECOND` type to support `24:00:00`, improving compatibility with databases like PostgreSQL and DuckDB. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:TIME%20support%20for%20240000)

* **ADBC driver migration status:** Update on the migration of BigQuery, Databricks, and Snowflake drivers from the main ADBC repository to the dedicated `adbc-drivers` organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Missing IPC Message-level metadata testing:** Identification of a gap in integration testing for `custom_metadata` attached to `RecordBatch` messages within the IPC format, potentially causing implementations to drop this metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Missing%20test%20coverage%20for%20RecordBatch%20based%20custom_metadata%20in%20IPC%20Test%20Files)

* **LLM automation for project maintenance:** Exploration of using Large Language Models to assist with project maintenance tasks, such as automatically triaging issues and applying component labels to unlabelled tickets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)

* **Rust and Go release updates:** The community approved and released version 58.0.0 (and 58.1.0 patch) of the Rust implementation and version 18.5.2 of the Go implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.0.0%20RC1) [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Go%20Release%20Apache%20Arrow%20Go%2018.5.2%20RC0)
