* **Arrow Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, based on Rust bindings, following successful IP clearance and repository transfer. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **AVX2 SIMD optimizations for Parquet Bloom Filters:** Proposal and PR to implement branchless AVX2 and NEON-accelerated Split Block Bloom Filter (SBBF) probes to improve performance in high-throughput query engines. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Proposed canonical range extension type:** Discussion on adding `arrow.range` as a standard extension type to represent mathematical intervals, with ongoing debate over handling uniform vs. per-value boundary inclusivity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **ADBC Partitioned Bulk Ingest API:** Proposal to add a coordinator-worker write API to ADBC, allowing distributed writers like Spark or Flink to perform atomic, parallel bulk ingestion into RDBMS and table formats. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **New Flight SQL dialect metadata:** Addition of several SqlInfo codes to enable clients to detect backend support for specific SQL features like LIMIT/OFFSET syntax and NULLS ordering for better pushdown optimization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Benchmarking infrastructure (Conbench) migration:** Discussion on the health of the project's benchmarking servers and a plan to refactor the Conbench backend into Go for improved stability and performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Automated GitHub Copilot reviews:** The project is trialing automated AI-generated reviews on pull requests to help reduce the human review burden, while establishing protocols for how contributors should interact with AI feedback. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **Arrow Rust 59.0.0 and Java 20.0.0 updates:** Voting is underway for the major Rust release (RC2), and triage has begun for the upcoming milestone release of the Java implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)
