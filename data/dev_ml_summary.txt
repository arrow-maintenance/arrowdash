* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes 259 resolved issues across the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Donation of pyarrow-stubs:** The project voted to accept the donation of the `pyarrow-stubs` codebase, which will now be maintained within the main Arrow repository to provide official type annotations for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **LargeMap format addition:** Discussion on extending the Arrow specification to include a `LargeMap` type using 64-bit offsets, supporting datasets that exceed the limits of the standard 32-bit Map type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Experimentality of C Device Data Interface:** Developers debated the "experimental" status of the C Device Data Interface and explored whether to introduce an async version of the non-device API for better ADBC integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL execution path signaling:** Proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` to help clients distinguish between query and update execution paths in Flight SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IPC representation for dictionary-encoded extension types:** Investigation into a limitation in the IPC schema serialization that prevents distinguishing between dictionary-encoded extension types and extension types with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Canonical BigDecimal extension type:** A proposal for a new canonical extension type to support arbitrary-precision decimals with variable scale per row, improving compatibility with PostgreSQL and BigQuery. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Rust implementation security policy:** Discussion on establishing a formal security policy for the Arrow-rs and Parquet-rs crates to clarify the process for reporting and handling vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)
