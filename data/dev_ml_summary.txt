* **ADBC Partitioned Bulk Ingest API:** Proposal for a new API to support atomic, parallel data ingestion from distributed writers like Spark or Flink via a coordinator-worker model. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Parquet Bloom Filter Optimization:** Discussion on significantly speeding up Split Block Bloom Filter (SBBF) probes in C++ using AVX2 and autovectorization techniques. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet/bloom_filter.cc)

* **Erlang implementation donation:** The community voted to accept the donation of an Arrow Erlang implementation, which currently utilizes bindings for the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **Arrow .NET 23.0.0 release:** The .NET implementation reached version 23.0.0, featuring updated Flight SQL support and a new PowerShell script for release candidate verification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20.NET%2023.0.0%20RC0)

* **Rust implementation security patches:** Several patch releases (58.3.0, 57.3.1, and 56.2.1) were issued to address potential offset overflow issues across multiple versions of the Rust crates. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Planned%20patch%20releases%20this%20week)

* **Flight SQL dialect metadata:** Proposal to add new SqlInfo codes to help clients identify backend-specific SQL syntax for features like limit/offset grammar and boolean literals. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **IBM Db2 support:** Discussion on adding IBM Db2 support to the Arrow ecosystem, starting with an ADBC driver hosted in the independent driver foundry. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **UTF-8 restrictions for metadata:** Technical discussion on whether the C++ implementation and IPC format should strictly enforce UTF-8 validation for field and schema metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field/schema/custom%20metadata%20restriction%20to%20UTF8%3F)

* **Pyarrow stubs donation:** The formal transfer of the `pyarrow-stubs` library to the Apache Arrow project was completed to provide official type hinting support for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)
