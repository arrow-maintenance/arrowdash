* **Arrow 25.0.0 Release:** The community passed the vote for the 25.0.0 release, featuring over 200 resolved issues. A potential 25.0.1 patch release was discussed to include performance optimizations for PyArrow `to_pylist` conversions requested for Spark 4.2. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **JSON representation of Arrow schemas:** Discussion on creating a canonical human-readable JSON format for Arrow schemas to simplify application-level tasks like REST API contracts and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:JSON%20representation%20of%20Arrow%20schemas)

* **Conbench v2 development:** Wes McKinney proposed a major overhaul of the benchmarking dashboard using Go and Svelte to improve performance and maintainability. The new version aims to optimize result storage using DuckDB and Parquet files. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Deprecating IPC Tensor messages:** A proposal was made to deprecate experimental Tensor and SparseTensor IPC messages. Developers are encouraged to use canonical extension types (FixedShapeTensor/VariableShapeTensor) which integrate better with standard RecordBatch columns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **ADBC Extension for DuckDB:** Columnar proposed donating a hand-coded DuckDB extension for ADBC to the Arrow project to provide a neutral place for collaboration. The discussion highlighted competition with existing community extensions and the trade-offs of ASF governance versus development speed. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **GitHub Action resource optimization:** Following a report that Arrow was a top consumer of ASF shared runner resources, the team successfully reduced CI usage by approximately 73%. Ongoing efforts focus on maintaining efficiency within the 250,000-minute weekly limit. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Variant Type Support in C++:** Progress is being made on implementing Variant type support in the C++ and Parquet implementations. While the Rust implementation serves as a reference, maintainers emphasized a strict review process to ensure C++ performance and stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **Issue cleanup and "Bug Squashing":** A call for volunteers was issued to help reproduce and close out older bugs and stale issues. A public spreadsheet has been created to coordinate efforts across different language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)

* **C FFI spec clarification for field names:** Discussion regarding how to handle null bytes in field names within the C Data Interface. Maintainers suggest that producers should error out when detecting embedded null bytes to avoid safety issues during string parsing. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)
