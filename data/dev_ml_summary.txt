* **Arrow 24.0.0 release:** Voting has commenced for the major 24.0.0 release, featuring over 250 resolved issues and technical discussions regarding Gandiva test failures in specific timezones. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20%2D%20RC0)

* **ADBC 23 release:** The community announced the release of ADBC 23, including 41 bug fixes and improvements across the C++, Go, Java, and Python drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20%2D%20RC1)

* **Numba support for PyArrow:** Discussion on integrating a Numba layer directly into PyArrow to enable JIT-compiled speedups for Arrow data while addressing concerns about maintenance burden and ABI stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **Donation of pyarrow-stubs:** A vote passed to officially donate the `pyarrow-stubs` codebase to the Apache Arrow project to provide better-maintained PEP 484 type annotations for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type for arbitrary-precision decimals with variable scale per row to ensure high-fidelity transport between Arrow and engines like PostgreSQL or Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **JDK 17 minimum for Java:** Consensus was reached to upgrade the minimum requirement for the Arrow Java 20.0.0 release to JDK 17, following a period of lazy consensus. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type using 64-bit offsets to support map structures that exceed the 2GB capacity of standard 32-bit offsets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Flight SQL execution paths:** Proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` to allow servers to explicitly signal whether a prepared statement should follow the query or update execution flow. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Dictionary-encoded extension types in IPC:** Investigation into IPC serialization gaps that prevent distinguishing between a dictionary of extension types and an extension type using a dictionary storage type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary%2Dencoded%20extension%20types)

* **C Device Data Interface experimentality:** Developers discussed the criteria for moving the C Device Data Interface out of experimental status, emphasizing the need for more diverse non-CUDA implementation usage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)
