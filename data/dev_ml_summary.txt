* **Donation of pyarrow-stubs:** The community approved the donation of the `pyarrow-stubs` codebase to provide official static type hints for Python, which will be integrated directly into the Arrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Proposal for Canonical BigDecimal Type:** Discussion is underway for a new canonical extension type to support arbitrary-precision decimals with variable scales, facilitating high-fidelity data exchange with systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC Dictionary Representation:** Developers are addressing a limitation in IPC serialization where dictionary-encoded extension types are indistinguishable from extension types with dictionary storage, proposing new metadata fields to resolve the ambiguity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Apache Arrow 24.0.0 Release:** The community successfully released version 24.0.0, which includes over 250 resolved issues across multiple language implementations and components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Rust Implementation Security Policy:** A new security policy is being established for the Rust implementation to provide formal guidelines for reporting vulnerabilities and distinguishing them from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **IBM Db2 Integration:** IBM developers have proposed adding native Arrow support for Db2, starting with the development of an ADBC driver to be hosted under the community-maintained driver foundry. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **UTF-8 Metadata Requirements:** A discussion has been raised regarding whether to strictly enforce UTF-8 encoding for field and schema metadata to ensure compatibility with Flatbuffers and various language bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8)

* **ADBC Logo Proposal:** The community is reviewing a proposed visual identity for the ADBC subproject to differentiate it while maintaining alignment with the broader Apache Arrow brand. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)

* **Community Hackathon Planning:** Preliminary interest is being gathered for an Arrow hackathon track at the Community Over Code Glasgow 2026 conference to tackle bug fixes and documentation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Hackathon%20at%20Community%20Over%20Code%20Glasgow%202026%20%E2%80%94%20interest%20from%20Arrow)
