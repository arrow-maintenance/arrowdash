* **Apache Arrow 24.0.0 release:** The community announced the general availability of version 24.0.0, encompassing over 250 resolved issues across the multi-language ecosystem. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Canonical BigDecimal extension type:** Proposal for a new extension type to support arbitrary-precision decimals with variable scale per row, improving interoperability with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IBM Db2 integration:** Discussion on adding Arrow support for IBM Db2, focusing on the development of an ADBC driver and potential Flight SQL implementation for high-performance data transport. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **AVX2 Split Block Bloom Filter probes:** Proposal to port AVX2-optimized Bloom filter probing from the Go implementation to C++ to significantly improve Parquet read performance on supported hardware. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter%2Ecc)

* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Arrow, expanding the ecosystem to the BEAM virtual machine. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Dialect-specific SqlInfo codes:** Proposal to add new Flight SQL metadata codes to describe backend-specific SQL syntax for LIMIT/OFFSET and NULL ordering, aiding pushdown logic in drivers and Spark integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Rust security policy:** Proposal for a formal security policy for arrow-rs and parquet-rs to establish clear guidelines for reporting and handling vulnerabilities following an increase in security reports. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **UTF-8 metadata enforcement:** Discussion on whether to strictly enforce UTF-8 for key-value metadata to align with Flatbuffers requirements and prevent interoperability issues between implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%20schema%20custom%20metadata%20restriction%20to%20UTF8)

* **ADBC logo and branding:** Proposal and feedback session for a new logo for the ADBC subproject to provide it with a distinct visual identity within the Arrow project family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **IPC file and stream format equivalence:** Draft proposal to update the format documentation to clarify the equivalence and specific deviations between IPC file and stream formats. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Equivalence%20of%20IPC%20file%20and%20stream%20formats)
