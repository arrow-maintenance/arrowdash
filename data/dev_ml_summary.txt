* **Arrow 25.0.0 release:** The release candidate for version 25.0.0 was approved, featuring discussions on a potential patch release to address a performance regression in PyArrow's `to_pylist` that affects Spark 4.2 UDFs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 migration:** Wes McKinney proposed a modernization of the Conbench benchmarking tool, featuring a new Go-based backend and Svelte frontend to improve UI performance and maintainability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Deprecating Tensor and SparseTensor IPC messages:** A proposal to deprecate dedicated Tensor IPC messages in favor of using existing canonical extension types, which allow tensors to be passed as standard RecordBatch columns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Canonical JSON representation of Arrow schemas:** Discussion on creating a human-readable JSON format for Arrow schemas to simplify integration with REST APIs and metadata boundaries in ADBC. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **GitHub Actions resource optimization:** In response to ASF Infrastructure limits, the project successfully reduced its weekly CI usage by over 25% through workflow streamlining and efficiency improvements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **DuckDB ADBC extension donation:** Proposal to donate a DuckDB ADBC extension to the Arrow project, prompting a debate regarding governance and the technical merits of hand-coded versus AI-assisted development. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Variant type specification alignment:** A technical discussion on allowing the `value` field to be omitted in Parquet's Variant encoding to improve consistency with Arrow's specification for shredded values. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **Arrow Erlang donation:** The community confirmed the successful repository transfer of the Erlang implementation to the Apache organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RESULTVOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Rust Object Store 0.14.0 release:** The Rust implementation of the Object Store interface was released, including several performance improvements and updated dependencies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%20Object%20Store%200.14.0%20RC1)

* **Stale issue cleanup:** An initiative was launched to recruit volunteers and maintainers to triage the project's large backlog of bug reports and feature requests. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
