* **Arrow 24.0.0 release:** The community approved and released version 24.0.0, which includes over 250 resolved issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** The community voted to accept the donation of the `pyarrow-stubs` codebase, which will bring PEP 484 type annotations directly into the Arrow repository for better maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** A proposal was introduced for a new canonical extension type to handle arbitrary-precision decimals with variable scales, facilitating high-fidelity data transport from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **ADBC 23 release:** The Arrow Database Connectivity (ADBC) subproject released version 23, featuring 41 bug fixes and improvements across C, Go, Java, Python, and Rust components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **C Device Data Interface status:** Developers discussed why the C Device Data Interface remains marked as experimental three years after introduction, citing low adoption and a lack of non-CUDA validation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL Prepared Statements execution path:** A new `is_update` boolean field was proposed for the Flight SQL protocol to help clients distinguish between query and update paths when executing prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IPC representation of dictionary-encoded extension types:** Discussion on a limitation in the IPC schema serialization that makes it impossible to distinguish between a dictionary with extension values and an extension type using a dictionary as storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Numba support for PyArrow:** Maintainers discussed the complexities of supporting Numba, including the potential for exposing a private ABI to handle non-trivial code generation routines while minimizing maintenance overhead. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Python%20Numba%20support%20for%20PyArrow)

* **LargeMap format addition:** A proposal was made to add a `LargeMap` type to the Arrow specification to support 64-bit offsets, addressing issues in libraries like Polars that rely exclusively on "Large" types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Rust Security Policy:** The `arrow-rs` community is establishing a formal security policy to define guidelines for differentiating between standard bugs and security vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)
