* **Arrow Java 19.0.0 release and JDK 17 proposal:** The community released version 19.0.0 of the Java implementation and is now discussing bumping the minimum requirement to JDK 17 for version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Java%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1) [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL prepared statement execution path:** Proposal to add an explicit `is_update` boolean to help clients distinguish between result-set queries and update counts, potentially leading to consolidated execution endpoints. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Introducing vgi-rpc for Python:** Announcement of a new RPC framework built entirely on Arrow IPC that supports typed proxies, shared memory transports for zero-copy transfers, and avoids gRPC/Protobuf dependencies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Introducing%20vgi-rpc%20%E2%80%94%20typed%20Arrow%20RPC%20for%20Python%20(and%20soon%20other%20languages))

* **Time type support for 24:00:00:** Discussion on revisiting the decision to exclude `24:00:00` from the Time type to ensure better data fidelity when round-tripping with databases like PostgreSQL and DuckDB. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:TIME%20support%20for%20240000)

* **IPC custom_metadata test coverage:** Proposal to add integration tests specifically for `custom_metadata` within `RecordBatch` messages to ensure cross-language implementations handle per-message metadata correctly. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Missing%20test%20coverage%20for%20RecordBatch%20based%20custom_metadata%20in%20IPC%20Test%20Files)

* **Map type field naming consistency:** Developers are debating whether Map type child fields must strictly use "key" and "value" names or if implementations should support custom field names for better format flexibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type's%20key%2Fitem%2Fvalue%20field%20names)

* **ADBC driver migration to dedicated organization:** Progress update on moving BigQuery, Databricks, and Snowflake drivers to the `adbc-drivers` GitHub org to decouple them from the main ADBC source tree. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Java Flight UDS regression:** Identification of a critical bug in Java Flight (versions 16-18) where using Unix Domain Sockets with Epoll triggers a `ClassCastException`. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Flight%20Critical%20Regression%20in%20v16-v18%20FlightClient%20UDS%20throws%20ClassCastException)

* **Rust implementation releases:** Recent activity includes the approval of Arrow Rust 58.0.0 and subsequent maintenance releases for the Rust object store and core libraries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20Rust%2058.0.0%20RC1) [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1)
