* **Arrow 24.0.0 Release:** Voting is underway for the 24.0.0 release candidate, which includes over 250 resolved issues across the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Discussion on integrating Numba JIT compiler support directly into PyArrow to enable high-performance computations on Arrow data while avoiding C++ ABI stability issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **pyarrow-stubs donation:** The community voted to accept the donation of the pyarrow-stubs codebase, bringing official PEP 484 type annotations into the core repository for better developer experience. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Java JDK 17 minimum:** Developers reached a consensus to set JDK 17 as the minimum requirement for Arrow Java 20.0.0, allowing the project to leverage newer language features and performance improvements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **C Device Data Interface stability:** A discussion regarding the experimental status of the C Device Data Interface and the potential for adding asynchronous stream support for non-CUDA workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Arrow ADBC 23 Release:** The project released version 23 of the Arrow Database Connectivity (ADBC) libraries, spanning improvements across C++, Go, Java, Python, and Rust. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **IPC serialization of extension types:** Investigation into improving how the IPC format represents dictionary-encoded extension types, with a proposal to use KeyValue metadata to distinguish them from standard dictionary types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL protocol update:** Proposal to add an `is_update` field to the Flight SQL protocol to help clients distinguish between query and update execution paths for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, similar to existing LargeList and LargeString types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)
