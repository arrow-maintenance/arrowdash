* **Arrow 24.0.0 release:** The community approved and released version 24.0.0, featuring over 250 resolved issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donation of pyarrow-stubs:** A vote passed to donate the pyarrow-stubs codebase to Apache Arrow, integrating first-class PEP 484 type annotations directly into the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type for arbitrary-precision decimals to support variable scales per row, common in engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion on adding a LargeMap type to the Arrow specification to support 64-bit offsets for map structures, addressing capacity limits in libraries like Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **ADBC 23 release and branding:** Version 23 of the ADBC libraries was released, and a discussion was initiated regarding a new official logo for the subproject to distinguish it within the ecosystem. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **IPC representation of dictionary-encoded extension types:** Investigation into a limitation in the IPC schema serialization that prevents distinguishing between certain dictionary and extension type nestings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL prepared statement updates:** Proposal to add an `is_update` boolean field to Flight SQL to help clients determine the correct network flow for executing prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust security policy:** The project is establishing a formal security policy for the Rust implementations of Arrow and Parquet to improve vulnerability reporting and handling. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **Async non-device API discussion:** Debate over providing a non-device-specific async C API to separate general asynchronous interests from experimental GPU-related interfaces. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **ADBC logo proposal:** Discussion on adopting a new visual identity for the ADBC subproject to maintain consistency with the Arrow brand while remaining differentiated. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)
