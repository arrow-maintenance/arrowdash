* **Flight SQL execution path indication:** Discussion and vote on adding an `is_update` boolean field to prepared statements to help clients correctly determine whether to use the query or update execution path. * **C Device Data Interface stability:** Developers discussed the experimental status of the C Device Data Interface, exploring its current usage in GPU workflows and the potential for an asynchronous version of the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Apache Arrow ADBC 23 release:** The ADBC subproject released version 23, including 41 bug fixes and improvements across the C++, Go, Python, Java, and Rust implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20%2D%20RC1)

* **LargeMap format addition:** Proposal to add a `LargeMap` type to the Arrow specification to support 64-bit offsets, enabling compatibility with engines like Polars that rely on large offset types for processed data. * **Apache Arrow 24.0.0 release:** Major release featuring over 250 resolved issues across the core library and multiple language bindings, including performance optimizations and expanded format support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024%2E0%2E0%20%2D%20RC0)

* **BigDecimal extension type proposal:** Discussion on a new canonical extension type for `BigDecimal` to handle arbitrary-precision decimals with variable scale per row, facilitating high-fidelity data transfer from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Security%20Policy)

* **Dictionary-encoded extension types in IPC:** Investigation into IPC serialization limitations for dictionary-encoded extension types, with a proposal to add metadata fields to the `DictionaryEncoding` table to resolve round-tripping issues.
