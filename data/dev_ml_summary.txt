* **JDK 17 minimum for Arrow Java 20.0.0:** Following the release of version 19.0.0, the community reached consensus on increasing the minimum requirement to JDK 17 for the next major release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Investigation into supporting PyArrow arrays within Numba JIT-compiled functions to improve performance in scientific computing workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Apache Arrow ADBC 23 release:** The latest ADBC release includes 41 resolved issues across C++, Go, Java, Python, and Rust, along with the migration of several drivers to a separate organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Donation of pyarrow-stubs:** A proposal to move the independently maintained `pyarrow-stubs` project into the official Apache Arrow repository to provide integrated PEP 484 type annotations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Status of C Device Data Interface:** Developers discussed the "experimental" status of the C Device Data Interface and the potential need for an asynchronous version of the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL protocol update:** A new `is_update` field was added to the `ActionCreatePreparedStatementResult` to help clients determine the correct network flow for executing prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type using 64-bit offsets to provide better compatibility with libraries like Polars that rely on Large* types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Dictionary-encoded extension types in IPC:** Research into serialization issues where dictionary-encoded extension types cannot be correctly represented in the IPC schema format. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
