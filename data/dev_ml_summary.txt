* **Column default value metadata in Flight SQL:** Discussion on adding `COLUMN_DEF` to the `table_schema` within `FlightSql.proto` to allow JDBC and other clients to retrieve column default values. This change aims to improve metadata parity with standard database interfaces. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJDBC%20Column%20Default%20Value%20Metadata%20-%20FlightSql.proto%20Changes)

* **Donation of Arrow Erlang implementation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which is currently based on Rust bindings. The project is transitioning from a personal repository to the official Apache infrastructure. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **AVX2 optimization for Parquet Bloom Filters:** A proposal to port AVX2-accelerated Split Block Bloom Filter (SBBF) probes from the Go implementation to C++. Benchmarks show significant performance gains, particularly for in-cache filter operations, by using branchless SIMD kernels. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new set of API functions to support parallel data ingestion in ADBC. This would allow distributed engines like Spark to coordinate parallel writes to RDBMS or table formats like Iceberg with atomic commit/abort semantics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **New canonical extension type for bounded ranges:** Discussion on introducing `arrow.range` to represent mathematical intervals (e.g., `[lower, upper)`). The proposal covers storage layouts using structs and metadata for handling inclusive/exclusive boundaries and PostgreSQL compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **GitHub Copilot for automated PR reviews:** The project is trialing automated GitHub Copilot reviews to alleviate maintainer burden by catching trivial issues early. Current discussions focus on refining the workflow, such as excluding draft PRs and establishing response expectations for contributors. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **Benchmarking infrastructure (Conbench) status:** Developers discussed the health of the Conbench benchmarking platform, including its migration to new servers and plans for a "v2" redesign to improve performance and stability. Historical benchmark data has been preserved during the infrastructure transition. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **FlightSQL dialect-related SqlInfo codes:** Proposal to add new metadata codes to Flight SQL to describe backend-specific SQL syntax, such as limit/offset grammar and null ordering. This metadata helps clients like Spark generate correct SQL for pushdown operations without hardcoded configuration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlightSQL%20Add%20dialect-related%20SqlInfo%20codes)
