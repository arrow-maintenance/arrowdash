* **Arrow 25.0.0 release:** Major release featuring over 220 resolved issues; a follow-up patch release is discussed to optimize nested type conversion in PyArrow for Spark users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 proposal:** Wes McKinney introduced a major update to benchmarking infrastructure, including a Go backend and Svelte frontend to improve performance and UI ergonomics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **JSON representation of Arrow schemas:** Discussion on establishing a canonical JSON format for schemas to improve interoperability for REST APIs and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **ADBC extension for DuckDB:** Proposal to donate a hand-coded ADBC client for DuckDB to the project, sparking debate on collaboration with existing community-led versions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Deprecating Tensor IPC messages:** Suggestion to deprecate experimental Tensor and SparseTensor IPC messages as modern extension types now provide more robust functionality. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Variant type support in C++:** Efforts to implement shredded and unshredded Variant types in C++ are underway to match the current Rust implementation and enhance Parquet compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **CI resource optimization:** The project successfully reduced its GitHub Actions usage by roughly 27% to comply with ASF shared runner resource policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Old issue triage effort:** A coordinated effort is starting to reproduction-test and close obsolete bug reports across all language implementations to clean up the issue tracker. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)

* **Parquet Variant specification:** Proposal to update the Parquet format specification to allow the omission of the Variant value field to align better with Arrow's specification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)
