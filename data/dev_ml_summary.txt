* **Arrow 24.0.0 release:** The major April release was finalized, resolving 259 issues across language implementations including C++, Python, and Java. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs codebase to provide official PEP 484 type annotations for PyArrow. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal for a new extension type to support arbitrary-precision decimals with variable scale per row, improving compatibility with PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type using 64-bit offsets to the Arrow specification to handle maps with more than 2 billion entries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **IPC representation of dictionary-encoded extension types:** Investigation into a limitation where IPC serialization cannot currently distinguish between dictionary-encoded extension values and extension types using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL prepared statement execution:** Proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` to help clients correctly route execution between query and update paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust implementation security policy:** Ongoing work to establish a formal security policy for `arrow-rs` and `parquet` to better manage vulnerability reporting and bug classification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **ADBC project logo:** Discussion on adopting a formal logo for the Arrow Database Connectivity (ADBC) subproject to align with the project's visual identity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)

* **Apache Arrow ADBC 23 release:** The latest version of the ADBC libraries was released, featuring 41 bug fixes and improvements across Go, Java, and Python drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)
