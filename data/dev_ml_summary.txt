* **Numba support for PyArrow:** Discussion on integrating a Numba layer into PyArrow to enable JIT-compiled computation on Arrow data, balancing the benefits of performance against concerns regarding maintenance and ABI stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs codebase, which will bring official PEP 484 type stub annotations to the PyArrow project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **C Device Data Interface stability:** Discussion regarding whether the C Device Data Interface should remain marked as experimental, with contributors suggesting it needs more non-CUDA usage to validate its design. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Java JDK 17 minimum requirement:** The project reached a consensus to set JDK 17 as the minimum required version for Arrow Java starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Apache Arrow ADBC 23 release:** Version 23 of the Arrow Database Connectivity (ADBC) libraries has been released, featuring 41 resolved issues across C, C++, Go, Java, and Python. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **LargeMap format proposal:** Proposal to add a LargeMap type to the Arrow specification to support 64-bit offsets, addressing interoperability issues with libraries like Polars that use "Large" variants by default. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC dictionary-encoded extension types:** Developers identified an issue where dictionary-encoded extension types cannot be correctly distinguished in IPC schema serialization and proposed adding custom metadata to resolve it. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL execution path updates:** A vote was initiated to add an `is_update` field to the Flight SQL protocol, allowing servers to inform clients whether a prepared statement should follow a query or update path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Arrow 24.0.0 release candidate:** Voting is underway for the 24.0.0 release candidate, which includes 258 resolved GitHub issues and improvements to C++, Python, and binary verification workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)
