* **Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes 259 resolved issues across various implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Apache%20Arrow%2024.0.0%20released)

* **Canonical BigDecimal Extension Type:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scale per row, facilitating better data movement between Arrow and engines like PostgreSQL or Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Rust implementation security policy:** Discussion on establishing a formal security policy for `arrow-rs` and Parquet in Rust to handle the increasing volume of security reports and clarify the distinction between bugs and vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which is currently built as a binding to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **UTF-8 metadata restrictions:** Discussion regarding whether to strictly enforce UTF-8 encoding for field and schema metadata to resolve potential IPC serialization issues within the Flatbuffers and C Data Interface specifications. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **AVX2 Bloom Filter implementation:** Proposal to port high-performance SIMD-accelerated Split Block Bloom Filter (SBBF) probes to the Parquet C++ implementation, leveraging xsimd for cross-platform efficiency. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter%2Ecc)

* **ADBC Partitioned Bulk Ingest API:** A proposal to add a new API to ADBC that supports coordinated, parallel data ingestion from distributed writers like Spark or Flink into databases and table formats like Iceberg. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Flight SQL dialect metadata:** Proposal to add new `SqlInfo` codes to Flight SQL to help clients identify backend-specific SQL dialect features like LIMIT/OFFSET syntax and boolean literal formats. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **IBM Db2 support for Arrow:** IBM engineers proposed adding official support for IBM Db2 through ADBC drivers and potentially Arrow Flight to improve performance for analytical workloads. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)
