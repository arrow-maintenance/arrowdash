* **Conbench v2 Proposal:** Wes McKinney introduced a Go-based backend and TypeScript frontend rewrite of the project's benchmarking dashboard to improve performance and simplify maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **ADBC Extension for DuckDB:** Columnar proposed donating a new DuckDB extension for ADBC to the ASF, sparking a discussion on community fragmentation and the role of AI-assisted development in open source. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **JSON Representation of Arrow Schemas:** Discussion on establishing a canonical, human-readable JSON format for Arrow schemas to simplify integration with REST APIs and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating IPC Tensor Messages:** Proposal to deprecate experimental Tensor and SparseTensor IPC messages in favor of using modern canonical extension types within standard RecordBatches. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Arrow 25.0.0 Release:** The community approved the 25.0.0 release, deciding to follow up with a 25.0.1 patch to address specific PyArrow performance improvements requested for Spark 4.2. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Parquet Variant Specification Alignment:** Proposal to modify the Parquet format description to allow the `value` field to be omitted in Variant types, bringing it into alignment with the Arrow specification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **C Data Interface NUL Bytes:** Developers explored how the C Data Interface should handle NUL bytes in field names to ensure security and prevent crashes during FFI exchanges. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **CI Resource Management:** Following a request from ASF Infrastructure, the project successfully reduced its GitHub Actions runner consumption by approximately 25% through workflow optimizations.
