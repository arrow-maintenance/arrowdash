* **Numba support for PyArrow:** Discussion on a proposal to integrate Numba JIT support into PyArrow, which would allow Python developers to write high-performance functions that operate directly on Arrow data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **JDK 17 minimum for Arrow Java:** The community has reached consensus to move the minimum requirement for the Java implementation to JDK 17 starting with the 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Donation of pyarrow-stubs:** The project accepted the donation of the pyarrow-stubs codebase, which will integrate official PEP 484 type annotations directly into the PyArrow library. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Arrow 24.0.0 release:** Voting is underway for the 24.0.0 release candidate, which includes over 250 resolved issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Flight SQL Prepared Statement updates:** A protocol change was approved to add an `is_update` field to the prepared statement creation result, allowing clients to correctly distinguish between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **ADBC 23 release:** The ADBC subproject released version 23, delivering 41 bug fixes and improvements across its C++, Go, Python, and Java components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **LargeMap format addition:** Discussion on extending the Arrow specification to include a `LargeMap` type, supporting 64-bit offsets for map arrays to better handle large-scale datasets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC dictionary-encoded extension types:** Developers are investigating a limitation in the IPC format that prevents correctly distinguishing between dictionary-encoded extension types and extension types with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Rust implementation updates:** Recent activity includes successful releases for Arrow Rust version 58.1.0 and the Rust Object Store implementation version 0.13.2. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1)
