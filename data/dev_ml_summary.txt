* **Apache Arrow 24.0.0 release:** The community released version 24.0.0, resolving 259 issues across the ecosystem including significant updates to the C++, Python, and Java implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the `pyarrow-stubs` codebase, which will bring official PEP 484 type annotations into the main Arrow repository for Python developers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Proposal for LargeMap format:** Discussion began on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, mirroring existing `LargeList` and `LargeString` types for high-volume data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Canonical BigDecimal extension type:** A new proposal introduces a canonical extension type for `BigDecimal` to handle arbitrary-precision decimals with variable scales, improving interoperability with systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Apache Arrow ADBC 23 release:** Version 23 of the Arrow Database Connectivity (ADBC) libraries was released, featuring 41 bug fixes and performance improvements across multiple language drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **IPC representation of dictionary-encoded extensions:** Developers are discussing a specification gap where IPC serialization cannot distinguish between dictionary-encoded extension types and extension types using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL prepared statement updates:** A proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` aims to help clients correctly route execution between query and update paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust implementation security policy:** The `arrow-rs` community is establishing a formal security policy and clear guidelines for distinguishing between standard bugs and exploitable vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **ADBC project logo:** The community is evaluating a new visual identity and logo for the ADBC subproject to differentiate it while maintaining consistency with the broader Arrow family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
