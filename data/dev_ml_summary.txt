* **Arrow 23.0.1 release and provenance improvements:** The patch release 23.0.1 was approved following a discussion on release verifiability, leading to a new requirement to include SHA256 hashes directly in future release vote emails. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2023.0.1%20-%20RC0)

* **10-year anniversary of Apache Arrow:** The community celebrated the project's tenth anniversary since its founding in February 2016, reflecting on its evolution into a universal standard for columnar data exchange. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20is%2010%20years%20old%20%F0%9F%8E%89)

* **Release artifact integrity and GitHub stability:** Developers discussed SHA512 mismatches for ADBC 21, reinforcing that GitHub-generated source archives are not stable and canonical Apache release artifacts must be used for reliable verification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:SHA512%20mismatch%20for%20apache-arrow-adbc-21.tar.gz)

* **IPC format extensions for application data:** Proposal to extend the IPC format with `ApplicationData` fields on messages and a new `Empty` message type to support custom metadata transmission without full record batches. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20ApplicationData%20and%20IPC%20Empty%20Message%20Type%20Discussion%20-%20Document)

* **Support for 24:00:00 in TIME types:** Discussion on expanding the `TIME` type range to include `24:00:00` to improve data fidelity and compatibility with databases like PostgreSQL and DuckDB. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:TIME%20support%20for%2024%2000%2000)

* **vgi-rpc framework introduction:** Announcement of an open-source, typed RPC framework built on Arrow IPC that supports shared memory transports and provides a Python-first alternative to Flight/gRPC. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Introducing%20vgi-rpc%20%E2%80%94%20typed%20Arrow%20RPC%20for%20Python%20%28and%20soon%20other%20languages%29)

* **Arrow Rust 58.0.0 release:** The Rust implementation released version 58.0.0, which includes breaking API changes for performance optimizations and improved ergonomics in the core crates. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.0.0%20RC1)

* **Gandiva Python bindings deprecation:** Proposal to deprecate `pyarrow.gandiva` due to low maintenance activity and lack of inclusion in PyPI wheels, focusing development on more widely used components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Deprecate%20pyarrow.gandiva)

* **Java Flight UDS regression:** Report of a critical bug in Java Flight versions 16-18 where using Unix Domain Sockets with Epoll triggers a `ClassCastException`, breaking compatibility with certain Linux deployments. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Critical%20Regression%20in%20v16-v18%20FlightClient%20UDS%20throws%20ClassCastException)
