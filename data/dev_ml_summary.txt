* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, mirroring existing Large types and improving compatibility with libraries like Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Pyarrow-stubs donation:** The community voted to accept the donation of the `pyarrow-stubs` codebase to the project, which will integrate formal PEP 484 type annotations directly into the PyArrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Arrow 24.0.0 release:** The project successfully released version 24.0.0, resolving over 250 issues across multiple language implementations and improving cross-platform compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type to support arbitrary-precision decimals, facilitating high-fidelity data exchange with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Dictionary-encoded extension types in IPC:** Investigation into a serialization gap where IPC headers currently cannot distinguish between dictionary-encoded extension types and extension types using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL prepared statement execution:** Proposal to add an `is_update` field to prepared statement results, allowing servers to signal the correct network flow (Query vs. Update) to clients without requiring heuristics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust and Parquet security policy:** Discussion on establishing a formal security policy for the Rust implementation to better manage the increasing volume of vulnerability reports and clarify the distinction between bugs and security flaws. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Security%20Policy)

* **IBM Db2 integration:** IBM developers proposed adding native Arrow support for Db2, leading to a discussion on implementing a new ADBC driver and exploring Flight SQL as a high-performance transport layer. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **ADBC logo proposal:** A new logo for the ADBC subproject was proposed to establish a distinct brand identity while maintaining visual alignment with the broader Apache Arrow family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)
