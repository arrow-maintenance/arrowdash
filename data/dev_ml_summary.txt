* **Apache Arrow 24.0.0 release:** Major release including 259 resolved issues across the project; the .NET and Go implementations also saw significant version updates this month. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Canonical BigDecimal extension type:** Proposal for a new extension type to support arbitrary-precision decimals with variable scales per row, facilitating high-fidelity data transport from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IBM Db2 support via ADBC:** Engineers from IBM proposed adding Arrow support for Db2, starting with a new driver in the ADBC driver foundry and exploring future Flight SQL integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Metadata UTF-8 restriction:** Discussion on whether Arrow implementations should strictly enforce UTF-8 validation for field and schema metadata to align with Flatbuffers specifications and improve cross-language reliability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field/schema/custom%20metadata%20restriction%20to%20UTF8%3f)

* **Rust security policy and patch releases:** Proposal to establish formal security guidelines for the Rust implementation alongside a series of patch releases (58.3.0, 57.3.1, 56.2.1) to address potential offset overflow issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Erlang implementation donation:** Progress on IP clearance and software grant documentation for the Erlang implementation to become an official part of the Apache Arrow project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Erlang%20Arrow%20Erlang's%20Grant%20Documents)

* **ADBC logo proposal:** Design discussion for a dedicated logo for the Arrow Database Connectivity subproject to provide visual differentiation while maintaining brand consistency. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **pyarrow-stubs donation:** Successful vote to donate the `pyarrow-stubs` repository to the Apache Arrow project to provide better type-hinting support for Python developers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL prepared statement improvements:** Proposal to add an `is_update` field to prepared statement results, helping drivers automatically determine the correct execution path for DDL/DML queries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)
