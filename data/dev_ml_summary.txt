* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which is currently based on Rust bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new API to support distributed write workloads (e.g., Spark/Flink) by allowing multiple workers to write partitions that are later committed atomically by a coordinator. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical Range extension type:** Discussion on adding `arrow.range` for representing mathematical intervals, including debates on handling inclusive/exclusive boundaries at the column vs. value level. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **AVX2 optimization for Parquet Bloom Filters:** Proposal and PR to implement AVX2-optimized SBBF probes in C++, yielding up to a 5x performance improvement for in-cache filters through SIMD and autovectorization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter%2Ecc)

* **Flight SQL dialect metadata:** Proposal to add new `SqlInfo` codes for LIMIT/OFFSET, NULL ordering, and literals to help clients emit backend-specific SQL without hardcoded driver logic. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **Automated GitHub Copilot reviews:** The project is trialing automated Copilot reviews to catch trivial issues early, with ongoing discussion on best practices for responding to AI comments and avoiding noise on draft PRs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **Conbench infrastructure future:** Developers discussed refactoring the continuous benchmarking project (Conbench) to improve stability and maintainability after migrating it to new community-managed infrastructure. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20+%20conbench%20OSS%20project)

* **Arrow Rust 59.0.0 release:** The community approved and published a new major release of the Rust implementation, along with several maintenance releases for older versions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)

* **JDBC default values via Flight SQL:** A vote passed to update the Flight SQL contract to include column default value support, specifically targeting JDBC connections. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20FLIGHT%20JDBC)

* **Pyarrow-stubs donation complete:** The formal software grant for `pyarrow-stubs` has been filed, successfully bringing the Python type hint repository under the official Apache Arrow project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)
