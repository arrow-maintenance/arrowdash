* **Arrow 24.0.0 release candidate:** Voting has commenced for the 24.0.0 release, featuring over 250 resolved issues across the project's various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **JDK 17 minimum for Arrow Java:** Developers reached a consensus to upgrade the minimum requirement for Arrow Java to JDK 17, starting with the 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** A proposal to integrate Numba support into PyArrow would allow JIT-compiled scientific calculations to run directly on Arrow data, bypassing potential ABI stability issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Apache Arrow ADBC 23 released:** The latest version of the Arrow Database Connectivity (ADBC) libraries was released, including 41 bug fixes and improvements for C, C++, Go, Python, Java, and other languages. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the `pyarrow-stubs` codebase, which will bring official PEP 484 type annotations directly into the PyArrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL execution path updates:** A new `is_update` field was voted into the Flight SQL specification to help clients correctly identify the network flow required for prepared statement execution. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Experimentality of C Device Data Interface:** Developers discussed the "experimental" status of the C Device Data Interface, debating whether to finalize it or focus on a new async non-device API for broader adoption. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **IPC gaps for dictionary-encoded extension types:** A technical discussion on how to represent dictionary-encoded extension types in IPC has led to a proposal for adding `KeyValueMetadata` to dictionary encoding tables. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **LargeMap format addition:** A proposal was made to add a `LargeMap` type to the Arrow specification to support 64-bit offsets, bringing the Map format in line with other `Large*` types like `LargeList`. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)
