* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes 259 resolved issues across the project's various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **PyArrow stubs donation:** The `pyarrow-stubs` codebase was formally donated to the project to provide official PEP 484 type annotations, improving the development experience for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **ADBC 23 release and branding:** Version 23 of the ADBC libraries was released with 41 fixes, and a new discussion was initiated regarding a dedicated logo proposal for the subproject. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **Dictionary-encoded extension types:** Discussion on refining the IPC format to correctly distinguish between dictionary-encoded extension types and extension types that use dictionary-encoding as their storage type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Canonical BigDecimal extension type:** A proposal was introduced for a new canonical extension type to support arbitrary-precision decimals (BigDecimal) with variable scale, facilitating high-fidelity data transport from engines like PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Flight SQL execution path:** A vote was held to add an `is_update` field to Flight SQL prepared statements, allowing servers to indicate whether a statement should follow a query or update execution path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Proposal to add a `LargeMap` type to the Arrow specification to support 64-bit offsets, addressing issues in libraries like Polars that rely on large-offset types for high-volume data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Rust security policy:** The Rust community is establishing a formal security policy for `arrow-rs` and Parquet to define clear guidelines for reporting vulnerabilities and distinguishing them from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)
