* **Arrow 24.0.0 release:** The community released Apache Arrow 24.0.0, resolving over 250 issues and addressing verification edge cases in Gandiva and S3 credential resolution. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **ADBC 23 release:** Apache Arrow ADBC version 23 was released, featuring 41 bug fixes and improvements across C++, Go, Java, Python, and Rust drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Numba support for PyArrow:** Discussion on implementing Numba integration for PyArrow to allow native unwrapping and wrapping of arrays, potentially using a private ABI within PyArrow to simplify development. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Donation of pyarrow-stubs:** The independent `pyarrow-stubs` project is being donated to the Apache Arrow repository to ensure type annotations for Python stay in sync with core library changes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type for variable-scale arbitrary-precision decimals to support data fidelity when moving data from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification using 64-bit offsets to support maps that exceed the 2GB limit of standard 32-bit offset Map types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **C Device Data Interface experimentality:** Maintainers discussed why the C Device Data Interface remains experimental and debated adding an asynchronous stream interface to handle non-CUDA workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Rust Security Policy:** A proposal to establish a formal security policy for the `arrow-rs` and `parquet` crates to standardize how vulnerabilities are reported and distinguished from regular bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **ADBC Logo proposal:** Discussion and feedback on a proposed new logo for the ADBC subproject to give it a distinct identity within the broader Arrow brand family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **Dictionary-encoded extension types in IPC:** Developers identified a gap in the IPC schema serialization for dictionary-encoded extension types and proposed adding KeyValueMetadata to the DictionaryEncoding table as a solution. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
