* **Arrow 25.0.0 release:** The first release candidate for version 25.0.0 was published with 220 resolved issues; a patch release 25.0.1 is already being discussed for PyArrow performance optimizations for Spark. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 development:** Discussion on migrating the benchmarking dashboard to a Go backend and Svelte frontend to resolve UI timeouts and modernize the infrastructure. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Deprecating Tensor IPC messages:** Proposal to deprecate experimental Tensor and SparseTensor IPC messages in favor of using canonical extension types like FixedShapeTensor within RecordBatches. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **JSON representation of Arrow schemas:** Proposal to define a canonical human-readable JSON format for schemas to improve ergonomics for REST APIs and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **ADBC extension for DuckDB:** Columnar.tech proposed donating their DuckDB ADBC extension to the ASF, sparking a discussion on how to coordinate with existing community-maintained extensions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Variant type implementation:** Coordination on mirroring Rust’s Variant type support in C++ and clarifying the Parquet specification regarding optional value fields in shredded variants. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support) | [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **CI resource reduction:** The project successfully reduced its GitHub Actions runner consumption by approximately 30% to comply with ASF Infrastructure resource usage policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **C Data Interface safety:** Discussion on clarifying the specification to handle null bytes in field names within the C FFI to prevent potential crashes during schema exchange. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **Bug squash initiative:** A call for community volunteers to help triage, reproduce, and close stale issues across the project's various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)

* **Rust Object Store releases:** Versions 0.14.0 and 0.14.1 were released, featuring improved integration testing for Azure Storage and Microsoft OneLake. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%20Object%20Store%200.14.0%20RC1)
