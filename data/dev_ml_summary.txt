* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow (originally `arrow-erlang`), which has now cleared IP checks and is being transferred to the official Apache repositories. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new write-side API to support distributed ingestion, allowing systems like Spark or Flink to coordinate parallel writes into databases or table formats like Iceberg with atomic commit/abort semantics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical Bounded Range extension type:** Discussion on adding `arrow.range` as a canonical extension type to represent mathematical intervals, including debates on how to handle per-value closedness flags for PostgreSQL compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Metadata UTF-8 enforcement:** Investigation into enforcing UTF-8 validation for field and schema metadata to ensure compatibility with Flatbuffers and improve interoperability across different language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field/schema/custom%20metadata%20restriction%20to%20UTF8%3F)

* **Parquet Bloom Filter SIMD optimizations:** Proposal and PR to add AVX2 and NEON SIMD optimizations for Split Block Bloom Filter (SBBF) probes in C++, yielding up to 5x performance improvements for in-cache lookups. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet/bloom_filter.cc)

* **IBM Db2 support via ADBC:** Engineers from IBM proposed adding Arrow support for Db2, starting with a new ADBC driver to be hosted under the community-led `adbc-drivers` organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Flight SQL dialect metadata:** Proposal to add new `SqlInfo` codes to describe backend-specific SQL dialect features, such as `LIMIT/OFFSET` grammar and boolean literal support, to help clients generate correct SQL for pushdown operations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **Automated GitHub Copilot reviews:** The project is testing automated GitHub Copilot PR reviews to reduce the burden on human maintainers and catch trivial issues early in the contribution process. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **Rust implementation security patches:** Coordination and release of multiple patch versions (58.3.0, 57.3.1, 56.2.1) to address potential offset overflow issues in the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Planned%20patch%20releases%20this%20week)
