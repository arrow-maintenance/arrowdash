* **Numba support for PyArrow:** Discussion on integrating Numba JIT support directly into PyArrow to enable high-performance computations on Arrow data while avoiding stable ABI issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Arrow Java JDK 17 minimum:** The community reached lazy consensus to set JDK 17 as the minimum requirement for the upcoming Arrow Java 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Arrow 24.0.0 release:** The project approved the release of Apache Arrow 24.0.0, which resolves over 250 GitHub issues across the monorepo. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** Members voted to accept the donation of the `pyarrow-stubs` codebase to officially provide and maintain PEP 484 type annotations within the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **ADBC 23 release:** Apache Arrow ADBC version 23 was released, including significant updates to the C, C++, Go, Java, Python, and Rust implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Canonical BigDecimal extension type:** Proposal for a new extension type to support arbitrary-precision decimals with variable scale, improving compatibility with database engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC dictionary-encoded extension types:** Discussion on addressing an IPC serialization limitation where it is currently impossible to distinguish between a dictionary with extension values and an extension type using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **LargeMap format addition:** Proposal to add a `LargeMap` type (using 64-bit offsets) to the Arrow specification to ensure consistency with existing `LargeList` and `LargeString` types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Flight SQL execution path signaling:** Proposal and vote to add an `is_update` field to prepared statement results to help clients determine whether to use the query or update execution path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)
