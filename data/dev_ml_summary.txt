* **IPC file and stream equivalence:** Discussion on whether IPC files should be guaranteed to be readable as valid streams, following findings that Schema headers in the file footer may occasionally differ from the stream head. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Equivalence%20of%20IPC%20file%20and%20stream%20formats)

* **Flight SQL prepared statement paths:** Proposal to add an explicit `is_update` boolean to prepared statement results to help clients correctly choose between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Introduction of vgi-rpc:** A new Arrow-native RPC framework for Python was presented, offering typed proxies and shared-memory transports without gRPC or Protobuf dependencies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Introducing%20vgi-rpc%20%E2%80%94%20typed%20Arrow%20RPC%20for%20Python%20%28and%20soon%20other%20languages%29)

* **TIME support for 24:00:00:** Debate over extending the `TIME` type to support `24:00:00` for better database compatibility, possibly through a new `TIME_WITH_LEAP_SECOND` type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:TIME%20support%20for%2024%2000%2000)

* **Arrow Java 19.0.0 and JDK 17 requirement:** Following the release of Arrow Java 19.0.0, the community is discussing moving the minimum requirement to JDK 17 for version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **RecordBatch custom_metadata coverage:** Investigation into a testing gap for `custom_metadata` on `RecordBatch` messages in the IPC format, leading to new PRs across the Rust, Go, and JS implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Missing%20test%20coverage%20for%20RecordBatch%20based%20custom_metadata%20in%20IPC%20Test%20Files)

* **Map type field name consistency:** Discussion on why certain implementations ignore custom field names for Map types ("key", "value", "entries") while others preserve them. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type%27s%20key%2Fitem%2Fvalue%20field%20names)

* **ADBC driver migration:** An update on the ongoing effort to move BigQuery, Databricks, and Snowflake drivers from the main ADBC repository to a dedicated community organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)
