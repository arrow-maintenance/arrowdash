* **Apache Arrow 24.0.0 Release:** The community released version 24.0.0, a major update featuring over 250 resolved issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Numba JIT support for PyArrow:** Discussion on integrating Numba support into PyArrow to enable high-performance scientific calculations directly on Arrow data while navigating C++ ABI stability challenges. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Donation of pyarrow-stubs:** The project voted to accept the donation of the `pyarrow-stubs` codebase, bringing formal PEP 484 type annotations into the core repository for better developer tooling support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **C Device Data Interface experimentality:** Developers debated whether the C Device Data Interface should remain marked as experimental and explored the potential for an asynchronous version targeting CPU-only workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **BigDecimal Extension Type proposal:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scales, facilitating high-fidelity data transfer from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Flight SQL execution path signaling:** Discussion and vote to add an `is_update` field to the Flight SQL protocol, allowing servers to explicitly signal whether a prepared statement should follow a query or update execution path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Proposal to add `LargeMap` to the Arrow specification to support 64-bit offsets, addressing scalability issues encountered when maps exceed 32-bit offset limits. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC dictionary-encoded extension types:** Discovery of a limitation in the IPC schema serialization that prevents distinguishing between dictionaries with extension values and extension types with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Rust Security Policy:** Proposed new guidelines for handling security vulnerabilities and bug reporting specifically for the `arrow-rs` and `parquet-rs` implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **ADBC 23 Release:** The Arrow Database Connectivity (ADBC) subproject released version 23, including significant updates to the C#, Java, and Rust libraries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)
