* **Apache Arrow 24.0.0 Release:** The community released version 24.0.0, resolving over 250 GitHub issues across the core libraries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Discussion on providing native Numba support within PyArrow to facilitate efficient code generation and unwrapping of Arrow arrays into native arrays. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs codebase to improve type annotation support and integration within the main Arrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type to represent arbitrary-precision decimals with variable scales, targeting better compatibility with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **ADBC 23 release:** The ADBC subproject announced the release of version 23, featuring 41 bug fixes and improvements across its multi-language drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **LargeMap format addition:** Discussion regarding the addition of a LargeMap type to the Arrow specification to support 64-bit offsets, preventing casting issues in libraries that rely on Large types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC representation of dictionary-encoded extensions:** Proposal to update the IPC schema serialization to properly distinguish between dictionary-encoded extension types and extension types using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL execution path indicators:** A proposal and vote to add an `is_update` field to the Flight SQL protocol to help clients determine whether to use query or update execution paths for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **ADBC logo proposal:** A new visual identity and logo were proposed for the ADBC subproject to differentiate it while maintaining its identity within the Arrow family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
