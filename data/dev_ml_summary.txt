* **Arrow 24.0.0 release:** The community approved the release of Apache Arrow 24.0.0, which includes over 250 resolved issues across the project's various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Discussion on integrating Numba JIT support directly into PyArrow to enable high-performance scientific calculations on Arrow data while managing C++ ABI stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **JDK 17 minimum for Arrow Java:** Lazy consensus was reached to move the minimum requirement for the Arrow Java implementation to JDK 17 starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs codebase, bringing official PEP 484 type annotations for PyArrow into the main Arrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **ADBC 23 release:** The Arrow Database Connectivity (ADBC) subproject released version 23, including 41 bug fixes and improvements for C, C++, Go, Java, and Python drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **BigDecimal canonical extension type:** A proposal was introduced to add a canonical extension type for BigDecimal to support arbitrary-precision decimals with variable scale per row, improving compatibility with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Arrow C Device Data Interface experimentality:** Contributors discussed the long-standing experimental status of the Device Data Interface and the potential for developing an async version of the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL execution path improvements:** Proposal to add an `is_update` field to prepared statement results to help Flight SQL clients correctly choose between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Dictionary-encoded extension types in IPC:** Developers are exploring ways to better represent dictionary-encoded extension types within the IPC schema serialization to prevent errors during deserialization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, facilitating better integration with libraries that default to large offset types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)
