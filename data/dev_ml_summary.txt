* **Numba support for PyArrow:** Discussion on integrating a Numba JIT compiler layer into PyArrow to enable high-performance calculations on Arrow data while managing the maintenance burden and ABI stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Arrow 24.0.0 release:** Voting is underway for the major 24.0.0 release, with current troubleshooting focused on Gandiva test failures related to specific timezone configurations on macOS. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** The community approved the donation of the pyarrow-stubs codebase to the project, which will integrate PEP 484 type annotations directly into PyArrow for better developer tooling support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Arrow ADBC 23 release:** The Arrow Database Connectivity (ADBC) subproject released version 23, featuring 41 bug fixes and improvements across its C++, Go, Java, and Python implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **C Device Data Interface stability:** Developers debated whether to remove the "experimental" tag from the C Device Data Interface, concluding that more non-CUDA implementation experience is needed first. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **LargeMap format proposal:** Proposal to add a `LargeMap` type to the official Arrow specification to allow for 64-bit offsets, preventing overflow issues when encoding large datasets in engines like Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC dictionary-encoded extensions:** Investigation into a limitation in the IPC schema serialization that makes it difficult to distinguish between dictionary-encoded extension types and extensions using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **JDK 17 for Arrow Java:** Consensus reached to raise the minimum requirement to JDK 17 for the Arrow Java 20.0.0 release to leverage modern Java features and simplify maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL protocol improvements:** Proposal and vote to add an `is_update` field to `ActionCreatePreparedStatementResult` to help clients correctly choose between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)
