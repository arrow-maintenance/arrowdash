* **Numba support for PyArrow:** Discussion on integrating Numba JIT compiler support into PyArrow to enable fast computation on Arrow data while managing ABI stability and maintenance concerns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs codebase to provide official PEP 484 type annotations for PyArrow. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Arrow 24.0.0 release candidate:** Voting has commenced for version 24.0.0, which includes over 250 resolved issues across the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Flight SQL protocol enhancements:** Proposals to add an is_update field to prepared statements to allow servers to indicate the correct execution path to clients. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **C Device Data Interface stability:** Discussion on whether the C Device Data Interface should remain marked as experimental and how to approach async support for non-CUDA devices. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **LargeMap format addition:** A proposal to add a LargeMap type to the specification to support 64-bit offsets, enabling the processing of maps larger than 2GB. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Dictionary-encoded extension types in IPC:** Investigation into IPC serialization limitations that prevent distinguishing between dictionary-encoded extension types and extension types with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **ADBC 23 Release:** The ADBC subproject released version 23, including driver updates for BigQuery, Snowflake, and Databricks. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **JDK 17 requirement for Java:** Consensus was reached to set JDK 17 as the minimum requirement for Arrow Java starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)
