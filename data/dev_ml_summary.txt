* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which resolves over 250 GitHub issues across the project's various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Donation of pyarrow-stubs:** The project voted to accept the donation of the independently maintained `pyarrow-stubs` codebase to provide official PEP 484 type annotations for PyArrow users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal Extension Type:** A proposal was introduced for a new canonical extension type to support arbitrary-precision decimals (variable scale) to improve interoperability with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **ADBC 23 release:** The Arrow Database Connectivity (ADBC) subproject released version 23, featuring 41 bug fixes and improvements across its C, Go, Java, Python, and Rust drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **Flight SQL prepared statement execution path:** Discussion and voting occurred on adding an `is_update` boolean to the `ActionCreatePreparedStatementResult` to help clients choose the correct network flow for executing prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Experimentality of C Device Data Interface:** Developers discussed the stability of the C Device Data Interface and considered whether to introduce a CPU-based async stream interface to validate the design for non-CUDA workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **LargeMap format addition:** A proposal was made to add a `LargeMap` type to the Arrow specification using 64-bit offsets, supporting use cases where map data exceeds the 32-bit offset capacity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Security Policy for Rust implementations:** The community is reviewing a proposed security policy for `arrow-rs` and `parquet-rs` to establish clear guidelines for reporting and handling vulnerabilities in the Rust ecosystem. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **ADBC Logo Proposal:** A new logo design for the ADBC subproject was proposed to provide a distinct identity while maintaining visual consistency with the core Apache Arrow branding. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
