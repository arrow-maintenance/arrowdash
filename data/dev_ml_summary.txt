* **Donation of Erlang implementation:** The community voted to accept the donation of a new Erlang implementation for Apache Arrow, which currently leverages bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new API to support distributed write workloads, allowing multiple workers to write data partitions in parallel followed by a single atomic commit by a coordinator. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **New `arrow.range` canonical extension type:** Discussion on adding a standardized extension type to represent bounded mathematical intervals, such as numeric ranges or timestamp periods, to improve interoperability across libraries like Pandas and Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **SIMD optimizations for Parquet Bloom filters:** Implementation of AVX2 and branchless autovectorized probes for Split Block Bloom Filters in C++, providing significant performance improvements for large-scale data scans. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Flight SQL protocol updates:** Proposals to add an `is_update` flag to prepared statements and include `COLUMN_DEF` metadata to better support DML/DDL operations and default column values. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IBM Db2 support for Arrow:** Engineers from IBM are initiating work on an ADBC driver and Arrow Flight support for IBM Db2 to enable high-performance, columnar data exchange. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Automated GitHub Copilot PR reviews:** Exploration of using GitHub Copilot to provide automated preliminary reviews to help alleviate limited human review resources and catch trivial issues earlier in the PR lifecycle. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **UTF8 restrictions for IPC metadata:** Discussion on whether the C++ IPC writer should strictly enforce UTF8 validation for field and schema metadata to ensure compatibility with Flatbuffers string requirements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Dialect-related SqlInfo codes for Flight SQL:** Proposal to add new metadata codes (such as `LIMIT/OFFSET` grammar and `NULLS` ordering support) to help clients generate backend-specific SQL for pushdown operations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)
