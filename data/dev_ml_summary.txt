* **Numba support for PyArrow:** Discussion on integrating Numba support into PyArrow to enable JIT compilation for Arrow data, weighing the benefits of deep C++ integration against maintenance overhead. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Arrow Java 19.0.0 and JDK 17 bump:** Following the successful release of Arrow Java 19.0.0, the community reached consensus to raise the minimum required JDK to version 17 for the upcoming 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL execution paths:** Proposal to consolidate execution modes for Prepared Statements to better support both queries and updates in a single function call across different drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **C Device Data Interface experimentality:** Contributors discussed why the C Device Data Interface remains marked as experimental three years after introduction, citing low adoption outside of CUDA/C++ environments. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **ADBC driver migration:** Progress update on moving BigQuery, Databricks, and Snowflake drivers from the main ADBC repository to the dedicated `adbc-drivers` GitHub organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Map type field name consistency:** Investigation into whether implementations should strictly use "key", "value", and "entries" as field names for Map types to improve cross-language compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type's%20key%2Fitem%2Fvalue%20field%20names)

* **LLMs for project maintenance:** Exploratory discussion on using LLMs to automate aspects of repo management, such as automatically applying component labels to unlabelled GitHub issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)

* **Rust and Object Store releases:** The community approved and released version 58.1.0 of the Arrow Rust implementation and version 0.13.2 of the Rust Object Store. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1)
