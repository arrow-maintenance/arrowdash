* **Canonical BigDecimal extension type:** Proposal for a new extension type to represent arbitrary-precision decimals with variable scales, facilitating high-fidelity data transport from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes over 250 resolved issues across various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Rust security policy:** Discussion on establishing a formal security policy for the Rust and Parquet implementations to better handle the increase in security reporting for open-source projects. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Erlang implementation donation:** The community voted to accept the donation of a new Erlang implementation of Apache Arrow, which is currently built using Rust bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Flight SQL is_update field:** Proposal to add an `is_update` field to prepared statement results to help clients determine whether to use `Execute` or `ExecuteUpdate` endpoints without using heuristics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **ADBC support for IBM Db2:** IBM engineers proposed adding Arrow support for Db2 via a new ADBC driver, which will be hosted in the community-maintained `adbc-drivers` organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **SIMD optimizations for Parquet Bloom Filters:** Technical discussion on porting AVX2-optimized Split Block Bloom Filter (SBBF) probes to C++ using `xsimd` to improve performance for readers like PyArrow and ClickHouse. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new API to support parallel, distributed bulk writes to databases and table formats like Iceberg, featuring a coordinator-worker model for atomic commits. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Metadata UTF8 enforcement:** Debate over whether to strictly enforce UTF8 encoding for field and schema metadata to ensure compatibility with Flatbuffers, following reports of some implementations allowing arbitrary binary data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8)

* **Dialect-related SqlInfo codes:** Proposal to add new Flight SQL codes for describing backend dialect features like LIMIT/OFFSET grammar and NULL ordering to assist with SQL pushdown operations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)
