* **Numba support for PyArrow:** Discussion on integrating a Numba-specific layer into PyArrow to enable JIT-compiled scientific calculations directly on Arrow data. The community is weighing the benefits of direct C++ API access against the maintenance burden of a new specialized integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Arrow Java 19.0.0 release and JDK 17 transition:** Apache Arrow Java 19.0.0 has been released, and a consensus was reached to raise the minimum requirement to JDK 17 for version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEJava%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1) | [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL Prepared Statement execution paths:** Proposal to add a boolean field to the Flight SQL protocol to help clients explicitly distinguish between query and update execution paths. This change aims to improve DML support across JDBC, ODBC, and ADBC driver implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Map type field naming:** Technical discussion regarding whether implementations should strictly enforce "entries", "key", and "value" as field names for Map types. It was clarified that the specification allows flexibility to maintain compatibility with different Parquet naming conventions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type%27s%20key%2Fitem%2Fvalue%20field%20names)

* **ADBC driver organization migration:** Migration is progressing for BigQuery, Databricks, and Snowflake drivers from the main ADBC repository to a dedicated "adbc-drivers" organization. This move is intended to allow for independent release cycles and specialized maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **C Device Data Interface experimentality:** Investigation into why the C Device Data Interface remains marked as experimental three years after introduction. Maintainers noted that further verification across a wider variety of non-CUDA hardware is needed before stabilizing the API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Arrow Rust 58.1.0 release:** The Rust implementation has released version 58.1.0, alongside Object Store 0.13.2, including various performance and maintenance updates. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1)

* **LLMs for project maintenance:** Exploration of using Large Language Models to automate routine repository management tasks. Initial experiments showed success in using LLMs to automatically assign component labels to unlabeled issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)
