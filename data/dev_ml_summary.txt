* **Numba support for PyArrow:** Discussion on a proposal to add a Numba integration layer to PyArrow, allowing compiled Python functions to operate directly on Arrow data with competitive performance. The proposed implementation may involve exposing a Numba-specific ABI within PyArrow to maintain compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Java 19.0.0 release and JDK 17 transition:** Apache Arrow Java 19.0.0 has been released, marking the first release from its separate repository. The community also reached consensus to set JDK 17 as the minimum required version starting with the Arrow Java 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Java%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1) | [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL prepared statement execution paths:** Proposal to add a field to the Flight SQL protocol that indicates the expected network flow for prepared statements, resolving ambiguity in how drivers choose between query and update endpoints. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Map type field name consistency:** Investigation into whether specific child field names like "key", "value", and "entries" should be strictly enforced for Map types. It was clarified that while the specification suggests these names, strict enforcement is avoided to maintain compatibility with Parquet naming conventions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type%27s%20key%2Fitem%2Fvalue%20field%20names)

* **LLMs for maintenance automation:** Discussion on using Large Language Models to assist with project maintenance, such as automatically applying component labels to untriaged issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)

* **ADBC driver migration:** Progress update on moving BigQuery, Databricks, and Snowflake drivers from the core ADBC repository to a separate community-governed organization to support independent release cycles. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Rust implementation releases:** The community successfully released version 58.1.0 of the core Rust implementation and version 0.13.2 of the Object Store crate, addressing various maintenance fixes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1) | [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%20Object%20Store%200.13.2%20RC2)
