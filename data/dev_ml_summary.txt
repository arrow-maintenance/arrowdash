* **Arrow 24.0.0 release candidate:** The community voted on RC0 for version 24.0.0, which resolves 258 issues; the discussion touched on Gandiva test failures related to specific timezones on macOS. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Proposal to integrate a Numba JIT compiler layer into PyArrow to enable high-performance calculations on Arrow data while managing C++ ABI stability concerns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Donation of pyarrow-stubs:** The community approved the donation of the `pyarrow-stubs` project to the main Arrow repository to ensure PEP 484 type annotations are officially maintained. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **JDK 17 minimum for Java 20.0.0:** Developers reached a consensus to bump the minimum requirement for the Arrow Java implementation to JDK 17 to modernize the codebase and support newer features. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Canonical BigDecimal extension type:** A proposal was introduced for a new extension type to support arbitrary-precision decimals with variable scale per row, improving interoperability with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC representation of dictionary-encoded extensions:** Investigation into a limitation in the IPC format that prevents distinguishing between dictionary-encoded extension types and extension types with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Apache Arrow ADBC 23 release:** The ADBC subproject released version 23, featuring 41 bug fixes and improvements across C, C++, Go, Java, and Python drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **C Device Data Interface experimentality:** Discussion on the roadmap for marking the C Device Data Interface as stable and the potential need for an asynchronous version of the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL execution path updates:** Proposal to add an `is_update` field to `ActionCreatePreparedStatementResult` to help clients determine the correct network flow for prepared statement execution. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the specification to support 64-bit offsets for map types, mirroring the existing LargeList and LargeBinary types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)
