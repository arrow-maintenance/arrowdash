* **Donation of Erlang implementation:** The community voted to accept a new Erlang implementation of Arrow, which will be moved to the official Apache Arrow Erlang repository for further development. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** A proposal to add parallelized data ingestion to ADBC, allowing distributed writers like Spark or Flink to perform atomic commits across multiple partitions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **UTF8 metadata restrictions:** Discussion on whether to strictly enforce UTF8 encoding for field and schema metadata in the C++ and IPC implementations to ensure cross-language interoperability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Parquet Bloom Filter SIMD optimization:** Performance improvement proposal for Parquet Bloom Filter probes using xsimd or autovectorization, showing significant speedups for in-cache and out-of-cache lookups. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Canonical extension type for bounded ranges:** Proposal to add `arrow.range` as a canonical extension type to represent mathematical intervals (e.g., `[lower, upper)`), improving interoperability with systems like pandas and PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **IBM Db2 integration with Arrow:** IBM engineers are proposing and implementing a new ADBC driver and exploring Flight SQL support for IBM Db2 to enable faster data movement. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Flight SQL execution path indicator:** The community voted to add an `is_update` field to `ActionCreatePreparedStatementResult` to help drivers correctly route queries between execution and update paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **New dialect-related SqlInfo codes:** Proposal to add specific metadata codes for SQL dialects (like limit/offset grammar and null ordering) to help engines like Spark optimize pushdown operations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **Automated GitHub Copilot reviews:** Trial of an automated workflow that uses GitHub Copilot to provide initial feedback on pull requests, aimed at reducing the manual review burden for maintainers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)
