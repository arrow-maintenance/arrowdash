* **Arrow 24.0.0 release:** The community released version 24.0.0 with 259 resolved issues, following discussions regarding timezone-related test failures discovered in Gandiva on newer macOS versions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **BigDecimal extension type proposal:** A new canonical extension type is proposed for arbitrary-precision decimals to support variable scale per row, facilitating high-fidelity transport from databases like PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Rust security policy:** Discussions are underway to establish a formal security policy for `arrow-rs` and Parquet in Rust to better manage vulnerability reporting and bug classification. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Erlang implementation donation:** The community voted to accept the donation of a new Erlang implementation of Arrow, which currently utilizes bindings to the Rust library. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Metadata UTF-8 validation:** Investigation into whether field and schema metadata should be strictly restricted to valid UTF-8 to ensure compliance with Flatbuffers specifications and IPC standards. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **AVX2 Bloom Filter acceleration:** A proposal to implement AVX2-accelerated Split Block Bloom Filter (SBBF) probes in C++ using xsimd aims to significantly improve Parquet read performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Flight SQL dialect metadata:** New SqlInfo codes are proposed to help Flight SQL clients handle backend-specific dialect differences such as LIMIT/OFFSET syntax and NULL ordering. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **ADBC Partitioned Bulk Ingest:** Proposal for a new ADBC API to support parallel data ingestion, enabling distributed writers to perform atomic commits across multiple partitions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **IBM Db2 support for Arrow:** IBM proposed adding Arrow support for Db2, including the development of an ADBC driver and potential integration with Arrow Flight for high-performance transport. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSADBCDB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **ADBC logo design:** The community is reviewing a proposed logo for the ADBC subproject, focusing on visual consistency with the Arrow brand and compliance with Apache trademark guidelines. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
