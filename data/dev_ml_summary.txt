* **Apache Arrow 25.0.0 release:** The community approved the 25.0.0 release, which includes over 200 resolved issues; a follow-up patch release is planned to address performance regressions in PyArrow's `to_pylist` method. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 modernization:** Wes McKinney presented a major overhaul of the Conbench benchmarking tool, moving to a Go backend and Svelte/Vite frontend to improve performance and simplify deployment. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Deprecating IPC Tensor messages:** Proposal to deprecate experimental Tensor and SparseTensor IPC messages in favor of using canonical extension types, which would reduce the project's security surface area and maintenance burden. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **JSON representation for Arrow schemas:** Discussion on establishing a canonical human-readable JSON format for schemas to improve ergonomics for REST APIs and ADBC metadata exchange. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **ADBC Extension for DuckDB:** Columnar proposed donating a new hand-coded ADBC extension for DuckDB to the ASF, sparking a debate regarding coordination with existing AI-assisted community extensions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Variant type support in C++:** Following the Rust implementation, developers are discussing the technical requirements for adding Variant type support to the C++ and Parquet implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **Spec clarification on NUL bytes in field names:** Investigation into how the C Data Interface handles NUL bytes in UTF-8 field names, with a consensus to error out on the producer side to avoid reader panics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **CI resource usage reduction:** The project successfully reduced its weekly GitHub Actions consumption by approximately 27% to align with updated ASF infrastructure resource policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Rust Object Store releases:** The community approved and released versions 0.14.0 and 0.14.1 of the Arrow Rust Object Store implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%2BRUST%20Release%20Apache%20Arrow%20Rust%20Object%20Store%200.14.0%20RC1)

* **Call for issue triaging:** Maintainers are requesting community help to reproduce old bug reports and close stale issues to manage the project's growing GitHub issue backlog. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
