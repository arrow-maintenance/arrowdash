* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which is currently built on top of the Rust implementation's bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **ADBC partitioned bulk ingest API:** A new proposal aims to add a partitioned write API to ADBC to support atomic commits from distributed writers like Spark and Flink. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical range extension type:** Discussion is underway regarding a new `arrow.range` extension type to represent bounded mathematical intervals, designed to improve interoperability with Pandas and PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **SIMD optimization for Parquet Bloom filters:** A proposal was submitted to implement AVX2-accelerated Split-Block Bloom Filter probes in C++, utilizing xsimd and branchless autovectorization for significant performance gains. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **GitHub Copilot automated reviews:** The project is testing automated GitHub Copilot PR reviews to identify trivial issues early, though maintainers are refining how to handle draft PRs and contributor expectations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **Conbench benchmarking infrastructure:** Following the migration of benchmarking data and servers, developers are planning a "v2" redesign of Conbench to improve sustainability and performance tracking. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Arrow Rust 59.0.0 release:** Voting has commenced for version 59.0.0 of the Rust implementation, following the release of several maintenance versions earlier this month. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)

* **Flight SQL dialect metadata:** New `SqlInfo` codes have been proposed to allow Flight SQL drivers to self-describe dialect-specific behavior for limit/offset, null ordering, and literals. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)
