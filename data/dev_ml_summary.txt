* **Erlang implementation donation:** The community voted to accept the donation of a new Erlang implementation for Apache Arrow, which is currently based on Rust bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **IBM Db2 support:** Engineers from IBM proposed adding Apache Arrow support for Db2 via a new ADBC driver, aiming to improve data movement performance for analytical workloads. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Parquet SIMD optimization:** Discussion on implementing AVX2 and NEON SIMD probes for Split Block Bloom Filters (SBBF) in the C++ Parquet reader, which shows significant performance gains for in-cache lookups. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Canonical range extension type:** Proposal to introduce `arrow.range` as a standard extension type for representing bounded mathematical intervals (e.g., numeric or temporal ranges) to improve interoperability between libraries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **ADBC Partitioned Bulk Ingest:** A new API proposal for partitioned bulk ingestion designed to support distributed writers like Spark and Flink when targeting RDBMS or table formats like Iceberg. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **UTF-8 metadata enforcement:** Discussion on whether to strictly enforce UTF-8 encoding for field and schema metadata to ensure compatibility with the Flatbuffers specification and other language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Flight SQL dialect metadata:** Proposal to add new SqlInfo codes for dialect-specific features like LIMIT/OFFSET grammar and NULLS ordering to help clients generate backend-specific SQL more effectively. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **Automated AI code reviews:** The project has enabled automated GitHub Copilot reviews for pull requests to help identify trivial problems and reduce the burden on human maintainers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)
