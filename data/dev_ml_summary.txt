* **Erlang implementation donation:** The community approved the donation of an Erlang implementation of Apache Arrow, which has been transferred to the official organization repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **Parquet SIMD optimizations:** Discussion and implementation of AVX2 optimizations and branchless autovectorization for Parquet Split Block Bloom Filters to improve read-path performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Flight SQL dialect metadata:** Proposal to add new SqlInfo codes to Flight SQL for describing backend-specific dialect details like LIMIT/OFFSET syntax and NULL ordering to assist client-side SQL generation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **ADBC Partitioned Bulk Ingest API:** A new proposal for a partitioned write API in ADBC to enable distributed systems like Spark and Flink to perform parallel ingest with atomic commit/abort semantics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical range extension type:** Discussion on adding `arrow.range` as a canonical extension type for bounded intervals, with debate over handling uniform vs. per-value inclusive/exclusive bounds. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Automated GitHub Copilot reviews:** The project is trialing automated PR reviews using GitHub Copilot to provide initial feedback on trivial issues and help manage limited human review resources. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **Conbench benchmarking status:** Discussion regarding the migration of benchmarking infrastructure and a potential rewrite of the Conbench backend in Go to improve stability and maintainability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Arrow Rust 59.0.0 release:** The community successfully voted on and released version 59.0.0 of the Rust implementation, which is now available on crates.io. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)

* **Variant type coordination:** Multiple contributors are coordinating efforts to implement a Variant data type in Arrow C++ to support Iceberg V3 requirements and ensure cross-implementation compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)
