* **Arrow 25.0.0 release:** The community approved the version 25.0.0 release, covering over 220 resolved issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 infrastructure proposal:** Wes McKinney proposed a major overhaul of the benchmarking infrastructure using Go and Svelte to improve the performance and maintainability of the results dashboard. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20+%20conbench%20OSS%20project)

* **JSON representation of Arrow schemas:** Discussion is underway to define a canonical JSON format for schemas to assist with ADBC metadata APIs and other human-readable persistence use cases. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating IPC Tensor messages:** A proposal suggests deprecating specialized IPC messages for Tensors and SparseTensors in favor of using canonical extension types within standard RecordBatches. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages)

* **DuckDB ADBC extension donation:** Columnar-tech expressed interest in donating a hand-coded ADBC client extension for DuckDB to the Arrow project to serve as an official community-governed tool. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **CI resource optimization:** To comply with ASF infrastructure policies, the project successfully reduced its GitHub Actions runner consumption by approximately 27% through targeted workflow improvements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Variant type specification alignment:** Developers are working to align the Parquet Variant encoding with Arrow’s canonical extension by allowing the omission of the `value` field when shredded columns are present. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **Bug triaging initiative:** A "bug squash" effort is seeking volunteers to reproduce and validate older bug reports against recent Arrow versions to help clean up the project's issue backlog. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
