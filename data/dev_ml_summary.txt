* **Arrow Java 19.0.0 release:** The community released version 19.0.0 of the Java implementation, which was the final version to be compiled with JDK 11 support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Java%2019.0.0%20RC1)

* **JDK 17 minimum for Arrow Java 20.0.0:** Following the 19.0.0 release, the community reached a lazy consensus to upgrade the minimum requirement to JDK 17 for all future Java development starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Proposal to integrate a Numba extension layer directly into PyArrow to enable JIT-compiled scientific calculations on Arrow data while avoiding C++ ABI stability issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **ADBC 23 release:** Apache Arrow ADBC version 23 has been released, including 41 resolved issues and updates across C/C++, Go, Python, Java, C#, Rust, and R components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Flight SQL prepared statement execution path:** Discussion and vote on adding an `is_update` boolean field to `ActionCreatePreparedStatementResult` to help clients determine whether to use query or update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Donation of pyarrow-stubs:** The project is preparing to accept the donation of the `pyarrow-stubs` codebase to provide official PEP 484 type annotations and improve the developer experience for PyArrow users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Stability of C Device Data Interface:** Developers discussed the experimental status of the C Device Data Interface and explored the potential for an asynchronous version of the non-device API to support broader use cases. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **ADBC driver repository migration:** Progress update on moving BigQuery, Databricks, and Snowflake drivers from the core ADBC repository to a dedicated `adbc-drivers` organization for independent maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **LargeMap format addition:** A proposal to extend the Arrow specification with a `LargeMap` type that utilizes 64-bit offsets, addressing compatibility issues with libraries like Polars that default to large types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)
