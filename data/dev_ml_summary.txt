* **Erlang implementation donation:** The community voted to accept the donation of a new Erlang implementation of Apache Arrow, which is currently based on Rust bindings and will be moved to a dedicated repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **IBM Db2 ADBC driver proposal:** Discussion on adding a native ADBC driver for IBM Db2 to enable high-performance data exchange; development will proceed under the community-maintained adbc-drivers organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSADBCDB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Parquet Bloom filter optimizations:** A new implementation using SIMD (AVX2/NEON) and branchless autovectorization has been introduced to significantly improve performance for Bloom filter probes in the C++ implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** Proposal to add a new API for parallel data ingestion, allowing distributed writers like Spark to coordinate multi-connection writes with atomic commit and rollback capabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Enforcing UTF-8 for metadata:** Discussion regarding whether to strictly validate that field and schema metadata are valid UTF-8, as the underlying Flatbuffers serialization technically requires it. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **New arrow.range extension type:** Proposal for a canonical extension type to represent mathematical or temporal intervals (e.g., `[lower, upper)`), improving interoperability between Arrow, Pandas, and databases like PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Flight SQL dialect metadata:** Addition of new SqlInfo codes to the Flight SQL protocol to help clients identify backend-specific SQL grammar, such as LIMIT/OFFSET syntax and NULLS ordering support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Rust implementation patch releases:** Several patch versions (58.3.0, 57.3.1, and 56.2.1) were released for the Rust implementation to backport critical fixes for potential offset overflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%2058.3.0%20RC1)

* **Automated GitHub Copilot reviews:** Discussion on trialing automated AI-generated code reviews for the main repository to assist with the high volume of incoming pull requests and identify trivial issues faster. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)
