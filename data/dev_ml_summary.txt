* **StructArray validity clarification:** Discussion on clarifying whether non-nullable child fields within a nullable StructArray must strictly exclude nulls or can rely on the parent's validity map for semantic equality. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:add%20to%20the%20specification%20a%20clarification%20of%20validity%20buffer%20for%20non%20nullable%20field%20of%20nullable%20StructArray)

* **IPC ApplicationData message type:** Proposal to introduce a new IPC message type for carrying arbitrary application-specific bytes, avoiding the overhead and semantic mismatch of using zero-row RecordBatches for metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20MessageType%20-%20OpaqueBytes)

* **Release provenance and security:** Debate over the stability of GitHub-generated source archives versus canonical ASF artifacts, leading to a requirement to include SHA hashes directly in release vote emails. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:SHA512%20mismatch%20for%20apache-arrow-adbc-21.tar.gz)

* **Apache Arrow 10th Anniversary:** The community celebrated the 10-year anniversary of the project's official establishment and its first git commit in February 2016. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20is%2010%20years%20old%20%F0%9F%8E%89)

* **Equivalence of IPC file and stream formats:** Discussion on whether to explicitly require that IPC files remain valid when read as streams (after skipping the header) to simplify reader implementations across different languages. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Equivalence%20of%20IPC%20file%20and%20stream%20formats)

* **Time type support for 24:00:00:** Re-evaluating the inclusion of "24:00:00" in Time types to maintain data fidelity when inter-operating with databases like PostgreSQL and DuckDB that support this value. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:TIME%20support%20for%20240000)

* **Flight SQL execution path signaling:** Proposal to add an `is_update` boolean to prepared statement results to help clients determine whether to use the query or update execution path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Rust implementation major release:** The Arrow Rust (arrow-rs) community released version 58.0.0, which includes several breaking API changes and performance enhancements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.0.0%20RC1)
