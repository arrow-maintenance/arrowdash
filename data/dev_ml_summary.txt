* **Arrow Java 19.0.0 and JDK 17 upgrade:** Apache Arrow Java 19.0.0 has been released, and the community reached a consensus to set JDK 17 as the minimum requirement starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Java%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1) [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Discussion on integrating Numba support into PyArrow to enable JIT-compiled scientific calculations on Arrow data while addressing concerns regarding C++ ABI stability and maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **ADBC 23 release and driver migration:** The community is voting on the ADBC 23 release and continuing the migration of BigQuery, Databricks, and Snowflake drivers to a dedicated community-led organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1) [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Flight SQL protocol improvements:** Proposal to add a boolean field to the Flight SQL protocol to explicitly indicate whether a prepared statement follows a query or update execution path, improving driver compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Stability of C Device Data Interface:** Developers discussed the experimental status of the C Device Data Interface, noting that wider adoption beyond CUDA and further design validation for non-GPU devices is needed before stabilization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Proposal for LargeMap format:** Discussion initiated to add a LargeMap type to the Arrow specification, allowing maps to use 64-bit offsets to better support large-scale datasets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **LLM-aided maintenance:** Exploration of using Large Language Models to automate repo management tasks, such as automatically adding component labels to unlabelled issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)

* **Arrow Rust and Object Store releases:** The community approved and published new releases for the core Arrow Rust implementation (58.1.0) and the Rust Object Store library (0.13.2). [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1) [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%20Object%20Store%200.13.2%20RC2)
