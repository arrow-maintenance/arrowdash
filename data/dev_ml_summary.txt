* **Arrow 24.0.0 release:** The community released version 24.0.0, including over 250 resolved issues across C++, Java, Python, and other implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Discussion on implementing a Numba integration layer within PyArrow to allow JIT-compiled scientific calculations directly on Arrow data structures with high efficiency. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Canonical BigDecimal extension type:** Proposal for a new extension type to represent arbitrary-precision decimals with variable scales, facilitating better data interchange with systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the pyarrow-stubs project, bringing PEP 484 type stub maintenance directly into the main Arrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL prepared statements update:** Proposal to add an `is_update` field to the `ActionCreatePreparedStatementResult` protocol to help clients correctly route execution between query and update paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the specification using 64-bit offsets to support map structures that exceed the 2GB limit of the standard 32-bit `Map` type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC representation of dictionary-encoded extension types:** Investigation into a specification gap where the IPC schema cannot distinguish between a dictionary-encoded extension and an extension using a dictionary as its storage type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Arrow Java JDK 17 minimum:** Consensus was reached to set JDK 17 as the minimum required version starting with the Arrow Java 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Rust implementation security policy:** Proposal for a formal security policy for `arrow-rs` and `parquet-rs` to provide clear guidelines for reporting and handling security vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Arrow C Device Data Interface status:** Developers discussed whether to stabilize the C Device Data Interface, concluding it should remain experimental until more non-CUDA use cases are implemented. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)
