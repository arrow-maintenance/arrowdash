* **Arrow 24.0.0 release:** The community announced the release of version 24.0.0, resolving over 250 issues across the ecosystem since the last minor release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **ADBC Partitioned Bulk Ingest API:** A new proposal introduces a write-side mirror to the existing partitioned read API, allowing distributed writers like Spark and Flink to perform atomic, multi-connection data ingestion. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical BigDecimal extension type:** Discussion is underway for a new canonical extension type to represent arbitrary-precision decimals with variable scales per row, improving interoperability with systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **SIMD-accelerated Bloom Filter probes:** Developers are porting AVX2/SIMD optimizations for Split Block Bloom Filters from Go to C++ to significantly improve Parquet read performance on supported hardware. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet/bloom_filter.cc)

* **Arrow support for IBM Db2:** Engineers from IBM proposed adding native Arrow support for Db2, starting with a new ADBC driver and exploring future high-performance integration via Arrow Flight SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSADBCDB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Donation of Erlang implementation:** The community is voting on accepting a donation of an Erlang implementation of Apache Arrow, which currently utilizes bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Metadata UTF-8 enforcement:** A discussion highlights potential IPC issues caused by non-UTF8 data in field and schema metadata, leading to a proposal for stricter validation across different language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field/schema/custom%20metadata%20restriction%20to%20UTF8?)

* **Rust implementation security policy:** A formal security policy has been proposed for the Arrow-rs and Parquet-rs crates to provide clear guidelines for reporting and handling vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Flight SQL dialect metadata:** Proposal to add new SqlInfo codes to the Flight SQL protocol that allow servers to report dialect-specific syntax for LIMIT/OFFSET, NULL ordering, and literals to assist client-side query generation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlightSQL%20Add%20dialect-related%20SqlInfo%20codes)
