* **Numba support for PyArrow:** Discussion on integrating a Numba extension into PyArrow to enable high-performance JIT compilation for Arrow data structures while bypassing C++ ABI stability issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **JDK 17 baseline for Java:** Following the release of Arrow Java 19.0.0, the community reached a consensus to raise the minimum requirement to JDK 17 for Arrow Java 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL Prepared Statement improvements:** Proposal to add a field to the protocol indicating whether a prepared statement is a query or an update, improving how drivers handle DML vs. DQL execution. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Map type field naming:** Clarification that Map type child field names ("entries", "key", "value") are recommended by the specification but not strictly enforced, leading to a revert of a C++ change that attempted to mandate them. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type%27s%20key%2Fitem%2Fvalue%20field%20names)

* **ADBC driver migration:** Update on moving BigQuery, Databricks, and Snowflake drivers out of the main ADBC repository and into a dedicated community organization for independent maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Stability of C Device Data Interface:** Discussion on the experimental status of the C Device and Async Stream interfaces, noting that wider non-CUDA adoption is required before they are considered stable. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Rust implementation 58.1.0 release:** The Rust implementation has been updated to version 58.1.0, alongside a release of the object store crate version 0.13.2. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1)

* **LLMs for project maintenance:** Exploration of using LLMs to automate repository management tasks, such as automatically adding component labels to unlabelled GitHub issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)
