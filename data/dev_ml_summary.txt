* **Arrow Erlang donation:** The community approved the donation of a new Erlang implementation of Apache Arrow, which is currently being transferred to the official Apache repositories. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **SIMD optimizations for Parquet Bloom filters:** Discussion and implementation of AVX2 and NEON-accelerated Split Block Bloom Filter (SBBF) probes to improve read performance using xsimd and branchless autovectorization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new API to support distributed writers in parallel, allowing frameworks like Spark and Flink to perform atomic writes across multiple connections. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical extension type for bounded ranges:** Proposal to add `arrow.range` for representing mathematical intervals, with ongoing debate regarding how to handle per-value vs. per-column boundary closedness for PostgreSQL and pandas compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Automated GitHub Copilot reviews:** The project enabled automated AI-driven pull request reviews to assist human maintainers, though developers are refining rules to avoid clutter on draft PRs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **Flight SQL dialect-related metadata:** Proposal to add new SqlInfo codes for features like LIMIT/OFFSET and NULLS ordering to help clients generate backend-specific SQL without hardcoded configuration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Arrow Rust releases:** The Rust implementation completed several maintenance releases (56.2.1, 57.3.1, 58.3.0) and finalized version 59.0.0, addressing issues with release verification scripts. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)

* **Conbench infrastructure redesign:** Following a migration to new hosting, developers discussed a major "v2" redesign of the Conbench benchmarking platform, including a potential backend rewrite in Go. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **JDBC Flight SQL column defaults:** A vote passed to add support for column default values in the Flight SQL JDBC driver, requiring small updates to the table_schema contract in arrow-format. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFLIGHTJDBC)

* **Pyarrow-stubs donation:** The formal donation of the `pyarrow-stubs` library to Apache Arrow was concluded, providing official type hinting support for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)
