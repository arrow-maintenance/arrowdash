* **Arrow 25.0.0 release:** The community approved the version 25.0.0 release and discussed a potential 25.0.1 patch release to fast-track performance optimizations for PyArrow-to-Python conversions requested by the Spark team. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench benchmarking modernization:** A major overhaul of the Conbench benchmarking platform is underway, migrating the backend to Go and the frontend to Svelte/Vite to improve system performance and UI ergonomics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **JSON representation of Arrow schemas:** Discussion on establishing a canonical, human-readable JSON format for Arrow schemas to improve ergonomics in REST API contracts and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating IPC Tensor messages:** Proposal to deprecate dedicated Tensor and SparseTensor messages in the IPC format in favor of using FixedShapeTensor and VariableShapeTensor extension types for better integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **ADBC DuckDB client extension:** Columnar proposed donating a new DuckDB extension that enables connectivity to Snowflake, BigQuery, and other databases via ADBC drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Variant type specification alignment:** Discussion on allowing the "value" field to be optional in the Parquet Variant spec to support better data shredding and maintain consistency with Arrow's implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **CI resource optimization:** The project successfully reduced its weekly GitHub Actions runner consumption by approximately 27% to align with ASF Infrastructure resource policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Arrow Rust implementation release:** Version 59.1.0 of the Arrow Rust implementation was approved and released, addressing several maintenance issues and improving the crate's stability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%2059.1.0%20RC1)

* **Issue squashing initiative:** Maintainers are calling for volunteers to help reproduce old bugs and close stale issues to improve the health and manageability of the Arrow issue tracker. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
