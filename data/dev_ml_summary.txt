* **Arrow Java 19.0.0 release and JDK upgrade:** Following the release of Arrow Java 19.0.0, the community has reached a consensus to move the minimum requirement to JDK 17 for version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Discussion on integrating Numba JIT support into PyArrow to enable faster scientific calculations, focusing on maintenance burdens and ABI stability for external packages. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **ADBC 23 release and driver migration:** The community released ADBC 23 and is progressing with the migration of BigQuery, Databricks, and Snowflake drivers to a dedicated community organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **C Device Data Interface experimentality:** Contributors discussed why the C Device Data Interface remains experimental and debated creating a stable async non-device API to better support CPU-based asynchronous workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Donation of pyarrow-stubs:** A proposal is underway to donate the independently maintained `pyarrow-stubs` codebase to the Apache Arrow project to ensure type annotations stay in sync with PyArrow. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCCUSSPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL protocol enhancements:** A vote is open to add an `is_update` field to the `ActionCreatePreparedStatementResult` message, allowing servers to communicate the correct execution path to clients. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Proposal to add a `LargeMap` type using 64-bit offsets to the Arrow specification to prevent casting issues when working with high-volume data in libraries like Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC serialization of extension types:** Investigation into an issue where dictionary-encoded extension types cannot be correctly represented or round-tripped within the current IPC schema serialization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
