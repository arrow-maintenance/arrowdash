* **GitHub Runner resource optimization:** In response to high usage on shared ASF runners, the community successfully reduced CI build times by approximately 27% to comply with infrastructure policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Conbench V2 performance overhaul:** Proposal to rebuild the project’s benchmarking dashboard using a Go backend and Svelte frontend to address database slowness and improve data storage with DuckDB and Parquet. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **DuckDB ADBC Extension donation:** Columnar proposed donating a hand-coded ADBC client extension for DuckDB to the Arrow project, leading to a discussion on collaboration with existing community extensions and AI-assisted development. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Deprecation of IPC Tensor messages:** Discussion on deprecating experimental Tensor and SparseTensor IPC messages in favor of standard RecordBatches using canonical extension types for better integration with existing APIs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **JSON representation of Arrow schemas:** Proposal for a canonical human-readable JSON format for schemas to facilitate better integration with REST APIs and ADBC metadata boundaries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:JSON%20representation%20of%20Arrow%20schemas)

* **Apache Arrow 25.0.0 release:** The community approved the major 25.0.0 release, which includes over 200 resolved issues and improved conversion performance for Spark-related PyArrow workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **C Data Interface field name clarification:** Technical debate on handling UTF-8 field names that contain nul bytes, which are valid in Arrow but can cause issues when exported via the C FFI to nul-terminated C strings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **Variant type implementation progress:** Updates on the roadmap for implementing the new Variant type in C++ and Parquet, leveraging existing reference implementations from the Rust community. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **Rust and Object Store releases:** Approval and release of Arrow Rust 59.1.0 and Object Store 0.14.x, featuring improved Azure and Microsoft OneLake integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Rust%2059.1.0%20RC1)
