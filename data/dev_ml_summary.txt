* **Numba support for PyArrow:** Discussion on integrating a Numba extension into PyArrow to enable JIT compilation for Arrow data, allowing for high-performance scientific calculations in Python. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **JDK 17 minimum for Arrow Java 20.0.0:** The community reached a consensus to increase the minimum required Java version from JDK 11 to JDK 17 for the upcoming major release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL Prepared Statements execution path:** Proposal to add a boolean field to the protocol that helps clients determine whether to use the query or update network flow for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Status of C Device Data Interface:** Developers discussed whether the C Device Data Interface should remain "experimental" and explored the possibility of adding async support to the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **ADBC driver migration to contrib organization:** Progress update on moving the BigQuery, Databricks, and Snowflake drivers to a dedicated repository and organization to streamline maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **LargeMap format addition:** A new proposal suggests adding a LargeMap type to the Arrow specification to support 64-bit offsets, addressing limitations in libraries that rely on Large* types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Arrow Java 19.0.0 release:** The community approved and released version 19.0.0 of the Java implementation, which marks the final release to support JDK 11. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Java%2019.0.0%20RC1)

* **LLMs for project maintenance:** Exploration of using Large Language Models to automate repo management tasks, such as automatically assigning component labels to unlabelled issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LLMs%20to%20aid%20with%20maintenance)
