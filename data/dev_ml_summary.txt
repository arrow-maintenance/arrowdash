* **PyArrow stubs donation:** Proposal and vote to donate the `pyarrow-stubs` codebase to Apache Arrow, providing officially maintained PEP 484 type annotations for PyArrow. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which resolves 259 issues across the multi-language toolbox. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Canonical BigDecimal extension type:** Proposal for a new extension type to handle arbitrary-precision decimals with variable scale per row, improving compatibility with database engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Numba support for PyArrow:** Discussion on integrating Numba with PyArrow, debating whether to use a private Numba-specific ABI or the Arrow C Stream Interface for native array unwrapping. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, enabling map arrays with more than 2 billion entries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Experimentality of C Device Data Interface:** Debate on the "experimental" status of the C Device Data Interface and whether to introduce a non-device async API to facilitate easier adoption for CPU-only workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Dictionary-encoded extension types in IPC:** Investigation into IPC serialization issues where dictionary-encoded extension types are incorrectly exported, leading to a proposal for adding metadata to the `DictionaryEncoding` table. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Apache Arrow ADBC 23 release:** Announcement of the latest release for the Arrow Database Connectivity (ADBC) subproject, including significant updates to C#, Java, and Rust libraries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **Rust security policy:** Proposal to establish a formal security policy for `arrow-rs` and `parquet-rs` to provide clear guidelines for reporting and handling vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Flight SQL prepared statement execution:** Discussion on adding an `is_update` field to `ActionCreatePreparedStatementResult` to allow servers to indicate if a prepared statement follows a query or update path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)
