* **Donation of Erlang implementation:** The community voted to accept the donation of an Erlang implementation of Arrow, which currently provides bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Canonical range extension type:** Proposal to introduce `arrow.range` as a canonical extension type for mathematical intervals, including discussions on how to handle boundary flags for database compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **SIMD optimizations for Parquet Bloom Filters:** Technical discussion on implementing AVX2 and NEON-optimized probes for Parquet Split-Block Bloom Filters in C++, leveraging branchless autovectorization for significant performance gains. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%20bloom_filter.cc)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new API to support distributed data ingestion, allowing multiple workers to write data partitions in parallel with atomic commit/abort coordination. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **UTF-8 enforcement in metadata:** Discussion on strictly enforcing UTF-8 encoding for field and schema metadata in C++ and Python to ensure compliance with the Flatbuffers specification and avoid IPC errors. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%20schema%20custom%20metadata%20restriction%20to%20UTF8%3F)

* **Flight SQL dialect metadata:** New `SqlInfo` codes were proposed for Flight SQL to help clients identify backend capabilities such as LIMIT/OFFSET syntax and NULL ordering during SQL generation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Automated GitHub Copilot reviews:** The project is experimenting with automated Copilot reviews on Pull Requests to help contributors catch trivial issues and reduce the manual review burden for maintainers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **Future of Conbench benchmarking:** Ongoing discussion regarding the migration and maintenance of the Conbench benchmarking platform, including a potential "v2" rewrite and data preservation efforts. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Rust 59.0.0 release:** The Rust implementation is preparing for its next major release, with the latest release candidate currently under review and testing. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)

* **Flight SQL Prepared Statement updates:** A vote was held to add an `is_update` boolean to Flight SQL prepared statements to clarify whether a statement modifies data or returns a result set. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)
