* **FlightSQL Column Default Values:** Proposal to add `COLUMN_DEF` to `FlightSql.proto` to support retrieving column default value metadata, facilitating better integration for JDBC clients. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Column%20Default%20Value%20Metadata%20-%20FlightSql.proto%20Changes)

* **Erlang Implementation Donation:** The community successfully voted to accept the donation of an Erlang implementation of Arrow, which is currently based on Rust bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **Parquet Bloom Filter SIMD Optimizations:** Implementation of AVX2 and autovectorized NEON probes for Split Block Bloom Filters to provide significant performance gains on the Parquet read path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet/bloom_filter.cc)

* **FlightSQL Dialect Metadata:** Proposal to add new SqlInfo codes to the Flight SQL protocol to help clients identify backend-specific grammar for LIMIT/OFFSET, NULL ordering, and literals. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **ADBC Partitioned Bulk Ingest:** Introduction of a new API for distributed write workloads that allows coordinators to manage parallel ingestions with atomic commit and abort semantics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Canonical Range Extension Type:** Discussion on defining a standard `arrow.range` extension type to represent bounded mathematical intervals across different Arrow implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Automated GitHub Copilot Reviews:** Trialing an automated workflow where GitHub Copilot provides initial PR reviews to help catch trivial issues and reduce maintainer burden. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **Conbench Benchmarking Infrastructure:** Discussion regarding the migration of Arrow's benchmarking data to new servers and a potential "v2" redesign to improve stability and reduce costs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20+%20conbench%20OSS%20project)

* **Arrow Rust 59.0.0 Release:** The Rust implementation reached version 59.0.0, following several release candidates to address verification issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)
