* **Conbench v2 modernization:** Wes McKinney introduced a proposal and prototype for Conbench v2, featuring a Go backend, Svelte frontend, and Parquet-based storage to improve the performance and maintainability of Arrow's benchmarking infrastructure. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **JSON representation of Arrow schemas:** Discussion on creating a canonical, human-readable JSON format for Arrow schemas to improve ergonomics in non-binary contexts like web API contracts and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating IPC Tensor messages:** A proposal to deprecate the experimental Tensor and SparseTensor IPC messages in favor of using the more modern FixedShapeTensor and VariableShapeTensor canonical extension types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages?)

* **CI resource optimization:** Following pressure on ASF shared GitHub Actions resources, the project successfully reduced its runner consumption by approximately 27% through workflow streamlining and concurrency management. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **ADBC extension for DuckDB:** Discussion on the potential donation of a hand-coded DuckDB extension for ADBC to provide an official, ASF-governed client for connecting DuckDB to various database systems. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Variant type implementation:** Coordination continues regarding the complex "Variant" type for semi-structured data, focusing on ensuring C++ and Parquet implementations achieve native performance and spec compliance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **Automated GitHub Copilot reviews:** Debate on enabling automated AI code reviews for pull requests, focusing on setting community expectations for how contributors should respond to AI-generated feedback. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **Erlang repository transfer:** The community successfully voted to accept the donation of the Apache Arrow Erlang implementation, with the repository now officially transferred to the ASF infrastructure. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RESULT%20VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **Bounded range extension types:** A proposal to add `arrow.range` as a canonical extension type, with technical discussions on how to handle openness/closedness for range bounds at both the type and value level. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Bug squashing initiative:** Maintainers are calling for contributors to help clear out stale issues and reproduce old bug reports to improve the overall health of the project's issue tracker. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
