* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which is currently based on bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **IBM Db2 support proposal:** IBM engineers proposed adding Arrow support for Db2 via an ADBC driver and potentially native Arrow Flight SQL integration for improved analytical performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **ADBC Partitioned Bulk Ingest API:** A proposal to add a new API for parallelized writes, allowing distributed engines like Spark and Flink to coordinate atomic commits across multiple connections. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Parquet Bloom Filter optimizations:** Discussion on using AVX2 and branchless autovectorization to significantly speed up Split-Block Bloom Filter (SBBF) probes in the C++ and Go implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Canonical Range extension type:** Proposing a new `arrow.range` extension type to represent bounded mathematical intervals, including logic for handling inclusive/exclusive boundaries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Metadata UTF-8 enforcement:** Technical debate on whether key-value metadata in schemas and fields should be strictly restricted to valid UTF-8 to align with Flatbuffers and IPC constraints. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Automated GitHub Copilot reviews:** The project is enabling automated AI-driven PR reviews to help catch trivial issues and alleviate the maintenance burden on human reviewers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **Flight SQL Dialect metadata:** Proposal to add new `SqlInfo` codes for dialect-specific features like LIMIT/OFFSET syntax and NULL ordering to help clients generate backend-compatible SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)
