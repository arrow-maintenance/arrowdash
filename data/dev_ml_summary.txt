* **Arrow 24.0.0 release:** Voting and verification are underway for the main project's version 24.0.0 release, with discussions focused on resolving specific Gandiva test failures on macOS and S3 timeout issues in Almalinux 8. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Proposal for Canonical BigDecimal Extension Type:** A new extension type is proposed to support arbitrary-precision decimals with variable scale per row, aimed at providing high-fidelity transport for numeric types used in PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Numba JIT support for PyArrow:** Discussion on integrating a Numba-based layer directly into PyArrow to enable high-performance scientific calculations on Arrow data without manual conversion to NumPy. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Release of Apache Arrow ADBC 23:** The community announced the release of ADBC version 23, including significant updates to C++, Go, Java, and Python drivers for improved database connectivity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **JDK 17 minimum for Java 20.0.0:** Developers reached a lazy consensus to upgrade the minimum requirement for the Arrow Java implementation to JDK 17, starting with the upcoming 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Donation of pyarrow-stubs:** The project accepted the donation of the pyarrow-stubs codebase, which will move PEP 484 type stub annotations into the official repository to improve the Python development experience. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **C Device Data Interface experimentality:** Discussion on whether the C Device Data Interface should remain marked as "experimental," with maintainers debating adoption levels for non-CUDA devices and future async support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **LargeMap format addition:** A proposal to add a LargeMap type to the official Arrow specification, allowing maps to use 64-bit offsets to prevent casting issues in engines that primarily use large offset types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Flight SQL protocol improvements:** A vote was initiated to add an `is_update` field to the `ActionCreatePreparedStatementResult` to help clients distinguish between query and update execution paths in Flight SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Dictionary-encoded extension types in IPC:** Maintainers are exploring additions to the IPC specification to better distinguish between dictionaries containing extension values and extension types using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
