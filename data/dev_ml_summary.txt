* **Arrow 24.0.0 release:** The community approved and released Apache Arrow 24.0.0, which includes over 250 resolved issues across the project's various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **PyArrow type stubs donation:** The community voted to accept the donation of the pyarrow-stubs codebase, which will bring official PEP 484 type annotations into the core Arrow repository to improve the Python developer experience. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal to add a new canonical extension type for arbitrary-precision decimals to better support variable-scale data common in engines like PostgreSQL, Trino, and Oracle. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC dictionary-encoded extension types:** Discussion on resolving an IPC serialization ambiguity where dictionary-encoded extension types cannot be distinguished from extension types using dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Arrow C Device Data Interface status:** Developers discussed the experimental status of the C Device interface and whether to stabilize it or focus on adding an async version of the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL is_update field:** Proposal to add an is_update boolean to the ActionCreatePreparedStatementResult message to allow servers to signal the correct network flow for executing prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Arrow ADBC 23 release:** The Arrow Database Connectivity (ADBC) subproject released version 23, including 41 bug fixes and improvements for its C, Go, Java, and Python libraries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **LargeMap format addition:** Proposal to extend the Arrow specification with a LargeMap type using 64-bit offsets, ensuring compatibility with libraries like Polars that rely on Large* types for large datasets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Rust implementation security policy:** Discussion on establishing a formal security policy for arrow-rs and the Rust Parquet implementation to better manage vulnerability reporting and triage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)
