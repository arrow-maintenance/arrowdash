* **Apache Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which resolves 259 issues and includes improvements across several language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Apache Arrow ADBC 23 release:** ADBC version 23 has been released, featuring 41 bug fixes and subcomponent updates for C++, Go, Python, and other drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the `pyarrow-stubs` codebase to provide native PEP 484 type annotations within the PyArrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **BigDecimal canonical extension type:** Proposal for a new canonical extension type for arbitrary-precision decimals to support variable scale per row, improving compatibility with systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **C Device Data Interface experimentality:** Discussion regarding whether to remove the experimental tag from the C Device Data Interface and if the current design effectively supports non-CUDA hardware. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL prepared statement execution path:** Proposal to add an `is_update` boolean field to prepared statement results to help clients determine whether to use the query or update execution path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IPC representation of dictionary extension types:** Investigation into an IPC serialization limitation that prevents distinguishing between a dictionary with extension values and an extension type with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type using 64-bit offsets to match the `LargeList` and `LargeString` types, addressing potential overflow issues in large-scale data processing. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Rust implementation security policy:** Proposal for a formal security policy and guidelines for the `arrow-rs` and `parquet-rs` repositories to manage vulnerability reporting. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Security%20Policy)
