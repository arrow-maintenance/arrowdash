* **JDK 17 minimum for Java 20.0.0:** The community reached consensus to raise the minimum requirement for Arrow Java to JDK 17 for the upcoming version 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **pyarrow-stubs donation:** Approval of the donation of the pyarrow-stubs codebase to the project to provide official PEP 484 type hints maintained directly within the PyArrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Apache Arrow 24.0.0 release:** Voting is underway for the version 24.0.0 release candidate, which includes over 250 resolved GitHub issues across the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **C Device Data Interface experimentality:** Discussion on the experimental status of the interface, noting that wider adoption and validation across non-CUDA hardware is needed before stabilizing the spec. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL execution paths:** Proposal to add an `is_update` boolean field to the Flight SQL protocol to help clients correctly choose between query and update execution paths for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **LargeMap format proposal:** Discussion on adding a `LargeMap` type to the specification to support 64-bit offsets, addressing compatibility issues for libraries that rely on Large types for high-volume data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Dictionary-encoded extension types in IPC:** Technical investigation into resolving an ambiguity in the IPC schema serialization that prevents distinguishing between dictionary-encoded extensions and extensions with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Apache Arrow ADBC 23 release:** The ADBC subproject released version 23, including significant updates to C#, Java, R, and Rust libraries alongside improved database abstraction features. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)
