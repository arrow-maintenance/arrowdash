* **ADBC Partitioned Bulk Ingest API:** Proposal for a new write-side API mirror to support parallel data ingestion and atomic commits, designed specifically for distributed writers like Spark and Flink. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **AVX2 Bloom Filter Optimization:** Discussion on porting SIMD-accelerated Split Block Bloom Filter (SBBF) probes to the C++ implementation to significantly improve Parquet read performance on modern CPUs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **Erlang Implementation Donation:** The community is voting on accepting the donation of an Erlang implementation of Apache Arrow, which is currently built using bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **IBM Db2 Arrow Support:** Engineers from IBM proposed adding Arrow support for Db2 via a new ADBC driver, which will be maintained under the community "driver foundry" model for better vendor independence. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **UTF-8 Enforcement for Metadata:** Debate on whether Arrow should strictly enforce UTF-8 for field and schema metadata to align with Flatbuffers requirements and prevent interoperability issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Flight SQL Execution Path Indicators:** Proposal to add an `is_update` field to prepared statements in Flight SQL, allowing drivers to distinguish between queries and DDL/DML operations without relying on string-parsing heuristics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **New Flight SQL Dialect Metadata:** Discussion on adding `SqlInfo` codes to describe backend-specific SQL features like supported boolean literals, datetime formats, and limit/offset grammar to help clients generate correct SQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Release Activity for .NET and Rust:** Apache Arrow .NET 23.0.0 was released, alongside a series of rapid Rust implementation updates (58.x) to address critical potential offset overflow issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20.NET%2023.0.0%20released)
