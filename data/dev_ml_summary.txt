* **Donation of Erlang Implementation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which provides bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **IBM Db2 Arrow Support:** Engineers from IBM proposed adding Arrow support for Db2 via ADBC and Flight SQL, leading to the creation of a new Db2 driver repository under the ADBC Foundry organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **ADBC Partitioned Bulk Ingest:** Proposal for a new API to support partitioned bulk ingestion, enabling distributed writers like Spark or Flink to coordinate atomic commits across parallel worker connections. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **SIMD Parquet Bloom Filters:** Technical discussion on porting AVX2-accelerated Split Block Bloom Filter (SBBF) probes to the C++ Parquet implementation to significantly improve read-path performance using xsimd. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **UTF8 Metadata Validation:** Proposal to enforce strict UTF8 validation for field and schema metadata in IPC serialization to ensure compliance with Flatbuffers string requirements and cross-language interoperability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field%2Fschema%2Fcustom%20metadata%20restriction%20to%20UTF8%3F)

* **Flight SQL Dialect Metadata:** Proposal to add new SqlInfo codes to Flight SQL for describing backend-specific dialect details, such as supported LIMIT/OFFSET syntax and boolean literal forms, to improve query pushdown. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **Rust Security Patch Releases:** The Rust implementation released several patch versions (58.3.0, 57.3.1, and 56.2.1) to backport critical fixes for potential offset overflow issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Planned%20patch%20releases%20this%20week)

* **ADBC Logo Selection:** Discussion on a new visual identity for the ADBC subproject, focusing on a design that maintains the Arrow family aesthetic while providing distinct differentiation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)
