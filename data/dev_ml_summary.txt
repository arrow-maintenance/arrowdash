* **Apache Arrow 24.0.0 Release:** The community released version 24.0.0, which includes over 250 resolved issues across the C++, Java, Python, and R implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the `pyarrow-stubs` codebase to provide official type annotations, improving the development experience for downstream Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Apache Arrow ADBC 23 Release:** Version 23 of the ADBC libraries was released, featuring 41 bug fixes and improvements for database access abstraction across multiple languages. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Canonical BigDecimal Extension Type:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scale per row, facilitating data transport between engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC Representation of Extension Types:** Discussion on updating the IPC specification to better distinguish between dictionary-encoded extension types and extension types that use dictionary-encoded storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL Prepared Statement Updates:** A vote to add an `is_update` field to the Flight SQL protocol to allow servers to explicitly signal whether a prepared statement should be executed as a query or an update. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap Format Addition:** Discussion on extending the Arrow specification to include a `LargeMap` type utilizing 64-bit offsets to support very large map data structures. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Rust Security Policy:** Proposed guidelines for the `arrow-rs` subproject to formalize how security vulnerabilities are reported and distinguished from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Apache Arrow Go 18.6.0 Release:** Approval of the latest release for the Go implementation, resolving build and verification script issues on modern systems. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Go%20Release%20Apache%20Arrow%20Go%2018.6.0%20RC0)

* **ADBC Logo Proposal:** The community is evaluating a new logo design for the ADBC subproject to establish a distinct visual identity within the Arrow ecosystem. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
