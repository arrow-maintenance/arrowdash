* **Numba support for PyArrow:** Discussion on integrating Numba support directly into PyArrow to enable better interoperability with the scientific Python ecosystem while managing ABI stability and maintenance concerns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Arrow 24.0.0 release:** The community successfully released version 24.0.0, which includes 259 resolved issues across the project's various language implementations and libraries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** The independently maintained `pyarrow-stubs` codebase is being donated to the Apache Arrow project to provide official PEP 484 type annotations for PyArrow users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **C Device Data Interface experimentality:** Maintainers discussed the continued "experimental" status of the C Device Data Interface, deciding to wait for more diverse, non-CUDA usage before declaring it stable. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Flight SQL protocol update:** Proposal and vote for adding an `is_update` field to `ActionCreatePreparedStatementResult` to help clients correctly distinguish between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IPC dictionary-encoded extension types:** Discussion on addressing a limitation in the IPC schema serialization that currently prevents distinguishing between dictionaries with extension values and extension types with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **BigDecimal extension type:** Proposal for a new canonical extension type to represent arbitrary-precision decimals with variable scale per row, catering to data from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type with 64-bit offsets to the Arrow specification to support datasets that exceed the limits of the standard 32-bit `Map` type. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **ADBC 23 release:** Successful release of ADBC version 23, including 41 resolved issues and updates to C, C++, Go, Java, Python, and Rust components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **ADBC Logo proposal:** Design discussion for an official logo for the ADBC subproject to give it a distinct visual identity within the broader Arrow family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
