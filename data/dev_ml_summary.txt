* **Arrow 25.0.0 Release:** The first release candidate for version 25.0.0 is currently under vote, resolving 220 issues and triggering discussions on optimizing PyArrow's `to_pylist` performance for nested types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 Overhaul:** Wes McKinney proposed a major update to the project's benchmarking infrastructure, introducing a prototype with a Go-based backend and a modern Svelte frontend for improved performance and maintainability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **CI Resource Management:** In response to high consumption of shared GitHub Actions runners, the community implemented optimizations that reduced the project's CI execution time by 27%. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Canonical JSON Schemas:** Developers are discussing the creation of a standard human-readable JSON representation for Arrow schemas to improve ergonomics for REST APIs and metadata handling in ADBC. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:JSON%20representation%20of%20Arrow%20schemas)

* **IPC Format Deprecation:** A proposal was made to deprecate experimental Tensor and SparseTensor messages in the IPC format in favor of canonical extension types which offer better integration with high-level APIs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **ADBC Extension for DuckDB:** Discussion centered on the donation of a new hand-coded ADBC client for DuckDB and how it coordinates with existing community-maintained drivers like `adbc_scanner`. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Variant Type Support:** Ongoing work to implement the Parquet Variant type in C++ includes discussions on mirroring the Rust implementation and clarifying the specification for optional value fields. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **Arrow Erlang Donation:** The Erlang implementation has officially been transferred to the Apache Arrow repository following a successful donation vote. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **Issue Backlog Cleanup:** Maintainers are seeking community help to triage stale bug reports and verify if older issues can be reproduced against modern Arrow versions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
