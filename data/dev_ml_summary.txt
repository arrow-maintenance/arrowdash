* **CI resource usage reduction:** The community successfully reduced GitHub Actions runner consumption by 27% to comply with ASF Infrastructure policies following a request to manage shared resources more efficiently. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Conbench v2 development:** Wes McKinney introduced a prototype for a rewritten benchmarking system featuring a Go backend, a Svelte frontend, and DuckDB/Parquet for improved storage performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **ADBC extension for DuckDB:** A proposal was made to donate a new, hand-coded ADBC client extension for DuckDB to the Arrow project to provide a neutral, ASF-governed collaboration point. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **JSON representation for schemas:** Developers are discussing a canonical human-readable JSON representation of Arrow schemas to simplify integration with REST APIs and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **Apache Arrow 25.0.0 release:** Version 25.0.0 was officially released, resolving 220 GitHub issues across the project's various implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Deprecating Tensor IPC messages:** A proposal suggests deprecating the experimental Tensor and SparseTensor IPC messages in favor of the newer FixedShapeTensor and VariableShapeTensor extension types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Variant type implementation:** Discussion continues on implementing the Variant type in C++ and Parquet, including a proposal to align the Parquet format to allow omitting the value field. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **C Data Interface field naming:** Discussion regarding whether the C FFI should explicitly error out or panic when encountering embedded nul bytes in field names, which are valid in Arrow but problematic for C strings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **Rust implementation releases:** The community successfully released version 59.1.0 of the Arrow Rust implementation and versions 0.14.0 and 0.14.1 of the Rust Object Store. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTERUST%20Release%20Apache%20Arrow%20Rust%2059.1.0%20RC1)

* **Old issue triage:** A call for community volunteers to help reproduce and triage old bug reports to clean up the project's issue backlog. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
