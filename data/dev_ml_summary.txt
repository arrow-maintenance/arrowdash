* **JSON representation of Arrow schemas:** Proposal for a canonical, human-readable JSON schema representation to improve ergonomics for JSON APIs and metadata boundaries where binary IPC is less suitable. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating Tensor IPC messages:** Suggestion to deprecate experimental Tensor and SparseTensor IPC messages in favor of modern canonical extension types which provide better integration with existing Arrow tooling. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Modernizing Conbench infrastructure:** Proposal for "Conbench v2" featuring a Go-based backend and updated frontend to improve performance and UI ergonomics for benchmark analysis. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Range extension type:** Discussion on the structure and naming for a proposed `arrow.range` canonical extension type to represent intervals with fixed or variable closedness. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **DuckDB ADBC extension:** Proposal to donate a DuckDB ADBC extension to the ASF to provide a neutral, governed home for high-performance connectivity between DuckDB and Arrow-compliant sources. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **CI resource optimization:** The project successfully reduced its GitHub Actions runner consumption by over 25% following infrastructure-wide pressure on shared ASF runner resources.
