* **Numba support for PyArrow:** Proposal to integrate Numba JIT support into PyArrow to enable high-performance scientific computing directly on Arrow data types. The discussion focuses on whether to maintain this integration within the main PyArrow codebase to avoid ABI issues with Arrow C++ internals. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **Java JDK 17 minimum requirement:** Consensus was reached to upgrade the minimum requirement for Arrow Java to JDK 17 starting with version 20.0.0. This move aligns the project with modern Java ecosystems while acknowledging that some users still utilize older versions like JDK 8. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSJava%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL Prepared Statement execution:** Proposal to consolidate execution modes for prepared statements and introduce a boolean field in `ActionCreatePreparedStatementResult` to indicate network flow. This change aims to improve consistency across different driver implementations while maintaining backward compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFlight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **ADBC drivers migration:** Contributors provided an update on moving BigQuery, Databricks, and Snowflake drivers to a dedicated `adbc-drivers` organization. The goal is to allow these drivers to have independent release cycles and reduced maintenance overhead for the core ADBC repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Map type field name enforcement:** Discussion regarding whether language implementations should enforce specific field names like "entries", "key", and "value" for Map types. It was noted that while the specification suggests these names, they are not strictly enforced to allow for compatibility with formats like Parquet. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type%27s%20key%2Fitem%2Fvalue%20field%20names)

* **LLMs for project maintenance:** Exploration of using Large Language Models to automate repo management tasks, such as triaging issues and applying component labels. Initial tests showed decent results in identifying appropriate labels for unlabelled GitHub issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)

* **Arrow Java 19.0.0 release:** The community successfully released Arrow Java 19.0.0, which was compiled with JDK 11 and includes various performance improvements and bug fixes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEJava%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1)

* **Rust and Go releases:** Several sub-project releases were approved, including Arrow Rust 58.1.0, Rust Object Store 0.13.2, and Arrow Go 18.5.2. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEGo%20Release%20Apache%20Arrow%20Go%2018.5.2%20RC0)

* **10-Year Community Highlights:** The project celebrated its 10th anniversary by sharing a summary of major accomplishments and community growth throughout 2025. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Apache%20Arrow%20Community%20Highlights%20for%202025)
