* **Arrow 24.0.0 release:** The community announced the release of version 24.0.0, which includes 259 resolved GitHub issues across the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donation of pyarrow-stubs:** A vote successfully passed to donate the independently maintained `pyarrow-stubs` codebase to the official Apache Arrow project, providing PEP 484 type annotations for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **LargeMap format addition:** Proposal to add a `LargeMap` type that utilizes 64-bit offsets to support map structures exceeding 2GB, ensuring better compatibility with engines like Polars that rely on large offset types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Canonical BigDecimal extension type:** Discussion on a new extension type for arbitrary-precision decimals with variable scales per row to improve high-fidelity data transport from engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **IPC representation of dictionary-encoded extension types:** A proposed specification change aims to add `KeyValueMetadata` to dictionary encoding to help distinguish between dictionary-encoded extension types and extension types with dictionary storage during IPC serialization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL prepared statement execution:** A new `is_update` field was proposed for the Flight SQL protocol to allow servers to explicitly signal whether a prepared statement should follow a query or update execution path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Async support for C Data Interface:** Developers discussed the utility of an async version of the non-device C Data Interface to separate the validation of asynchronous workflows from the experimental Device Data Interface. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Apache Arrow ADBC 23 release:** The ADBC subproject released version 23, including 41 bug fixes and improvements across C, C++, Go, Java, Python, and Rust components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)
