* **Map type field name consistency:** Discussion on whether to enforce "entries", "key", and "value" field names in Map types. While the specification does not enforce these names, maintainers highlighted that consistent naming is often required for Parquet compatibility. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Map%20type's%20key%2Fitem%2Fvalue%20field%20names)

* **Arrow Java 19.0.0 release:** The community has officially released version 19.0.0 of the Java implementation. This release includes various improvements and bug fixes, and is the last version to support JDK 11. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Java%20Release%20Apache%20Arrow%20Java%2019.0.0%20RC1)

* **JDK 17 baseline for Java 20:** Consensus was reached to move the minimum requirement for Arrow Java to JDK 17 starting with version 20.0.0. This change aligns with other projects in the ecosystem and allows for modern Java features in the core library. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Proposal to integrate Numba support directly into PyArrow to enable JIT-compiled scientific calculations on Arrow data. The discussion weighs the benefits of performance and stable ABI access against the added maintenance burden. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Flight SQL Prepared Statements:** Proposal to add a boolean field to the protocol to indicate the execution path (query vs. update) for prepared statements. This change aims to help drivers like JDBC and ADBC decide which network flow to use without relying on fragile heuristics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **ADBC driver repository migration:** Ongoing status update on moving the BigQuery, Databricks, and Snowflake drivers from the main ADBC repository to a dedicated "adbc-drivers" GitHub organization. These drivers will be removed from the main source tree in the ADBC 24 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Stability of C Device Data Interface:** Developers discussed the experimental status of the C Device Data Interface, noting it will remain experimental until it gains wider adoption across non-CUDA devices and more implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Arrow Rust 58.1.0 Release:** The community successfully voted on and released version 58.1.0 of the Rust implementation. Additionally, version 0.13.2 of the Rust Object Store implementation was approved and published. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2058.1.0%20RC1)

* **LLMs for project maintenance:** Exploration of using LLMs to automate repository tasks such as applying component labels to unlabelled issues. Initial experiments showed promising results in simplifying issue triage for maintainers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LLMs%20to%20aid%20with%20maintenance)
