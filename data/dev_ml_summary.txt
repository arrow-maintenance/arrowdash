* **Arrow 24.0.0 release:** The community released Apache Arrow 24.0.0, resolving 259 issues and addressing verification challenges related to S3 tests and Gandiva time-zone handling. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Numba JIT support for PyArrow:** Proposal to integrate a Numba-specific layer into PyArrow to enable high-performance compiled scientific computing directly on Arrow arrays. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussPython%20Numba%20support%20for%20PyArrow)

* **pyarrow-stubs donation:** The community voted to accept the donation of the `pyarrow-stubs` codebase to provide official PEP 484 type annotations for PyArrow users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **BigDecimal canonical extension type:** Discussion on a proposed extension type for arbitrary-precision decimals with variable scales per row to better support databases like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Dictionary-encoded extension types in IPC:** Dewey Dunnington identified a gap in the IPC specification regarding dictionary-encoded extension types and proposed using `KeyValueMetadata` for roundtrip support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **ADBC 23 release:** The ADBC subproject released version 23, including 41 bug fixes and updates for language-specific components including C++, Go, Python, and Java. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **LargeMap format addition:** Proposal to add a `LargeMap` type with 64-bit offsets to the Arrow specification to prevent overflow issues when encoding large processed datasets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Flight SQL execution path signaling:** Proposal to add an `is_update` boolean to prepared statements to help Flight SQL clients correctly distinguish between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **C Device Data Interface experimentality:** Discussion on why the C Device Data Interface remains experimental and the potential utility of an async version for non-CUDA CPU workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)
