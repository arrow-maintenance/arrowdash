* **Arrow Java JDK upgrade:** Discussion and consensus to upgrade the minimum requirement for Arrow Java from JDK 11 to JDK 17 starting with version 20.0.0. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Numba support for PyArrow:** Investigation into supporting a subset of PyArrow within Numba JIT-compiled functions to enable high-performance scientific calculations. Discussion involves balancing maintenance burden against the need to access unstable C++ ABIs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **Apache Arrow ADBC 23 release:** The community released version 23 of the Arrow Database Connectivity (ADBC) libraries, including 41 resolved issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Apache%20Arrow%20ADBC%2023%20Released)

* **Pyarrow-stubs donation:** Proposal to donate the independently maintained `pyarrow-stubs` library to the official project to provide better PEP 484 type annotations and long-term maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL protocol update:** A vote was initiated to add an `is_update` boolean field to prepared statement results, allowing servers to indicate the correct execution path to clients. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Arrow C Device Data Interface status:** Developers discussed the "experimental" status of the C Device Data Interface and explored the possibility of creating an async version of the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **LargeMap format proposal:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, preventing cast issues in libraries that exclusively use large offset types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **IPC dictionary-encoded extension types:** Investigation into a limitation in the IPC schema serialization that prevents dictionary-encoded extension types from being roundtripped correctly. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
