* **Arrow 24.0.0 release:** The community released Apache Arrow 24.0.0, which includes over 250 resolved issues and various improvements across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Pyarrow-stubs donation:** A vote passed to donate the `pyarrow-stubs` codebase to the project, integrating PEP 484 type annotations directly into the Arrow repository for better long-term maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scales, facilitating high-fidelity data transport from systems like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, bringing it in line with other `Large*` types like `LargeList` and `LargeString`. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **Dictionary-encoded extension types in IPC:** Investigation into IPC schema limitations that prevent distinguishing between dictionary-encoded extension types and extension types using dictionary storage, with a proposal to add metadata fields to resolve the ambiguity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL update field:** A vote to add an `is_update` boolean field to the `ActionCreatePreparedStatementResult` to help clients determine the correct execution path for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **ADBC 23 release:** Apache Arrow ADBC version 23 was released, featuring over 40 bug fixes and improvements across its database abstraction drivers and language bindings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Rust security policy:** The project is establishing a formal security policy for the Rust implementation (`arrow-rs`) to define guidelines for vulnerability reporting and distinguish between standard bugs and security flaws. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **C Device Data Interface async separation:** Discussion on separating the experimental status of the Async API from the Device API to allow for faster validation of non-CUDA asynchronous workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **ADBC logo proposal:** Discussion on adopting a new logo for the ADBC subproject to provide it with a distinct identity while maintaining visual consistency with the broader Apache Arrow family. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)
