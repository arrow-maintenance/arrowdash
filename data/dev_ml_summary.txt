* **Erlang implementation donation:** The community voted to accept the donation of a new Erlang implementation of Apache Arrow, which is currently based on Rust bindings. The repository has successfully been transferred to the Apache organization as arrow-erlang. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **AVX2 optimization for Parquet Bloom Filters:** Discussion on porting SIMD-accelerated probes for Parquet’s Split Block Bloom Filters from arrow-go to the C++ implementation. Initial benchmarks show significant performance gains for in-cache and DRAM-bound workloads using autovectorization and xsimd. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom_filter.cc)

* **New SqlInfo codes for Flight SQL and ADBC:** Proposal to add metadata codes for describing backend-specific SQL dialects, such as limit/offset grammar and literal forms. These codes aim to help clients like Spark perform more efficient and accurate SQL pushdown without hardcoded per-dialect logic. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **ADBC Partitioned Bulk Ingest API:** A new API is proposed to support distributed write workloads, allowing workers to ingest data partitions in parallel while a coordinator manages atomic commits or aborts. This would provide a standardized way to handle parallel writes for RDBMS and table formats like Iceberg. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Bounded Range canonical extension type:** Discussion on introducing `arrow.range` as a canonical extension type to represent mathematical intervals with lower and upper bounds. The proposal covers various storage layouts to support different boundary behaviors, such as those found in Pandas and PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Automated GitHub Copilot reviews:** The project is experimenting with automated AI-generated PR reviews to assist with trivial issues and reduce the burden on human maintainers. Feedback has focused on reducing noise by disabling reviews for draft PRs and establishing clear expectations for how contributors should respond to AI suggestions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **Conbench benchmarking infrastructure redesign:** Following server downtime, the community is discussing a "v2" redesign of the Conbench benchmarking platform. Plans include a more maintainable backend, better orchestration to manage donated AWS credits, and migrating historical performance data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Variant type implementation coordination:** Developers are coordinating on adding a "Variant" type to Arrow C++ to support semi-structured data, a requirement for Iceberg V3 compatibility. The discussion aims to consolidate multiple internal efforts to ensure high performance and full specification compliance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **GitHub Actions resource constraints:** ASF Infrastructure flagged Apache Arrow as a top consumer of shared GitHub-hosted runner resources, leading to long queues. The project is exploring workflow optimizations, reducing concurrency, and potentially utilizing more self-hosted runners to stay within foundation limits. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Arrow Java 20.0.0 release preparation:** Coordination has started for the upcoming Java 20.0.0 release, including issue triaging and final feature inclusions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:HEADS%20UP%20Preparing%20Arrow%20Java%2020.0.0)
