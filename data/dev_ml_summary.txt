* **Arrow 25.0.0 release:** The community approved the 25.0.0 release, which includes 220 resolved issues; a follow-up patch release was suggested to address specific PyArrow performance regressions affecting Spark users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 infrastructure overhaul:** Wes McKinney proposed a major modernization of Arrow's benchmarking infrastructure, moving to a Go backend and TypeScript frontend to improve performance and UI ergonomics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Canonical JSON representation for schemas:** Discussion on establishing a human-readable JSON format for Arrow schemas to support REST APIs and ADBC metadata where binary IPC is less ergonomic. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecation of Tensor IPC messages:** A proposal suggests deprecating experimental Tensor and SparseTensor IPC messages in favor of modern canonical extension types like FixedShapeTensorArray. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **New DuckDB extension for ADBC:** Developers from Columnar introduced a new DuckDB extension for ADBC and proposed donating it to the ASF, sparking a debate on governance and collaboration with existing community projects. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Variant type specification alignment:** Technical discussion focused on aligning Arrow and Parquet specifications to allow the omission of the "value" field in shredded Variant types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **C Data Interface safety:** Fuzz testing revealed potential crashes in the C FFI when field names contain NUL bytes, leading to a consensus on requiring better validation on the producer side. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **Reduction in GitHub Actions usage:** The project successfully reduced its CI resource consumption by over 25% following new ASF infrastructure policies regarding shared runner resources. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Issue triage "bug squash" call:** Maintainers are seeking volunteers to help reproduce and triage a large volume of older bug reports across different language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)
