* **Arrow 24.0.0 release:** Voting on the latest release candidate for Arrow 24.0.0, which resolves over 250 issues and addresses verification timeouts on certain platforms. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Numba support for PyArrow:** Discussion on integrating Numba JIT compiler support into PyArrow to allow fast computations on Arrow data while managing maintenance and ABI stability concerns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Pyarrow-stubs donation:** The community voted to accept the donation of the `pyarrow-stubs` library to provide officially maintained PEP 484 type annotations within the Arrow repository. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **BigDecimal canonical extension type:** Proposal for a new canonical extension type for arbitrary-precision decimals (BigDecimal) to support variable scale per row as seen in engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **ADBC 23 release:** Announcement and verification of the ADBC version 23 release, including updates for the C++, Go, Java, and Python drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **Java JDK 17 minimum requirement:** Confirmation of the move to set JDK 17 as the minimum required version starting with the Arrow Java 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **Flight SQL prepared statement execution path:** Voting and approval for adding an `is_update` field to the protocol to help clients determine the correct execution path for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **LargeMap format addition:** Proposal to add a `LargeMap` type to the specification to support 64-bit offsets for map types, mirroring the existing `LargeList` and `LargeString` types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **IPC dictionary-encoded extension types:** Investigation into serialization gaps for dictionary-encoded extension types and a proposal to add metadata to the IPC Schema to support these complex nestings. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
