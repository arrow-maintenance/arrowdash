* **LargeMap format addition:** Proposal to add a `LargeMap` type using 64-bit offsets to the Arrow specification to support datasets exceeding 2GB, aligning with existing `LargeList` and `LargeString` types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **pyarrow-stubs donation:** The community voted to accept the donation of the `pyarrow-stubs` project, which will integrate PEP 484 type annotations directly into the Arrow repository for better long-term maintenance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Arrow 24.0.0 Release:** Apache Arrow 24.0.0 has been released, featuring over 250 resolved issues across multiple language implementations and core components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Apache%20Arrow%2024.0.0%20released)

* **Dictionary-encoded extension types in IPC:** Discussion on addressing an IPC serialization limitation where the schema cannot distinguish between a dictionary-encoded extension type and an extension type using a dictionary as storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **BigDecimal Extension Type Proposal:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scales, facilitating high-fidelity transport between Arrow and databases like PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Flight SQL is_update field:** A vote is underway to add an `is_update` flag to prepared statement results, allowing servers to signal whether a statement should be executed as a query or an update. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **ADBC Logo Proposal:** The community is discussing a new visual identity for the ADBC subproject to align with the Arrow brand while remaining distinct. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)

* **Rust Security Policy:** Proposal to establish a formal security policy for `arrow-rs` and Parquet in Rust to provide clear guidelines for reporting vulnerabilities and distinguishing them from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **Arrow C Device Data Interface:** Discussion on the experimental status of the async non-device API and the maintenance costs of duplicating definitions for async workflows across different devices. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)
