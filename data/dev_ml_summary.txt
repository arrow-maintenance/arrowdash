* **Arrow 24.0.0 release:** The community released version 24.0.0, which includes over 250 resolved issues across the C++, Python, and Java implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **pyarrow-stubs donation:** The codebase for `pyarrow-stubs` was formally donated to the Apache Arrow project to provide better PEP 484 type support for Python developers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Canonical BigDecimal Extension Type:** A proposal for a new canonical extension type to support arbitrary-precision decimals with variable scale per row, aiding interoperability with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion on adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, preventing casting issues in engines that primarily use "Large" types like Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **IPC representation of dictionary-encoded extensions:** Developers identified a limitation in the IPC schema where dictionary-encoded extension types are ambiguous, leading to a proposal for new `KeyValue` metadata to preserve type integrity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL Prepared Statement updates:** A vote passed to add an `is_update` field to `ActionCreatePreparedStatementResult`, allowing servers to explicitly signal whether a client should use a query or update path. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust implementation security policy:** A new security policy was proposed for `arrow-rs` to provide formal guidelines for reporting vulnerabilities and distinguishing security issues from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **ADBC 23 release:** The ADBC subproject released version 23, updating various drivers including C#, Java, R, and Rust to improve database access abstractions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **ADBC logo proposal:** A formal design proposal was introduced to create a unique logo for the ADBC subproject that aligns with the broader Apache Arrow branding. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)

* **Arrow Go 18.6.0 release:** The Go implementation saw a new release candidate to address specific library updates and dependency fixes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Go%2018.6.0%20RC0)
