* **Numba support for PyArrow:** Discussion on integrating Numba JIT support directly into PyArrow to enable high-performance scientific computing on Arrow data while addressing maintenance and ABI concerns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Arrow 24.0.0 release:** The community released version 24.0.0, resolving over 250 issues and addressing verification challenges related to test timeouts and Gandiva timezone handling. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Apache Arrow ADBC 23 release:** ADBC version 23 was released, featuring 41 improvements and bug fixes across multiple language bindings including C++, Python, Go, and Rust. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)

* **Donation of pyarrow-stubs:** The community voted to accept the donation of the `pyarrow-stubs` codebase, formally integrating PEP 484 type annotations for PyArrow into the main project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Stability of C Device Data Interface:** Developers discussed the current experimental status of the C Device Data Interface and the potential for adding an asynchronous stream interface to the non-device API. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Canonical BigDecimal extension type:** A proposal was introduced for a new `BigDecimal` extension type to support arbitrary-precision decimals with variable scales, improving compatibility with database engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Flight SQL prepared statement execution:** Proposal to add an `is_update` field to the Flight SQL protocol to help clients determine whether to use the query or update execution path for prepared statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust security policy:** Discussion on a new security policy for the Rust implementations of Arrow and Parquet to clarify how to report vulnerabilities and distinguish them from standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)
