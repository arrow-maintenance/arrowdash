* **Release candidate provenance and integrity:** Discussion on improving release transparency by including SHA hashes in vote threads and ensuring all artifacts are stored on ASF-controlled hardware to prevent tampering. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20candidate%20provenance%20question)

* **Apache Arrow 10-year anniversary:** The community celebrated the project's first decade, reflecting on its growth from a 2016 vision into a universal standard for efficient columnar data exchange. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Apache%20Arrow%20is%2010%20years%20old%20%F0%9F%8E%89)

* **IPC format extensions for control flow:** Proposal to introduce an "Empty Message" type and allow `ApplicationData` on IPC messages to support custom metadata and better stream signaling. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20ApplicationData%20and%20IPC%20Empty%20Message%20Type%20Discussion%20-%20Document)

* **Equivalence of IPC file and stream formats:** Investigation into formalizing the requirement that IPC files remain readable as valid IPC streams by skipping the file header and footer. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Equivalence%20of%20IPC%20file%20and%20stream%20formats)

* **vgi-rpc - Typed Arrow RPC for Python:** Introduction of a new open-source RPC framework that replaces gRPC/Protobuf with pure Arrow IPC and leverages shared memory for high-performance local communication. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Introducing%20vgi-rpc%20%E2%80%94%20typed%20Arrow%20RPC%20for%20Python%20(and%20soon%20other%20languages))

* **Flight SQL execution path signaling:** Proposal to add a boolean field to prepared statement results to help clients explicitly distinguish between result-set queries and update-count operations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Support for 24:00:00 time values:** Discussion on extending the `TIME` type to support "24:00:00" to ensure data fidelity when round-tripping with databases like PostgreSQL and DuckDB. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:TIME%20support%20for%20240000)

* **Arrow Rust 58.0.0 release:** Completion of the latest major Rust implementation release, featuring breaking API changes and improved Parquet and IPC performance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Rust%2058.0.0%20RC1)
