* **Apache Arrow 24.0.0 release:** The community released version 24.0.0 of the core project, resolving over 250 issues across multiple language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Canonical BigDecimal extension type:** Proposal for a new extension type to support arbitrary-precision decimals (BigDecimal) with variable scales to improve interoperability with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **Dictionary-encoded extension types in IPC:** Technical discussion on clarifying how dictionary-encoded extension types are represented in the IPC format to fix issues with round-tripping complex nested types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL prepared statement updates:** Approval of a protocol change adding an `is_update` field to prepared statement results, allowing clients to automatically determine the correct execution path for DML/DDL queries. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **IBM Db2 support via ADBC:** Engineers from IBM proposed adding native Apache Arrow support for Db2, starting with a new ADBC driver and exploring potential Flight SQL integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Donation of pyarrow-stubs:** The project voted to accept the donation of the `pyarrow-stubs` codebase, which will be integrated into the main repository to provide official type hinting for Python developers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **UTF-8 restrictions for metadata:** Discussion on whether to strictly enforce UTF-8 encoding for field and schema metadata to ensure compatibility with Flatbuffers and various implementation validators. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%20schema%20custom%20metadata%20restriction%20to%20UTF8)

* **Rust implementation security policy:** Proposal to establish a formal security policy for `arrow-rs` and Parquet in Rust to handle the increasing volume of vulnerability reporting and disclosure. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **Apache Arrow ADBC 23 Release:** The ADBC subproject reached version 23, including significant bug fixes and improvements to database connectivity abstractions across multiple drivers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20ADBC%2023%20Released)
