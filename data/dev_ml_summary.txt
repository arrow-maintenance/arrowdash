* **Conbench v2 proposal:** Wes McKinney proposed a significant redesign of the Conbench benchmarking tool, featuring a new Go-based backend and a TypeScript single-page application to improve performance and maintainability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **DuckDB ADBC extension donation:** Columnar proposed donating a new DuckDB extension for ADBC to the ASF, sparking a discussion regarding the role of AI in development and coordination with existing community extensions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **CI resource optimization:** Following a request from ASF Infrastructure to lower runner consumption, the project successfully reduced its GitHub Actions usage by approximately 73% to align with foundation policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Variant type in Arrow C++:** Developers are coordinating on the implementation of the Variant type in C++, focusing on performance and maintaining parity with the existing Rust implementation to support downstream projects like Iceberg. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **GitHub Copilot review guidelines:** The community is refining its automated PR review policy, deciding to disable AI reviews for draft PRs and establishing expectations for how contributors should respond to AI-generated suggestions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Auto%20GitHub%20Copilot%20review)

* **Canonical range extension type:** Discussion continues on the implementation of `arrow.range` for bounded ranges, specifically regarding naming conventions and how to represent boundary closedness at the type vs. value level. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Erlang implementation donation:** The donation of the Apache Arrow Erlang implementation is complete, with the repository successfully transferred to the official Apache organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **Rust and Object Store releases:** The community approved the releases of Arrow Rust 59.0.0 and the Object Store 0.14.0 crate, featuring various performance improvements and bug fixes. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20Rust%2059.0.0%20RC2)
