* **Numba support for PyArrow:** Discussion on integrating Numba support directly into PyArrow to allow Just-In-Time compilation of वैज्ञानिक calculations on Arrow data types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Python%20Numba%20support%20for%20PyArrow)

* **Java JDK 17 baseline:** Consensus was reached to move the minimum requirement for Arrow Java to JDK 17 starting with the 20.0.0 release. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Java%20Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **PyArrow type stubs donation:** The community voted to accept the donation of the pyarrow-stubs codebase to provide native PEP 484 type annotations within the project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20PYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Arrow 24.0.0 release:** Voting is underway for the major April release, which includes 258 resolved GitHub issues across all implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **ADBC driver migration:** Several ADBC drivers, including BigQuery and Snowflake, have been successfully migrated to the new adbc-drivers community organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20drivers%20contrib%20org)

* **Flight SQL protocol update:** A new `is_update` field was proposed for prepared statement results to help clients distinguish between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Flight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **C Device Data Interface stability:** Developers discussed whether the C Device Data Interface should remain experimental or be promoted to stable to support better async integration in ADBC. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **LargeMap format addition:** Proposal to add a `LargeMap` type to the Arrow specification to support 64-bit offsets for map entries in large-scale datasets. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20LargeMap%20format%20addition)

* **Dictionary-encoded extension types in IPC:** Investigation into a gap in the IPC schema serialization that prevents roundtripping dictionary-encoded extension types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)
