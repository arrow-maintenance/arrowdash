* **Flight SQL Prepared Statement Execution:** Discussion on adding an `is_update` field to prepared statement results to help drivers distinguish between queries and DDL/DML statements. This aim is to eliminate client-side heuristics and handle lazy execution issues in servers like GizmoSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Flight%20SQL%20Indicating%20execution%20path%20for%20Flight%20SQL%20Prepared%20Statements)

* **Arrow Erlang Implementation Donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, which is currently built on Rust bindings. The process included navigating IP clearance and Individual Contributor License Agreements for the contributing authors. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** Proposal for a new API to support distributed, atomic bulk ingestion, allowing multiple workers to write data partitions that are later committed by a coordinator. This is intended to support distributed engines like Spark and Flink and table formats like Iceberg. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **AVX2 Parquet Bloom Filter Optimizations:** Developers are porting SIMD (AVX2) optimizations for Parquet Split-Block Bloom Filters from Arrow Go to Arrow C++. The implementation uses xsimd to improve probe performance for readers like ClickHouse and Velox. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%20bloom_filter.cc)

* **IBM Db2 Support for Arrow:** IBM engineers are working on adding Arrow support for Db2, starting with an ADBC driver maintained in the community driver foundry. Future plans may include Flight SQL integration for high-performance data transport. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Metadata UTF-8 Restrictions:** Discussion regarding strictly enforcing UTF-8 encoding for field and schema metadata to ensure compatibility with Flatbuffers and various language implementations. This follows reports of non-UTF8 data being accepted by PyArrow but causing issues in other readers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%20schema%20custom%20metadata%20restriction%20to%20UTF8%3F)

* **ADBC Logo Proposal:** A dedicated logo for the ADBC subproject has been proposed and discussed to provide better visual differentiation within the Arrow ecosystem. The design focuses on maintaining branding consistency with the core Arrow project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)

* **Arrow Go 18.6.0 Release:** The community approved the release of Arrow Go 18.6.0. The release process included a fix to the verification script regarding how the Apache KEYS file is downloaded. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Go%20Release%20Apache%20Arrow%20Go%2018.6.0%20RC0)

* **Arrow .NET 23.0.0 Release:** Version 23.0.0 of the .NET implementation was officially released, including updates to Flight and Flight SQL components. The release also introduced a new PowerShell-based verification script for Windows environments. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%20.NET%2023.0.0%20released)

* **Community Over Code Hackathon Planning:** Coordination has begun for an Arrow project hackathon during the Community Over Code Glasgow event in October 2026. Maintainers are curating a list of "needs champion" issues for attendees to work on. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Hackathon%20at%20Community%20Over%20Code%20Glasgow%202026%20%E2%80%94%20interest%20from%20Arrow%3F)
