* **Arrow 25.0.0 release:** The community approved the release of version 25.0.0, which includes over 200 resolved issues across all implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 development:** Proposal to modernize the benchmarking infrastructure by moving to a Go-based backend and a Svelte-based frontend for better performance and maintainability. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **JSON representation for Arrow schemas:** Discussion on establishing a canonical human-readable JSON format for schemas to improve interoperability in web APIs and ADBC metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Format%20JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating IPC Tensor messages:** Proposal to deprecate specific Tensor and SparseTensor IPC messages in favor of using regular extension types, reducing maintenance and security overhead. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20IPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **DuckDB extension for ADBC:** A new extension allows DuckDB users to connect to ADBC-compliant databases, with discussions regarding its donation to the Arrow project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **CI resource optimization:** Developers successfully reduced the project's GitHub Actions usage by over 25% to align with Apache Infrastructure's shared resource policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub-hosted%20Runner%20Resources)

* **Variant type implementation:** Ongoing work to implement the Variant type in C++, focusing on maintaining parity with the existing Rust implementation and ensuring strict validation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20Type%20Support)

* **C Data Interface field name safety:** Investigation into how producers should handle UTF-8 field names containing embedded null bytes to avoid issues when consumed via the C FFI. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **Issue backlog triage:** A call for community help to reproduce and validate older bug reports to improve project health and reduce the number of stale issues. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)

* **Arrow Rust 59.1.0 release:** Version 59.1.0 of the Rust implementation has been released, continuing the project's frequent release cadence for the Rust crates. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20RUST%20Release%20Apache%20Arrow%20Rust%2059.1.0%20RC1)
