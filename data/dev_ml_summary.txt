* **DuckDB ADBC extension donation:** Columnar proposed donating a hand-coded DuckDB extension for ADBC to the Arrow project to provide a neutral, community-governed client for connecting DuckDB to ADBC-compliant databases. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Deprecation of Tensor IPC messages:** Proposal to deprecate the experimental Tensor and SparseTensor IPC messages in favor of using modern canonical extension types like FixedShapeTensor. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Canonical JSON representation of schemas:** Discussion on establishing a standard JSON format for Arrow schemas to improve ergonomics for REST APIs and hand-authoring metadata. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **Apache Arrow 25.0.0 release:** The project announced the release of version 25.0.0, which resolves over 220 issues and includes significant performance optimizations for nested data conversions. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Semantics of non-nullable nested fields:** Technical debate regarding the definition of the `nullable` flag for child fields when a parent structure, such as a Struct, contains nulls. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Semantics%20of%20non-nullable%20fields%20with%20non-trivial%20types)

* **Conbench v2 benchmarking migration:** Updates on the migration to Conbench v2, featuring a Go-based backend and modern frontend to improve benchmark visibility and performance tracking. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Variant and Parquet spec alignment:** Identification of inconsistencies between the Arrow Variant extension and the Parquet shredding spec, specifically concerning support for unsigned integer types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Variant%20extension%20spec%20inconsistent%20with%20the%20Parquet%20shredding%20spec)

* **C Data Interface field name validation:** Discussion on how the C FFI should handle embedded null bytes in UTF-8 field names to ensure security and prevent crashes during import. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)
