* **Conbench v2 overhaul:** Wes McKinney proposed a major architecture update for Conbench featuring a Go backend and a modern web frontend to improve performance and address slowness in the current benchmarking suite. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **DuckDB ADBC extension:** A new extension for DuckDB was introduced that enables direct querying of external databases via ADBC drivers, sparking community discussion on project governance and potential donation to the ASF. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **Deprecating Tensor IPC messages:** Developers proposed deprecating dedicated Tensor and SparseTensor messages in the IPC format in favor of using newer canonical extension types for N-dimensional data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DiscussIPC%20Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **JSON Schema representation:** Discussion is underway regarding a canonical JSON representation for Arrow schemas to support human-readable metadata for REST APIs and ADBC. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20JSON%20representation%20of%20Arrow%20schemas)

* **Apache Arrow 25.0.0 release:** The community released version 25.0.0, which includes over 200 resolved issues and performance optimizations for Python UDFs used in Spark. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Release%20Apache%20Arrow%2025.0.0%20%2D%20RC1)

* **CI resource efficiency:** Maintenance efforts resulted in a 73% reduction in GitHub Actions usage for the main repository to comply with ASF Infrastructure resource policies. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Consumption%20of%20ASF%20Shared%20GitHub%2Dhosted%20Runner%20Resources)

* **C Data Interface field names:** Technical discussion explored how the C Data Interface should handle NUL bytes in field names, recommending producer-side validation to ensure safety during FFI transfers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Spec%20clarification%20on%20field%20names)

* **Variant format alignment:** A proposal to allow the omission of the `value` field in the Variant format was submitted to align the Parquet-format and Arrow canonical extension specifications. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Allow%20VARIANT%20value%20field%20to%20be%20omitted)

* **Issue triage call for action:** Community members issued a call for volunteers to help reproduce bugs and triage older issues to maintain a healthy project backlog. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20%2D%20extra%20help%20wanted)
