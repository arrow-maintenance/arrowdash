* **Erlang implementation donation:** The community voted to accept the donation of an Erlang implementation of Apache Arrow, expanding the ecosystem to the BEAM VM via bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTE%20Donate%20Apache%20Arrow%20Erlang)

* **ADBC Partitioned Bulk Ingest API:** A new proposal introduces a write-side mirror to the existing partitioned read API, enabling coordinated parallel data ingestion for distributed engines like Spark and Flink. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Parquet Bloom Filter performance:** Discussion and PR to implement SIMD-accelerated (AVX2/NEON) probes for Parquet Bloom Filters, yielding significant performance gains for filtering operations on modern CPUs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20AVX2%20SBBF%20probe%20for%20parquet%2Fbloom%5Ffilter.cc)

* **Canonical extension type for ranges:** Proposal to add `arrow.range` to represent mathematical intervals (e.g., `[lower, upper)`), facilitating better interoperability with pandas and PostgreSQL range types. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Add%20arrow.range%20canonical%20extension%20type%20for%20bounded%20ranges)

* **Strict UTF-8 enforcement for metadata:** Developers are debating whether the C++ and IPC implementations should strictly enforce UTF-8 for field and schema metadata to ensure specification compliance. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%20schema%20custom%20metadata%20restriction%20to%20UTF8%3F)

* **Flight SQL Dialect and Metadata additions:** Proposals were made to add new SqlInfo codes to identify backend SQL dialects (e.g., LIMIT/OFFSET syntax) and include column default values in metadata results. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20FlightSQL%20Add%20dialect-related%20SqlInfo%20codes)

* **Automated GitHub Copilot reviews:** The project is experimenting with automated AI-generated PR reviews to help catch trivial issues early and alleviate the high workload on human maintainers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20Auto%20GitHub%20Copilot%20review)

* **IBM Db2 ADBC Driver:** IBM engineers proposed and initiated work on a new ADBC driver for Db2, which will be developed within the community-maintained `adbc-drivers` organization. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20DB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)
