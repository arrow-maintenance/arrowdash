* **Arrow 24.0.0 release:** The community released Apache Arrow 24.0.0, resolving over 250 issues across the project's various language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2024.0.0%20-%20RC0)

* **Donate pyarrow-stubs to Apache Arrow:** A vote passed to donate the independently maintained pyarrow-stubs repository to the project to provide native PEP 484 type annotations for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Proposal for a Canonical BigDecimal Extension Type:** A proposal was introduced for a new canonical extension type to support arbitrary-precision decimals with variable scales, facilitating high-fidelity data transfer from engines like PostgreSQL. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **LargeMap format addition:** Discussion began regarding the addition of a LargeMap type using 64-bit offsets to support large-scale map data without precision loss during conversion from engines like Polars. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)

* **IPC representation of dictionary-encoded extension types:** Developers identified a limitation in IPC serialization where dictionary-encoded extension types are indistinguishable from extensions with dictionary storage, proposing the addition of KeyValueMetadata to resolve the ambiguity. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **Flight SQL is_update field addition:** A new is_update field was proposed for prepared statement results to allow servers to explicitly signal the correct execution path for DML and DDL statements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust Security Policy:** The Rust community is discussing a formal security policy for arrow-rs and parquet-rs, including new guidelines for distinguishing between standard bugs and security vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Security%20Policy)

* **Experimentality of Arrow C Device Data Interface:** Discussion centered on whether to separate the experimental status of the async API from the Device Data Interface to encourage faster validation of async workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **Apache Arrow ADBC 23 release:** The ADBC subproject released version 23, including approximately 40 bug fixes and improvements across its database access abstraction components. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **ADBC Logo Proposal:** A formal logo for the ADBC subproject has been proposed to provide distinct branding while maintaining visual alignment with the main Apache Arrow project. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Logo%20Proposal)
