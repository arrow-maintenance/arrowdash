* **ADBC Partitioned Bulk Ingest API:** Proposal for a new write-side API that allows distributed systems like Spark or Flink to perform parallel data ingestion with atomic commit/rollback capabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Partitioned%20Bulk%20Ingest%20API)

* **Erlang implementation donation:** The community voted to accept a new Erlang implementation of Arrow into the project, which currently provides bindings to the Rust implementation. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20Apache%20Arrow%20Erlang)

* **SIMD acceleration for Parquet Bloom filters:** Discussion on utilizing xsimd to port AVX2-accelerated Split Block Bloom Filter (SBBF) probes to the C++ implementation, offering significant performance gains for readers. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:AVX2%20SBBF%20probe%20for%20parquet/bloom_filter.cc)

* **Metadata UTF-8 enforcement:** Technical debate regarding whether Arrow should strictly validate that field and schema metadata is UTF-8 encoded to ensure compatibility with Flatbuffers IPC requirements. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Field/schema/custom%20metadata%20restriction%20to%20UTF8)

* **New Flight SQL codes for SQL pushdown:** Proposal to add SqlInfo codes that allow clients to identify backend-specific SQL dialects for operations like LIMIT/OFFSET, NULL ordering, and boolean literals. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20dialect-related%20SqlInfo%20codes)

* **IBM Db2 support for ADBC:** Proposal from IBM engineers to contribute a Db2 driver for ADBC, which will be maintained within the community-led driver foundry. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Rust patch releases for security:** Coordination and release of multiple Rust implementation patches (58.3.0, 57.3.1, 56.2.1) to address potential offset overflow vulnerabilities. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Planned%20patch%20releases%20this%20week)

* **pyarrow-stubs donation:** Successful completion of the IP clearance process to bring the pyarrow-stubs library into the official Apache Arrow project for improved Python typing support. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Column default values in Flight SQL:** Discussion on modifying the Flight SQL protocol to include column default value metadata, enabling better integration with JDBC and other database clients. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Column%20Default%20Value%20Metadata%20-%20FlightSql.proto%20Changes)
