* **Numba support for PyArrow:** Discussion on integrating Numba support into PyArrow to enable JIT-compiled scientific calculations on Arrow data while managing the maintenance burden and ABI concerns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Numba%20support%20for%20PyArrow)

* **Java JDK 17 baseline:** The community reached a consensus to set JDK 17 as the minimum requirement for the upcoming Arrow Java 20.0.0 release to modernize the codebase. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Set%20JDK%2017%20minimum%20for%20Arrow%20Java%2020.0.0)

* **ADBC 23 release:** Apache Arrow ADBC version 23 was released, featuring 41 resolved issues across C/C++, Go, Python, Java, and other language implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%20ADBC%2023%20-%20RC1)

* **pyarrow-stubs donation:** A vote passed to donate the independently maintained pyarrow-stubs codebase to the Apache Arrow project to provide better type hints and static analysis for PyArrow users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **Flight SQL is_update field:** A proposal and vote were initiated to add an `is_update` field to `ActionCreatePreparedStatementResult` to help clients correctly choose between query and update execution paths. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **C Device Data Interface experimentality:** Developers discussed the experimental status of the C Device Data Interface and explored adding async support to improve ADBC integration for non-CUDA workflows. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Experimentality%20of%20Arrow%20C%20Device%20Data%20Interface)

* **IPC representation for extension types:** Discussion on a limitation in the IPC format where dictionary-encoded extension types cannot be correctly distinguished from extension types with dictionary storage. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:IPC%20representation%20of%20dictionary-encoded%20extension%20types)

* **LargeMap format addition:** A new proposal suggests adding a `LargeMap` type to the Arrow specification to support 64-bit offsets, addressing potential overflow issues in complex nested data. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:LargeMap%20format%20addition)
