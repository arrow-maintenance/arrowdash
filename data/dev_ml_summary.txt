* **Arrow 24.0.0 release:** The community announced the release of version 24.0.0, resolving over 250 issues across the project's libraries and implementations. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ANNOUNCE%20Apache%20Arrow%2024.0.0%20released)

* **Canonical BigDecimal extension type:** Proposal for a new canonical extension type to support arbitrary-precision decimals with variable scale per row, addressing compatibility with engines like PostgreSQL and Trino. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSFormat%20Proposal%20for%20a%20Canonical%20BigDecimal%20Extension%20Type)

* **pyarrow-stubs donation:** The community voted to accept the donation of the pyarrow-stubs codebase to improve type hinting and development experience for Python users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEPYTHON%20Donate%20pyarrow-stubs%20to%20Apache%20Arrow)

* **IBM Db2 support for Arrow:** IBM engineers proposed adding Arrow support for Db2 via ADBC and Flight SQL to enable more efficient data exchange with analytical engines. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSSADBCDB2%20Proposal%20Adding%20Apache%20Arrow%20Support%20for%20IBM%20Db2)

* **Enforcing UTF8 for metadata:** Discussion on whether to strictly enforce UTF8 validation for field and schema metadata keys and values to ensure compliance with Flatbuffers specifications. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Discuss%20Field%20schema%20custom%20metadata%20restriction%20to%20UTF8?)

* **Flight SQL is_update field:** Proposal to add an optional `is_update` boolean to prepared statement results to help drivers differentiate between queries and DDL/DML updates without heuristics. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:VOTEFlight%20SQL%20Add%20is_update%20field%20to%20ActionCreatePreparedStatementResult)

* **Rust security policy:** Discussion on establishing a formal security policy for arrow-rs to better manage vulnerability reporting and distinguish between security issues and standard bugs. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:RUST%20Discuss%20Security%20Policy)

* **ADBC logo proposal:** Discussion on adopting a specific logo for the ADBC subproject to establish its identity while maintaining visual consistency with the main Arrow brand. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:DISCUSS%20ADBC%20Logo%20Proposal)
