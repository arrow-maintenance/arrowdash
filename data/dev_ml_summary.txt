* **ADBC Extension for DuckDB:** Discussion on donating a hand-coded ADBC extension for DuckDB to the Arrow project, which prompted a debate regarding development speed, maintenance through the ASF, and the role of AI-assisted community alternatives. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:ADBC%20Extension%20for%20DuckDB)

* **JSON representation of Arrow schemas:** Proposal to establish a canonical JSON format for Arrow schemas to improve ergonomics for REST APIs and ADBC metadata handling where binary IPC is cumbersome. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:JSON%20representation%20of%20Arrow%20schemas)

* **Deprecating Tensor IPC messages:** Proposal to deprecate specialized Tensor and SparseTensor IPC messages in favor of using canonical extension types, which allow tensors to be handled as standard RecordBatch columns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Deprecate%20Tensor%20and%20SparseTensor%20messages%3F)

* **Arrow 25.0.0 Release:** The community finalized the 25.0.0 release containing over 220 resolved issues, including a significant discussion on optimizing `to_pylist` performance for Spark users. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Release%20Apache%20Arrow%2025.0.0%20-%20RC1)

* **Conbench v2 migration:** Ongoing work to modernize the Arrow benchmarking infrastructure with a new Go-based backend and a Svelte-based frontend, addressing recent performance timeouts and UI regression concerns. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Status%20of%20Arrow%20conbench%20data%20%2B%20conbench%20OSS%20project)

* **Semantics of non-nullable fields:** A deep dive into how the `nullable` flag should be interpreted for complex types, such as when a child field is marked non-nullable but its parent struct contains nulls. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Semantics%20of%20non-nullable%20fields%20with%20non-trivial%20types)

* **Parquet Variant spec alignment:** Efforts to resolve inconsistencies between the Arrow Variant extension and the Parquet shredding specification regarding permitted primitive type mappings and required fields. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Variant%20extension%20spec%20inconsistent%20with%20the%20Parquet%20shredding%20spec)

* **Bug squashing and issue triage:** A call for volunteers to help reproduce and clear out stale bug reports to improve the project's issue backlog health. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:Closing%20out%20old%20issues%20-%20extra%20help%20wanted)

* **Extended range timestamp for Parquet:** Notice of a proposal in the Parquet community to support the full SQL range for nanosecond-level timestamps, which may eventually impact Arrow's Parquet integration. [Thread](https://lists.apache.org/list?dev@arrow.apache.org:lte=2M:FYI%20Parquet%20proposal%20for%20extended%20range%20timestamp%20type)
